软件安装：RPM与YUM

distribution 代表 软件管理机制 使用指令 在线升级机制（指令）
Red Hat/Fedora RPM rpn,rpmbuild YUM（yum）
Debian/Ubuntu DPKG dpkg APT（apt-get）

所以 CentOS 使用的软件管理机制是 RPM，采用 YUM 作为软件在线升级的方式

# 软件管理程序：RPM

    rpm -ivh package_name

    选项参数 说明
    -i install 的意思
    -v 查看更细部的安装信息画面
    -h 以安装信息列显示安装进度
    rpm -ivh /mnt/Packages/rp-pppoe-3.11-5.el7.x86_64.rpm
rpm -ivh a.i386.rpm b.i386.rpm *.rpm # 同时安装多个软件
# 直接安装网络上面的软件，通过网址来安装
rpm -ivh http://website.name/path/pkgname.rpm

    • -Uvh：后面接的软件即使没有安装过，则系统将予以直接安装； 若后面接的软件有安装过旧版，则系统自动更新至新版
    • -Fvh：如果后面接的软件并未安装到你的 Linux 系统上，则该软件不会被安装；亦即只有已安装至你 Linux 系统内的软件会被升级

    查询主要分为两部分：

    • 一个是查已安装到系统上面的的软件信息，这部份的信息都是由 /var/lib/rpm/ 所提供
    • 查某个 rpm 文件内容，等于是由 RPM 文件内找出一些要写入数据库内的信息，这部份就得要使用 -qp

    注意：查询本机上面的 RPM 软件相关信息时， 不需要加上版本的名称，只要加上软件名称即可

    选项参数 说明
    -q 仅查询，后面接的软件名称是否有安装
    -qa 列出所有的，已经安装在本机 Linux 系统上面的所有软件名称
    -qi 列出该软件的详细信息，包含开发商、版本与说明等（information）
    -ql 列出该软件所有的文件与目录所在完整文件名（list）
    -qc 列出该软件的所有配置文件（找出在 /etc/ 下面的文件名）
    -qd 列出该软件的所有说明文档（找出与 man 有关的文件）
    -qR 列出与该软件有关的相依软件所含的文件
    -qf 由后面接的文件名称，找出该文件属于哪一个已安装的软件
    -q --scripts 列出是否含有安装后需要执行的脚本档
    -qp[icdlR] -qp 后面接的所有参数与以上的说明一致
    但用途仅在于找出某个 RPM 文件内的信息，而非已安装的软件信息
    rpm -q logrotate  # 查询Linux上面是否有安装 logrotate 这个软件，不需要添加版本查询
rpm -ql logrotate # 查询该软件所提供的所有目录与文件
rpm -qi logrotate # 列出 logrotate 这个软件的相关说明数据
rpm -qc logrotate # 仅找出 logrotate 的配置文件
rpm -qd logrotate # 仅找出 logrotate 的说明文档
rpm -qR logrotate # 查看软件的依赖关系
rpm -qf /bin/sh   # 找出 /bin/sh 是那个软件提供的

    验证文件变化的语法

    • rpm -Va：列出目前系统上面所有可能被更动过的文件
    • rpm -V 已安装的软件名称：若该软件所含的文件被更动过，才会列出来
    • rpm -Vp 某个 RPM 文件的文件名：后面加的是文件名称，列出该软件内可能被更动过的文件
    • rpm -Vf 在系统上面的某个文件：列出某个文件是否被更动过

    内容显式说明：

    • S：文件的容量大小是否被改变
    • M：文件的类型或文件的属性（rwx）是否被改变？如是否可执行等参数已被改变
    • 5：MD5 这一种指纹码的内容已经不同
    • D：设备的主/次代码已经改变
    • L：Link 路径已被改变
    • U：文件的所属人已被改变
    • G：文件的所属群组已被改变
    • T：文件的创建时间已被改变
    • P：功能已经被改变
    • c：配置文件（config file）
    • d：文件数据文件（documentation）
    • g：通常是该文件不被某个软件所包含，较少发生（ghost file）
    • l：授权文件（license file）
    • r：read me 文件
    rpm -qa | grep pam  # 找出与 pam 有关的软件名称
rpm -e pam          # 移除 pam 这个软件
rpm --rebuilddb     # 重建数据库

    # 在线升级机制：YUM

      指令语法：yum [option] [查询工作项目] [相关参数]

      选项参数 选项参数 说明
      [option] -y 当 yum 要等待使用者输入时，这个选项可以自动提供 yes 的回应
      --installroot=/some/path 将该软件安装在 /some/path 而不使用默认路径
      [查询工作项目] [相关参数] search 搜寻某个软件名称或者是描述（description）的重要关键字
      list 列出目前 yum 所管理的所有的软件名称与版本，有点类似 rpm -qa
      info 同上，不过有点类似 rpm -qai 的执行结果
      provides 从文件去搜寻软件，类似 rpm -qf 的功能

      指令语法：yum [option] [安装与升级的工作项目] [相关参数]

      选项参数 说明
      install 后面接要安装的软件
      update 后面接要升级的软件，若要整个系统都升级，就直接 update 即可

      yum 卸载软件：yum remove 软件名称
