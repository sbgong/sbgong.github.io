## 宽度和高度：width/height

通常，我们会明确定义一个元素的宽度，但是鉴于一定的历史原因，往往不明确定义元素的高度。默认情况下，元素的宽度指从左内边界到右内边界的距离，元素的高度指从上内边界到下内边界的距离。控制这两个距离的属性分别为 `width` 和 `height`

这两个属性有一点要注意：无法应用到行内非置换元素上。比如说，为正常流动模式下生成行内框的超链接声明的 width 和 height，在遵守 CSS 标准的浏览器中会被忽略

```
a:link {color: red; background: silver; width: 60px; height: 15px;}
```

那么，链接在未访问的状态下将呈现为银底红字，而其宽度和高度由链接的内容决定，而不是 60 像素宽，15 像素高。但是如果加上 display 属性，把值设为 inline-block 或 block，那么 width 和 height 的值将被采用，分别设定链接内容区的宽度和高度

## 内边距：padding

紧邻元素内容区的是内边距（padding），位于内容区和边框之间

各边的值可以混用不同的长度类型：`padding: 14px 5em 0,1in 3ex;`

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab "padding"
| 功能 | 说明 |
| :----: | --------------------------------- |
| 取值 | `[<length> / <percentage>] {1,4}` |
| 初始值 | 简写形式没有 |
| 适用于 | 所有元素 |
| 百分数 | 相对容纳块的宽度 |
| 继承性 | 否 |
| 动画性 | 是 |
| 备注 | padding 不能取负值 |
:::
::: tab "padding-top/padding-right/padding-bottom/padding-left"
| 功能 | 说明 |
| :----: | --------------------------------------- |
| 取值 | `<length> / <percentage>` |
| 初始值 | 0 |
| 适用于 | 所有元素 |
| 百分数 | 相对容纳块的宽度/相对父元素内容区的宽度 |
| 继承性 | 否 |
| 动画性 | 是 |
| 备注 | padding 不能取负值 |
:::
::: tab "padding 属性值顺序"

- 四个属性值：上 → 右 → 下 → 左
- 三个属性值：上 → 左/右 → 下
- 两个属性值：上/下 → 左/右
- 一个属性值：上/下/左/右

:::
::: tab "行内非置换元素"
行内非置换元素的内边距：

- 针对行内非置换元素的上下两边：应用到行内非置换元素上的内边距对行高没有任何影响。因为没有背景的内边距是透明的，但是要小心，如果行内非置换元素有背景色和内边距，其背景会向元素上下延伸（行高没有边，但是背景色延伸到内边距区域了，所以背景与前面的行出现了重叠）
- 针对行内非置换元素的左右两边：确实能看到效果，在左右两边都出现了空白；但是如果行内费置换元素断成多行，情况稍微有点变化，左内边距应用到元素的开头，右内边距应用到元素的末尾。默认情况下，内边距不应用到各行的左右两侧

```
# 上下两边
strong {padding-top: 0.5em; background-color: silver;}
# 左右两边
strong {padding-left: 25px; padding-right: 25px; background-color: silver;}
```

:::
::: tab "行内置换元素"
内边距可以应用到行内置换元素上。不管置换元素是块级还是行内的，内边距都出现在内容四周，而且背景色会填充到内边距区域。行内非置换元素的内边距对文本行的高度没有影响，但置换元素截然相反，行内置换元素的内边距对行高有很大的影响
:::
::::

## 边框

元素的内边距之外是边框，边框是元素的内容和内边距周围的一到多条线段。默认情况下，元素的背景在边框的外边界处终止，不会延伸到外边距区域。边框在外边距内侧

边框有三个要素：宽度、样式、颜色

- 边框宽度的默认值为 medium，对应的具体值视情况而定，通常为 2 像素
- 一般看不到边框，因为边框样式默认为 none
- 默认的边框颜色是元素自身的前景色，如果没有为边框声明颜色，默认与元素中的文本颜色相同；如果元素中没有文本，例如只含图像的表格，那么边框的颜色将与父元素的文本颜色相同

行内元素的边框：

- 不管把行内元素的边框设为多宽，元素的行高都不会变。然而边框是可见的，因此会渲染出来
- 左右两边的边框：如果为行内元素添加左边框或右边框，边框不仅会显示出来，还会把周围的文本推开


## 边框的样式：border-style

样式是边框最重要的要素，不是因为它控制着边框的外观，而是因为倘若没有样式，你根本看不到边框

- 一个边框可以有多个样式，多个值的顺序和 padding 一样。例如：`p.aside {border-style: solid dashed dotted solid;}`
- 单边样式：`border-top-style`、`border-right-style`、`border-bottom-style`、`border-left-style`
- 注意：若要使用单边样式，请把单边属性放在简写形式后面。因为简写形式中的值会把单边属性中的值覆盖掉

## 圆角边框：border-radius

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab "border-radius"
| 功能 | 说明 |
| :----: | -------------------------------------------------------------------- |
| 取值 | `[<length> / <percentage>]{1,4} [/ [<length> / <percentage>]{1,4}]?` |
| 初始值 | 0 |
| 适用于 | 除表格内的元素之外的所有元素 |
| 继承性 | 否 |

:::
::: tab "圆角各边的顺序"

- 设置一个值：表示 4 个圆角都使用这个值
- 设置两个值：表示左上角和右下角使用第一个值，右上角和左下角使用第二个值
- 设置三个值：表示左上角使用第一个值，右上角和左下角使用第二个值，右下角使用第三个值
- 设置四个值：依次对应左上角、右上角、右下角、左下角
- 设置两对四个值：前的四个数值表示圆角的水平半径，后面四个值表示圆角的垂直半径

```

```

|  功能  | 说明                                                                                                                                             |
| :----: | ------------------------------------------------------------------------------------------------------------------------------------------------ |
|  取值  | `<border-image-source> // <border-image-slice> [/ <border-image-width> / <border-image-width> / <border-image-outset>] // <border-image-repeat>` |
| 初始值 | none                                                                                                                                             |
| 适用于 | 除 border-collapse 属性的值为 collapse 的表格内的元素之外的所有元素                                                                              |

| :----: | ------------------------------------------------------------------- |

| 初始值 | none                                                                |
| 适用于 | 除 border-collapse 属性的值为 collapse 的表格内的元素之外的所有元素 |
| 继承性 | 否                                                                  |

## 图像边框的切割：border-image-slice

border-image-slice 属性在图像上放置四条裁剪线，这四条线围聚得到的部分用作绘制边框的图像。这个属性最多接受四个值，按顺序定义距上右下左四边的偏移，少于四个值时，也应用前面讲的复制规则。因此只提供一个值时，为四边设定相等的偏移。

|  功能  | 说明                                                                |
| :----: | ------------------------------------------------------------------- |
|  取值  | `[<number> / <percentage>]{1,4}&& fill?`                            |
| 初始值 | 100%                                                                |
| 适用于 | 除 border-collapse 属性的值为 collapse 的表格内的元素之外的所有元素 |
| 继承性 | 否                                                                  |
| 动画性 | `<number>,<percentage>`                                             |

- 该值接受数值、百分数，默认单位是 px，但不能在数值后面加 px，否则这句 CSS 将不被浏览器解析
- 四条线将背景图分隔成了 9 部分，其中 1，2，3，4，6，7，8，9 这八个区域将会被切割，作为图片边框
- 如果除了设置数值或百分数，还加了一个可选项 fill，那么区域 5 就会作为背景填充进元素 content
  - 例如：`border-image-slice: 25 11 fill;`

## 图像边框的外推边框：border-image-outset

一旦图像加载失败，或者浏览器不支持图像边框，就会留出多余的空白。为了处理这种情况，规范引入了 `border-image-outset` 属性

不管是长度还是数字，border-image-outset 都把边框图像向边框外侧推

|  功能  | 说明                                                                |
| :----: | ------------------------------------------------------------------- |
|  取值  | `[<length> / <number>]{1,4}`                                        |
| 初始值 | 0                                                                   |
| 适用于 | 除 border-collapse 属性的值为 collapse 的表格内的元素之外的所有元素 |
| 继承性 | 否                                                                  |
| 动画性 | 是                                                                  |
|  备注  | 不能取负值                                                          |

## 调整图像边框的重复方式：border-image-repeat

|  功能  | 说明                                                                |
| :----: | ------------------------------------------------------------------- |
|  取值  | `[stretch / repeat / round / space]{1,2}`                           |
| 初始值 | stretch                                                             |
| 适用于 | 除 border-collapse 属性的值为 collapse 的表格内的元素之外的所有元素 |
| 继承性 | 否                                                                  |
| 动画性 | 否                                                                  |

- repeat：平铺图像，直到占满每一边的边框区域为止，达到边界处的图像可能会被截断
- round：同样是平铺图像，但是 round 会处理得更平滑。浏览器会先拿各边的边框区域长度除以图像的尺寸，然后取整为最近的整数，平铺这一数量的图像。在这个过程中，浏览器可能会拉伸或压缩图像的尺寸，保证平铺的图像恰好相互接触
- space：space 类似与 round，每一边的边框区域也是先除以要平铺的图像的尺寸，然后取整。只不过，此时始终向下取整，而且图像不扭曲，而是均匀分布在边框区域中

## 轮廓：outline

CSS 定义了一种特殊的装饰方式，名为轮廓。轮廓对布局完全没有影响，轮廓只是视觉上的效果。

实际使用中，轮廓一般直接绘制在边框外侧，但情况并不总是这么简单。根据规范，轮廓与边框的区别体现在三个方面：

- 轮廓不占空间
- 轮廓不一定是矩形的，也不一定是连续的
- 用户代理通常在元素处于 `:focus` 状态时渲染轮廓
- 无法单独为一边设置独特的轮廓

## 外边距：margin

常规流动模式下，元素之间有间隔的原因是元素有外边距（margin）。外边距在元素周围添加额外的空白。空白一般指其他元素不能共存的区域，而且在这片区域中，父元素的背景是可见的

|  功能  | 说明                                    |
| :----: | --------------------------------------- |
|  取值  | `[<length> / <percentage> / auto]{1,4}` |
| 初始值 | 未定义                                  |
| 适用于 | 所有元素                                |
| 百分数 | 相对容纳快的宽度计算                    |
| 继承性 | 否                                      |
| 动画性 | 是                                      |

- 如果未声明值，默认没有外边距
- 支持单边属性：`margin-top`、`margin-right`、`margin-bottom`、`margin-left`、
