Linux-目录结构

# Linux 目录配置的依据——FHS

Linux 目录配置的方法——FHS 标准（FileSystem Hierarchy Standard），它们的主要目的是希望使用者可以了解到已安装软件通常放置在哪个目录下，所以他们希望独立的软件开发商、操作系统制定者以及想要维护系统的用户都能够遵循 FHS 的标准。也就是说 FHS 的重点在于规范每个特定的目录下应该要放置什么样子的数据

FHS 根据文件系统使用的频繁与否与是否允许使用者随意更动，而将目录定义为四种交互作用的形态

可分享的(shareable) 不可分享的(unshareable)
不变的(static) /usr（软件放置处） /etc（配置文件）
/opt（第三方协力软件） /boot（开机与核心文件）
可变的(variable) /var/mail（使用者邮件信箱） /var/run（程序相关）
/var/spool/news（新闻组） /var/lock（程序相关）
  • 可分享的：可以分享给其他系统挂载使用的目录（能够分享给网络上其他主机挂载用的目录）
  • 不可分享的：自己机器上运作的配置文件或是与程序有关的 socket 文件等（仅与自身机器有关的文件自然不适合分享给其他主机）
  • 不变的：有些数据是不会经常变动，例如函数库、文件说明文档、系统管理员所管理的主机服务配置文件等
  • 可变动：经常改变的数据，例如登录文件等

实际上，FHS 针对目录树架构仅定义出三层目录底下应该放置什么数据：

  • /（root 根目录）：与开机系统有关
  • /usr（unix software resource）：与软件安装/执行有关
  • /var（variable）：与系统运作过程有关

# 根路径

根路径是整个系统最重要的一个目录，因为不但所有的目录都是由根目录衍生出来的，同时根路径也与开机/还原/系统修复等动作相关。由于系统开机时需要特定的开机软件、核心文件、开机所需程序、函式库等文件数据，若系统出现错误时，根目录也必须要包含有能够修复文件系统的程序才行

因为根路径是这么重要，所以 FHS 希望根路径不要放在非常大的分区槽内，因为越大的分区槽只会放入越多的数据，如此一来根路径所在分区槽就可能有较多发生错误的机会。因此 FHS 标准建议：根目录所在分区槽应该越小越好，且应用程序所安装的软件最好不要与根目录放在同一个分区槽内，保持根目录越小越好

# FHS 要求必须存在的目录

目录 说明
/bin 存放所有用户都可以使用的基本操作命令（目录中多是可执行的二进制文件）主要有 cat/chmod/chown 等指令
/boot 存放系统启动时所需要的核心文件；包括 Linux 内核文件以及开机菜单与开机所需配置文件等
/dev 存放设备文件，如硬盘、光驱、鼠标等设备文件
/etc 存放系统主要的配置文件；一般普通用户是只读的，root 用户才能修改。FHS 建议不要将可执行文件放在这个目录中
/lib 存放系统开机时所需的函数库，以及在/bin 或/sbin 下面的命令会调用的函数库
/media 挂载点：用于挂载具有热插拔性质的设备（如 U 盘、软盘、光盘、DVD 等设备都暂时挂载在这里）
/mnt 用于临时挂载别的文件系统
/opt 第三方软件安装在这里（可选的，安装软件也可放在/usr/local 目录下），可以理解为 D:/Software。当你不需要时，直接 rm -rf 掉即可。在硬盘容量不够时，也可将/opt 单独挂载到其他磁盘上使用
/run 存放系统开机后所产生的进程相关 PID 文件；系统重启以后，这个目录会被清空；/run 目录以前是/var/run
/sbin 只有系统管理员 root 才能所使用的指令文件；如 fsck/ifconfig 等指令
/srv service 缩写，该目录存放一些网络服务启动之后需要提取的数据；如 www 服务需要访问的网页数据存放在/srv/www 内；不过还是建议放到/var/lib 底下
/tmp 让任何人或是正在执行的程序暂时存放文件的地方（存放各类临时文件）。通常在重启后清空，重要文件不要放这里
/usr linux 文件系统的第二大部分，重要程度仅次于根目录，里面存放可共享的只读文件；软件默认的安装位置，存放用户应用程序和文件，类似于 Windows 下的 Program Files
/var 存放一些经常变化的文件；这里的可变文件通常是会累积的数据，就像日志和缓存一样；建议单独分区，设置较大的磁盘空间

# FHS 建议可以存在的目录

目录 说明
/home 普通用户的家目录；在 Linux 中每个用户都有一个自己的目录（/home/普通用户名）
/root 超级用户 root 的家目录
/lib<qual> 用来存放与/lib 不同格式的二进制函数库，例如支持 64 位的/lib64 函数库

# 其他部分目录

目录 说明
/lost+found 这个目录是使用标准的 ext2/ext3/ext4 文件系统格式才会产生的一个目录，目的在于当文件系统产生错误时，将一些遗失的片段放置到这个目录下。如果采用的是 xfs 文件系统的话，就不会存在这个目录了
/proc 内核和进程信息虚拟文件系统；目录中所有信息都是内存的映射，通过这个虚拟的内存映射目录，可以和内核内部数据结构进行交互，获取有关进程的有用信息，同时也可以在系统运行中修改内核参数。与其他目录不同，/proc 存在于内存中，而不是硬盘上；例如进程、外部设备状态、网络状态等
/sys 内核和系统信息虚拟文件系统，同样不占硬盘容量；使用的是 sysfs 文件系统，存放的是关于设备、驱动程序和一些内核特性的信息

# /usr

系统默认的软件都会放置到/usr 下面，因此这个目录有点类似 Windows 系统的『C:\Program Files + C:\Windows\当中的一部分』这两个目录的综合体

# FHS 要求必须存在的目录

目录 说明
/usr/bin/ 系统预装的可执行程序，会随着系统升级而改变
/usr/lib/ 系统预装的可执行程序需要用到的函数库；如果是 X86_64 的 Linux 系统，那可能会有/usr/lib64/目录产生；理解为 C:/Windows/System32
/usr/local/ 用户级的程序目录，可以理解为 C:/Progrem Files/。用户自己编译的软件默认会安装到这个目录下
/usr/sbin/ 存放超级用户才能使用的应用程序，一般是与服务器软件程序命令相关的
/usr/share/ 放置共享文件的地方，在这个目录下放置的数据几乎是不分硬件架构均可读取的数据，因为几乎都是文本文件
  • /bin：存放所有用户皆可用的系统程序，即普通的基本命令，如：touch、ls
  • /sbin：存放超级用户才能使用的系统程序，即基本的系统命令，如：reboot、fdisk
  • /usr/bin：存放所有用户都可用的应用程序，你后期安装的一些软件的运行脚本，如：gcc、make、wget
  • /usr/sbin：存放超级用户才能使用的应用程序，一般是与服务器软件程序命令相关的，如：dhcpd、httpd
  • /usr/local/bin：存放所有用户都可用的第三方软件程序，如 MySQL
  • /usr/local/sbin：存放超级用户才能使用的第三方软件，如 Nginx
  • /lib：存放系统开机时会用到的函数库，以及/bin 和/sbin 目录下程序所需调用的函数库（内核级的）
  • /usr/lib：存放系统自带的可执行程序用到的函数库（系统级的）
  • /usr/local/lib：存放用户下载的应用程序可执行程序用到的函数库（用户级的）
  • /usr/share/man：联机帮助文件
  • /usr/share/info：联机帮助文件
  • /usr/share/doc：软件杂项的文件说明
  • /usr/share/zoneinfo：与时区有关的时区文件

# FHS 建议可以存在的目录

目录 说明
/usr/games/ 包含与游戏相关的文件，现在很少用到
/usr/include/ 各种系统头文件，编译文件时需要使用
/usr/libexec/ 某些不被一般使用者惯用的执行文件或脚本等都会放置在此目录中，例如大部分的 X window 中的操作指令很多都放在此目录下
/usr/lib<qual>/ 用来存放与/usr/lib 不同格式的二进制函数库，例如支持 64 位的/usr/lib64 函数库
/usr/src/ 系统级的源码目录
  • /usr/local/src：用户级的源码目录

# /var

目录 说明
/var/cache/ 应用程序本身运作过程中产生的一些暂存文件
/var/lib/ 程序本身执行过程中需要使用到的数据文件放置目录。各自软件都应该要有各自的目录
/var/lock/ 某些装置或者文件资源一次只能被一个应用程序所使用，如果同时有两个程序使用该装置时，就可能产生一些错误的状况，因此就得要将该装置上锁(lock)，以确保装置只会给单一软件所使用的。目前该目录也已经挪到/run/lock 中
/var/log/ 登录文件放置的目录
/var/mail/ 放置个人电子邮件信箱的目录，不过这个目录也已经挪到/var/spool/mail 中
/var/run/ 某些程序或者是服务启动后，会将它们的 PID 放置在这个目录下，这个目录被链接到/run 去了
/var/spool/ 放置一些队列数据（队列数据就是排队等待其他程序使用的数据），这些数据被使用后通常都会被删除

# CentOS 7 变化

CentOS 7 将原来应该要在根目录(/)里面的目录全部挪到/usr 里面去了，然后进行链接设定

  • /bin → /usr/bin
  • /sbin → /usr/sbin
  • /lib → /usr/lib
  • /lib64 → /usr/lib64
  • /var/lock → /run/lock
  • /var/run → /run
