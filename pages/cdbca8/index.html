Linux-计算机概论

计算机其实是『接受用户输入指令与数据，经由中央处理器的数学与逻辑单元运算处理后，以产生或储存成有用的信息』。因此只要有输入设备和输出设备，让你可以输入数据使该机器产生信息的，那就是一部计算机了

# 计算机硬件五大单元

计算机主要分为三部分:

  • 输入单元：包括键盘、鼠标、扫描仪、手写板、触摸屏等
  • 中央处理器（CPU）：含有算数逻辑、控制、记忆等单元
  • 输出单元：例如屏幕、打印机等

CPU 是一个具有特定功能的芯片，里面有微指令集。它主要用于管理和计算，因此可以分为两个主要的单元：控制单元 和 算数逻辑单元。控制单元主要协调各组件和各单元之间的工作；算数逻辑单元主要负责程序运算和逻辑判断

  • CPU 读取的数据都是从内存读取来的，内存的数据则是从输入单元传输进来的
  • CPU 处理完的数据也必须要先写回内存中，然后内存中的数据再传输到输出单元

  • 系统单元：包括 CPU、内存及主板相关组件
  • 记忆单元：包括主存储器（RAM:main memory）与辅助内存（硬盘/光盘）
  • 输入/输出单元：输入设备键盘鼠标；输出设备屏幕打印机

主存储器，简称为主存。其作用是存放指令和数据，并能由 CPU 直接随机存 内存：它是与 CPU 进行沟通的桥梁，计算机中所有程序的运行都是在内存中进行的，因此内存的性能对计算机的影响非常大。内存也被称为内存储器，其作用就是暂时存放 CPU 中的运算数据，以及与硬盘等外部存储器交换的数据

# CPU 微指令集

CPU 指令集的设计主要被分为两种设计理念

  • 精简指令集（RISC）：微指令集较为精简，每个指令的运行时间都很短，完成的动作也很单纯，指令的执行效能较佳；但若要做复杂的事情，就要由多个指令来完成。常见的 RISC 微指令集 CPU 有 ARM CPU 系列，在应用方面就是手机、网络设备（交换机/路由器）、导航系统
  • 复杂指令集（CISC）：每个小指令可以执行一些较低阶硬件操作，指令数目多而且复杂，每条指令的长度并不相同。因为指令执行较为复杂所以每条指令花费的时间较长，但每条个别指令可以处理的工作较为丰富。常见的微指令 CPU 主要有 AMD、Intel等的 x86 架构的 CPU。由于 AMD、Intel 所开发出的 x86 架构 CPU 被大量使用于个人计算机，因此个人计算机常被称为 x86 架构的计算机

x86 叫法的由来

  • 最早的那颗 Intel 发展出来的 CPU 代号称为 8086，后来依次架构又开发出 80286、80386……，因此这种架构的 CPU 就被称为 x86 架构了
  • 在 2003 年，原来的 x86 架构 CPU 由 8 位升级到 16、32 位，后来 AMD 依此架构修改新一代的 CPU 为 64 位，为了区别两者的差异，64 位的个人计算机 CPU 又被统称为 x86_64 架构
  • 所谓的位指的是 CPU 依次数据读取的最大量——64 位 CPU 代表 CPU 一次可以读写 64bits 的数据。因为 CPU 读取数据量有限制，因此能够从内存中读写数据也就有所限制

# 计算机分类

以计算机的复杂度与运算能力进行分类

  • 超级计算机：是运行速度最快的计算机，但是它的维护、操作费用也最高。主要用于需要高速计算的计划中。例如国防军事、气象观测、太空科技等
  • 大型计算机：通常也具有整个高速的 CPU，功能虽不及超级计算机，但也可用来处理大量资料与复杂的运算。例如大型企业的主机、全国性的证券交易所、大型企业的数据库服务器等
  • 迷你计算机：仍保有大型计算机同时支持多用于的特性，但主机可以放在一般作业场所，不必像前两个大型计算机需要特殊的空调场所。通常用来作为科学研究、工程分析与工厂的流程管理
  • 工作站：价格比迷你计算机便宜许多，是针对特殊用途而设计的计算机。主要用于学术研究与工程分析方面
  • 微电脑：也就是个人计算机，它的体积最小、价格最低、单功能还是五脏俱全，主要可分为桌上型、笔记型等

# 计算机常用计算单位

容量单位

计算机二进制（0 或 1）的单位称为 bit，8 个二进制位组成一个 byte字节：1Byte=8bits

1ZB = 1024 EB = 1024 PB = 1024 TB = 1024 GB = 1024 MB = 1024 KB = 1024 B = 8b

速度单位

CPU 的指令周期常使用 MHz 或者 GHz 之类的单位，这个 Hz 就是秒分之一。而在网络传输方面，由于网络使用的是 bit 为单位，因此网络常使用的单位是 Mbps每秒多少 Mbit

# 主板

一般消费者说的计算机通常指的是 x86 的个人计算机架构，而且 Linux 最早发展的时候，就是依据个人计算机的架构来发展的（以下以 Intel 主板架构来谈）

由于主板是链接各组件的一个重要项目，因此在主板上面沟通各部组件的的芯片组设计优劣，就会影响效能不少

早期的芯片组通常分为两个网桥来控制各组件的沟通，不过由于北桥最重要的就是 CPU 与主存之间的桥接，因此目前的主流架构中大多将北桥控制器整合到 CPU 封装当中了

  • 北桥：负责连接速度较快的 CPU、主存与显示适配器界面等组件
  • 南桥：负责连接速度较慢的装置接口，包括硬盘、USB、网络卡等

# CPU

CPU 内部含有 微指令集，不同的微指令集会导致 CPU 工作效率的优劣。除了这点之外，CPU 的效能比较的还有 CPU频率。CPU 频率就是 CPU 每秒钟可以进行的工作次数，频率越高表示这个 CPU 单位时间内可以做更多的事情

早期的 CPU 架构主要通过北桥来连接系统的 CPU、主存与显示适配器装置，这就导致所有的设备都得通过北桥来连接，这就要求每个设备的工作频率应该要相同，于是就有所谓的 前端总线（FSB）这个东西的产生。但因为 CPU 的指令周期比其他的设备都要来的快，又为了要满足 FSB 的频率，因此厂商就在 CPU 内部再进行加速，于是就有所谓的 外频与倍频 了，两者相乘就是 CPU 的频率速度

  • 外频: 在早期 CPU 设计中，外频指的是 CPU 与外部组件进行数据传输时的速度
  • 倍频: CPU 内部用来加速工作效能的一个倍数

但如此一来所有的数据都被北桥卡死了，北桥又不可能比 CPU 更快，因此它通常是系统效能的瓶颈。为了解决这个问题，新的 CPU 设计中，已经将内存控制器整合到 CPU 内部，因此 CPU 的频率设计就无须考虑得要同步的外频，只需考虑整体的频率即可

# 内存

只要计算机在运行中，CPU 就会把需要运算的数据调进内存中进行运算，当运算完成后 CPU 再将结果传送出来，因此内存的性能对计算机影响非常大（内存也被称为内存储器，其作用是用于暂时存放 CPU 中的运算数据，以及与硬盘等外部存储器交换的数据）

    主存储器（RAM:Main Memory），简称主存，其作用是存放指令和数据，并能由 CPU 直接随机存取

    • 个人计算机的主存储器主要组件为动态随机存取内存（DRAM:Dynamic Random Access Memory），随机存取内存只有在通电时才能记录与使用，断电后数据就消失了（通常所说的内存容量指的就是内存条 DRAM 的大小）
    • DRAM 根据技术的更新又分为好几代，而使用上较为广泛的有所谓的 SDRAM 和 DDR SDRAM（DDR:Double Data Rate）两种。这两种内存的差别除了在于脚位与工作电压的不同之外，DDR 是所谓的双倍数据传送速度，它可以在一次工作周期中进行两次数据的传送，所以传输速率方面比 SDRAM 还要好，新一代的 PC 大多使用 DDR 内存
    • DDR SDRAM 又依据技术的发展，有 DDR、DDR2、DDR3、DDR4 等

    在 80386 以上的微机中配置了高速缓冲存储器（Cache），这时内存包括主存和高速缓存

    • CPU 的数据都是由主存提供的，但 CPU 到主存之间还是得通过内存控制器。如果某些很常用的数据可以放置到 CPU 内部，纳秒 CPU 的数据读取就不需要跑到主存中重新读取了，这大大提升了效能，这就是缓存的设计概念（高速缓存 Cache 主要是为了解决 CPU 与主存速度不匹配而设计的）
    • 由于高速缓存整合到了 CPU 内部，因此它的速度必须要于 CPU 频率相同，使用 DRAM 是无法达到这个频率速度的，此时就需要静态随机存储内存（SRAM:Static Random Access Memory）：它的存取速度接近 CPU，快于 DRAM;存储容量小于 DRAM
    • 它比主存的优先级高，CPU 存取数据时优先访高速缓存，找不到的话再去主存 DRAM 中找，同时把数据从主存复制到 Cache 中

    只读存储器（ROM）：BIOS 程序是写死到主板上面的一个内存芯片中的，这个内存芯片在没有通电时也能够将数据记录下来，那就是只读存储器（ROM:Read Only Memory），它是一种非挥发性的内存

    # 硬盘与存储设备

    计算机总是需要记录和读取数据，而这些数据不可能每次都是由用户经过键盘来打字，所以就需要有储存设备。计算机系统上面的储存设备包括有:硬盘、软盘、CD、DVD、磁带机、闪存、蓝光光驱等，乃至于大型机器的局域网络存储设备（SAN/NAS）等，都是可以用来储存数据的，其中最常见的就是硬盘了

    # 计算机软件

    计算机只认识 0 和 1，计算机最重要的运算和逻辑判断在 CPU 内部，而 CPU 其实是具有微指令集的，因此当需要 CPU 帮忙工作时，就得要参考微指令集的内容，然后编写让 CPU 读得懂的脚本给 CPU 执行，这样就能让 CPU 工作了

    计算机语言

    • 机器语言: 就是 0 和 1
    • 汇编语言:
    • 高级语言: C、C++、Java、Python 等

    计算机的灵魂在于软件，目前计算机系统将软件分为两大类：一个是 系统软件，一个是 应用程序

    在早期想要让计算机执行程序就得要参考一堆硬件功能函数，并且学习及其语言才能够编写程序。同时每次写程序时都必须重新改写，因为硬件与软件功能不见得一致。那如果能够将所有硬件都驱动，并且提供一个发展软件的参考接口来给工程师开发软件，这就是操作系统

      操作系统也是一个软件，它在于管理计算机的所有活动以及驱动系统中的所有硬件。操作系统的功能就是让 CPU 可以开始判断逻辑与运算数值、让主存可以开始加载/读出数据与程序代码、让硬盘可以开始被存取、让网络卡可以开始传输数据、让所有周边可以开始运转等。总之，硬件的所有功能都必须要通过这个操作系统来达成。上述功能就是操作系统的核心。你的计算机能不能做某些事情，都与核心有关。核心程序非常重要，万一用户不小心将核心程序停止或破坏，将会导致整个系统的崩溃，因此核心程序所放置的内存区块是受保护的，并且开机后一直常驻内存当中

      为了更好的管理硬件，操作系统通常都会提供一整组的开发接口给工程师来开发软件，工程师只需要遵守该开发接口就能很容易开发软件了。举例来说，C 语言只需要参考 C 语言的函数就行，不需要再去考虑其他核心的相关功能，因为核心的系统调用会主动的将 C 语言的相关语法转换为核心可以了解的任务函数，那核心就能顺利运作该程序了
