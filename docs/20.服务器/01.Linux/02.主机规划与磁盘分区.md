> 磁盘的体系结构: `硬件` → `RAID` → `分区知识` → `格式化` → `挂载` → `日常维护`

## 机械硬盘外部结构

机械硬盘通过接口与计算机主板进行连接,硬盘的读取和写入速度与接口有很大关系.如果接口的性能很差,则会直接影响机械硬盘的性能

常见的硬盘的接口: `SATA`、`SAS`、`USB`、`SCSI`、`IDE`（SAS 取代 SCSI；SATA 取代 IDE，IDE 被时代抛弃）

- **IDE**：它是早期普通 PC 的标准接口
- **SATA**：使用 SATA 口的磁盘又叫串口磁盘，它所使用的的扁平电缆比较窄小，对于安装和机壳内的通风都比较好，因此原本的 IDE 粗扁平电缆界面就被 SATA 取代了。串口磁盘在很大程度上提高了数据传输的可靠性。串行接口还具有结构简单、可靠、性能高、支持热插拔的优点。目前已经是 SATA 三代,理论传输速度达到 600MB/s
- **SCSI**：SCSI 接口是一种广泛用于小型机上的高速数据传输技术，它具有引用范围广、多任务、带宽大、CPU 占用率低以及热插拔等优点，因较高的价格使得它很难如 IDE 磁盘般普及，因此 SCSI 磁盘主要应用于中高端服务器和高档工作站中
- **SAS**：它是新一代 SCSI 技术，和 SATA 磁盘相同，都是采用串行技术以获得更高的传输速度，并通过缩短连接线改善内部空间等。SAS 是并行 SCSI 接口之后开发出的全新接口，此接口的设计是为了改善存储系统的效能、可用性和扩充性，提供与串行 SATA 磁盘的兼容性。SAS 技术是结合了 SATA 与 SCSI 两者的优点而诞生的，同时串行 SCSI（SAS）是点到点的结构，因此除了提高性能之外，每个设备连接到指定的数据通路上提高了带宽，从而为数据传输与存取提供了必要保障
- **USB**：传统 USB2.0 仅有大约 60MB/s 的理论传输率;USB3.0 的速度为 500MB/s

## 机械硬盘内部结构

::: note 机械硬盘/传统硬盘（HDD:Hard Disk Drive）的物理组成
机械硬盘主要由 `磁盘盘片`、`磁头`、`磁臂` 和 `主轴马达` 组成的

- 数据就存放在磁盘盘片中：磁盘的表面是由一些磁性物质组成，可以用这些磁性物理记录二进制数据
- 磁盘是由多个盘片摞起来的：每个盘片有两个盘面，都能进行读写操作
- 每个盘面都对应一个磁头（head）：每个盘片是上下双磁头的，盘片在两个磁头中间高速旋转
- 所有的磁头都连在同一个磁臂上：磁臂可以沿着盘面作径向运动，从而带动磁头到达不同的磁道来对不同扇区进行读写操作

:::

## 磁盘内部相关名词介绍

一般来说，一块磁盘有 1 到数个盘片不等，其中每个盘片的有效盘面对应一个读写磁头，从上往下从 0 开始依次编号，不同的磁盘盘面在逻辑上被划分为磁道、扇区以及柱面，一般在出厂时就设定好了这些

![磁盘结构](https://cdn.jsdelivr.net/gh/sbgong/static/blog/server/linux/202205021427873.png)

### 磁头（head）

磁盘的每个盘片的每个有效盘面都会有一个读写磁头（磁头数=盘片个数\*2）。在磁盘不工作的时候，磁头停靠在靠近主轴接触盘面的表面，即线速度最小的地方，这里是一个不存放任何数据的特殊区域，称为启停区或着陆区，启停区以外就是数据区。在磁盘的最外圈，离主轴最远的磁道称为『0 磁道』，磁盘数据的存放就是从最外圈的 0 磁道开始的。0 磁道非常重要，系统的引导程序就在 0 柱面 0 磁道 1 扇区的前 446 字节

磁盘不工作时，磁头会自动停留在启停区，当磁盘需要读写数据时，磁盘主轴盘片开始旋转。旋转速度达到额定的高速时，磁头就会因盘片旋转产生的气流而抬起，这时磁头才向盘片存放数据的区域移动并开始读取数据。盘片旋转产生的气流很强，足够把磁头托起，并于盘面保持一个微小的距离。这个距离越小，磁头读写数据的灵敏度就越高，当然对磁盘各部件的要求也越高

气流既能使磁头脱离开盘面，又能使它保持在离盘面足够近的地方，非常紧密地跟随着磁盘表面呈起伏运动，使磁头飞行处于严格受控状态。磁头必须飞行在盘面上方，而不是接触盘面，这种位置可避免擦伤磁性涂层，而更重要的是不让磁性涂层损伤磁头。但是磁头也不能离盘面太远，否则就不能使盘面达到足够强的磁化，难以读出盘上的磁化翻转（磁极转换形式，是磁盘上实际记录数据的方式）

磁盘的磁头飞行悬浮高度低、速度快，一旦有小的尘埃进入磁盘密封腔内，或者一旦磁头与盘体发生碰撞，就有可能造成数据丢失，形成坏块，甚至造成磁头和盘体的损坏。所以，磁盘系统的蜜蜂一定要可靠，在非专业条件下绝对不能开启磁盘密封腔。

### 盘片

磁盘的每一个盘片都有两个盘面，即上、下盘面，一般来说，每个盘面都可以存储数据，成为有效盘面，也有极个别的磁盘盘面为单数。每一个这样的有效盘面都一个盘面号，按顺序从上至下从 0 开始依次编号。因为每一个有效盘面都有一个对应读写的磁头，盘面号又叫磁头号，

![磁盘盘面区域的划分](https://cdn.jsdelivr.net/gh/sbgong/static/blog/server/linux/202205021428108.png)

### 磁道（Track）

::: tip 磁道总结

- 每个盘片的盘面被划分成多个狭窄的同心圆环，数据就存储在这样的同心圆环上面，我们将这样的圆环称为磁道。每个盘面可以划分多个磁道，最外圈的磁道是 0 号磁道，向圆心增长依次为 1 磁道、2 磁道……磁盘的数据存放就是从最外圈开始的
- **注意**：磁道只是逻辑结构，在盘面上并没有真正的同心圆；硬盘的磁道密度非常高，通常一个盘面上就有上千个磁道，但是相邻的磁道之间并不是紧挨着，这是因为磁化单元相隔太近会相互产生影响

:::

磁盘在格式化时被划分成许多同心圆，这些同心圆的轨迹叫做磁道。磁道由盘面从外向内依次从 0 开始顺序编号。信息以脉冲串的形式记录在这些轨迹中，这些同心圆轨迹不是连续的记录数据，而是被划分成一段段的圆弧，这些圆弧的角速度一样。由于径向长度不一样，所以线速度也不一样，外圈的线速度比内圈的线速度大，即同样的转速下，外圈在同样的时间段里，划过的圆弧长度要比内圈划过的圆弧长度大（但读取到的数据是一样的）。这样的每段圆弧叫做一个扇区，扇区从 1 开始编号，每个扇区中的数据作为一个单元同时读出或写入。

**注意**：磁道是看不见的，只是在盘面上以特殊形式磁化了的一些磁化区

### 扇区（Sector）

每个磁道可以存储多达 KB 个数据，但计算机不必要每次读写这么多数据，因此再把磁道划分成为若干弧段，每个弧段就是一个**扇区(Sector)**。每个扇区就是一个"磁盘块"，它能存储的数据量相同。扇区也是硬盘上存储的物理单位，扇区的物理量设计有两种大小：512bytes 和 4Kbytes。因此即使计算机只需要某一个字节的数据，也得把这个扇区的数据全部读入内存，再选择所需要的那个字节

磁盘的第一个扇区很重要，因为它记录了整个磁盘的重要信息。这个扇区主要分为三部分：**主引导记录**、**分区表**和**结束标志**。因这一扇区中，引导程序代码占有巨大部分的空间，故而习惯将该扇区称为 **MBR 扇区**。该扇区作为管理整个磁盘空间的一个特殊空间，它不属于磁盘上的任何分区，因而分区空间内的格式化命令不能清除主引导记录的任何信息

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab 主引导记录
**主引导记录**（MBR: master boot record）：大小为 446 字节；主引导记录中存放最基本的开机管理程序（boot loader），这个开机管理程序的目的是在加载（load）核心文件/内核。由于每种操作系统的文件格式不同，每种操作系统的开机管理程序也不一样，不同的操作系统只有使用自己的开机管理程序才能加载自己的内核。开机管理程序不仅可以安装在 MBR 中，还能安装在每个文件系统的启动扇区（boot sector）中

- 如果服务器上只安装了一个操作系统，那么这个操作系统的启动引导程序就会安装在 MBR 中。BIOS 调用 MBR 时读取出启动引导程序就可以加载内核了
- 如果服务器上安装了多个操作系统，而每块硬盘只能有一个 MBR，系统只能在每个文件系统中单独划分出一个扇区，称作引导扇区（Boot Sector），即每个文件系统的引导扇区中也能安装启动引导程序。总结：在 MBR 和每个文件系统的引导扇区中都可以安装启动引导程序。这样多个操作系统才能安装在同一台服务器中，而且每个操作系统都是可以启动的
- 注意：在安装双系统的时候要先安装 windows 系统，然后再安装 Linux 系统。因为 Windows 在安装的时候会覆盖掉 MBR 以及自己所在分区的启动扇区，而 Linux 在安装的时候是可以选择将开机管理程序安装在 MBR 或其他文件系统的启动扇区中的

由于 MBR 仅有 446 字节，因此这个开机管理程序是非常小而精巧的，这个 boot loader 的主要任务有以下内容：

- 提供选单：用户可以选择不同的开机项目，这也是多重引导的重要功能
- 载入核心文件：直接指向可开机的程序区段来开始操作系统
- 转交给其他 loader：将开机管理功能转交给其他 loader 负责

由于具有控制权转交的功能，因此可以载入其他 boot sector 内的 loader。不过 Windows 的 loader 默认不具有控制权转交功能，因此不能使用 Windows 的 loader 来载入 Linux 的 loader，所以为什么需要先装 Windows 再装 Linux
:::
::: tab 主分区表
**主分区表**（partition table）记录整个硬盘分区的状态，仅有 64bytes。磁盘分区表需要保存每一个磁盘的开始和结束的柱面号码，而每一个位置需要 8 字节，故磁盘分区表最多可以有四组分区，每 16 个字节表示一个分区

| 分区表内容      | 长度 | 长度                                                                       |
| --------------- | ---- | -------------------------------------------------------------------------- |
| 第 1 个字节     | 1    | 引导标志：0x00 表示非活动分区，0x80 表示活动分区，四个分区只有一个是激活的 |
| 第 2-4 个字节   | 3    | 该分区第一个扇区的物理位置：本分区的起始磁头号、扇区号、柱面号             |
| 第 5 个字节     | 1    | 分区类型，0x83 就是 Linux 分区                                             |
| 第 6-8 个字节   | 3    | 该分区最后一个扇区的物理位置：本分区的结束磁头号、扇区号、柱面号           |
| 第 9-12 个字节  | 4    | 该分区第一个扇区的逻辑地址：本分区之前已用了的扇区数                       |
| 第 13-16 个字节 | 4    | 该分区的扇区总数                                                           |

因为硬盘最多只能有 4 个主分区，并不能满足实际需求，为此引入了扩展分区和逻辑分区的概念

- **扩展分区**是一个概念，它是一种特殊的主分区，可以用来保存逻辑分区信息，不能直接存储数据。扩展分区最多只能有一个，除了主分区外，剩余的磁盘空间就是扩展分区了
- 在扩展分区上面可以创建多个**逻辑分区**，它没有个数限制，用户可以在主分区和逻辑分区中存储数据
- 补充：**活动分区**就是当前活动的操作系统可以启动的分区

MBR 分区最多支持 2T 的磁盘,大于 2TB 的磁盘只能采用 GPT（GUID partition table），即全局唯一标识磁盘分区表，一种使用 UEFI 启动的磁盘组织方式。GTP 不再有分区的概念，所有 CDEF 盘都在一段信息中存储。因为兼容的问题，GPT 其实在引导的最开始部分也有一段 MBR 引导，也叫作保护引导，为了放置设备不支持 UEFI。GPT 在理论上是无限制的
:::
::: tab 结束标志
**硬盘有效标志**（BRID：boot record ID）：大小为 2 字节；BRID 的主要作用就是判断该设备是否可以用于启动：如果这 MBR 扇区的最后两个字节是 0x55 和 0xAA，表明这个设备可以用于启动；如果不是，表明设备不能用于启动，控制权于是被转交给"启动顺序"中的下一个设备
:::

### 柱面（Cylinder）

柱面是一个逻辑概念，也是分隔槽的最小单位。简单来说就是处于同一个垂直区域的磁道称为柱面，即各盘面上面相同位置磁道的集合。需要注意的是，磁盘读写数据是按柱面进行的，磁头读写数据时首先在同一柱面内从 0 磁头开始进行操作，依次向下在同一柱面的不同盘面（即磁头上）进行操作，只有在同一柱面所有的磁头全部读写完毕后磁头才转移到下一柱面。因为选取磁头只需通过电子切换即可，而选取柱面则必须通过机械切换。数据的读写是按柱面进行的，而不是按盘面进行，所以把数据存到同一个柱面是很有价值的

### 磁盘容量的计算方式

> 磁盘容量计算公式：硬盘容量 = 盘面数 × 柱面数 × 扇区数 × 每扇区字节数(512Bytes)

## 固态硬盘

传统硬盘有个很致命的问题，就是需要驱动马达区转动磁盘，这会造成很严重的磁盘读取延迟。因此有厂商拿闪存去制作高容量的设备，这些设备的连接界面也是通过 SATA 或 SAS，而且外形还做的和传统磁盘一样，这样的磁盘叫做**固态硬盘**（SSD:Solid State Disk）固态硬盘最大的好处是，它没有马达不需要转动，而是通过内存直接读写，因此除了没数据延迟且快速之外，还很省电

## 磁盘分区

磁盘在使用前一般需要进行分区，当然如果不分区直接格式化使用也是没问题的。磁盘分区的类型有 `主分区`、`扩展分区` 和 `逻辑分区` 之分。一块磁盘最多可以有四个分区表信息，其中一个主分区的位置可以用一个扩展分区替换，且一块磁盘最多只能有一个扩展分区。在这个扩展分区内可以划分多个逻辑分区（IDE 磁盘逻辑分区大概编号可以是 5-63，SATA 编号 5-15：分区数字编号 1-4 留给主分区或扩展分区使用，逻辑分区编号只能从 5 开始，即使 1-4 分区号还有剩余）

- 对磁盘分区，实际上就是修改磁盘的分区表，没有动磁盘数据

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab 主分区（Primary）
一般来说，主分区是磁盘上必须存在的分区，一般为磁盘的第一个分区，我们可以在这个主分区上面安装操作系统。在一个磁盘上最多只能有四个主分区。因为每个主分区都会在主引导扇区中生成自己的分区表（DPT），且每个分区表要占用 16bytes。当初的磁盘设计时，只给分区表留下来 65bytes 的存储空间，所以就只能最多有 4 个主分区

- 主分区最多 4 个，最少 1 个
- 分区数字编号只能是 1-4，例如 SCSI 分区名为：sda1,sda2,sda3,sda4
- 主分区是系统中必须要存在的分区，系统盘一般会选择主分区安装系统

:::
::: tab 扩展分区（Extend）
严格来说，扩展分区不能算一个正常的分区，而是一个链接，它起到一个指向的作用。我们可以在扩展分区内建立逻辑分区，与逻辑分区不同的是，扩展分区没有 MBR，而只有扩展分区表。这个扩展分区表没有 64bytes 的限制，所以可以在扩展分区划分多个逻辑分区

- 一个磁盘最多只能存在一个扩展分区，也可以没有扩展分区
- 扩展分区有独立的分区表，而且不能直接存放数据
- 必须在扩展分区上建立逻辑分区才能存放数据

:::
::: tab 逻辑分区（Logic）
不能在磁盘中单独直接划分逻辑分区，逻辑分区必须存在于扩展分区内。在扩展分区内可以划分多个逻辑分区，逻辑分区的编号从 5 开始

- 逻辑分区必须存在于扩展分区之上
- 数字编号只能从 5 开始
- 逻辑分区可以存放任意普通数据

:::
::::

## 磁盘设备命名

**在 Linux 系统中，每个装置都被当成一个文件来对待**，几乎所有的硬件装置都在 `/dev` 这个目录内

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab 设备名称的定义规则

- 系统的第一块 IDE 接口的磁盘：/dev/hda
- 系统的第二块 IDE 接口的磁盘：/dev/hdb
- 系统的第一块 SCSI/SATA/SAS 接口的磁盘：/dev/sda
- 系统的第二块 SCSI/SATA/SAS 接口的磁盘：/dev/sdb

每个分区则使用磁盘名称对应的数字编号表示：

- 系统的第一块 IDE 接口磁盘的第 1 个分区：/dev/hda1
- 系统的第一块 IDE 接口磁盘的第 5 个分区：/dev/hda5
- 系统的第二块 SCSI 接口磁盘的第 1 个分区：/dev/sdb1
- 系统的第二块 SCSI 接口磁盘的第 5 个分区：/dev/sdb5

:::

## CMOS、BIOS、UEFI

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab CMOS
> CMOS 是记录各项硬件参数且嵌入在主板上面的存储器

- 主板上的组件是非常多的，而每个组件的参数又具有可调整性。硬件的详细参数信息都被记录到了主板上的一个称为 CMOS 的芯片上，这个芯片需要借助额外的电源来发挥记录功能，这就是主板上为什么会有一颗电池的原因
- CMOS 主要用于记录主板上面的主要参数.包括系统时间、CPU 电压与频率、各项设备的 I/O 地址（可以理解为每个装备的门牌号码）与 IRQ（中断，可以理解为各个门牌号连接到 CPU 的专门路径）等

:::
::: tab BIOS
> BIOS 是一个写入主板上的软件程序

- BIOS 是一套程序，这套程序是写死到主板上面的一个内存芯片中，这个内存芯片在没有通电时也能将数据记录下来
- BIOS 对个人计算机来说非常重要，因为它掌握了系统硬件的详细信息与开机设备的选择等，并且是系统在开机时首先会去读取的一个程序
- BIOS 使用的是只读存储器 ROM，因此根本无法修正 BIOS 程序代码，为此现在的 BIOS 通常是写入类似闪存（flash）或 EEPROM 中

总结：它是写入到主板上某一块 flash 或 EEPROM 的程序，它可以在开机的时候执行，以加载 CMOS 当中的参数，并尝试调用存储设备中的开机程序，进一步进入操作系统中（BIOS 程序也可以修改 CMOS 中的数据）
:::
::: tab UEFI
GPT 可以提供到 64bt 的寻址，然后也能够使用较大的区块来处理开机管理程序。但是 BIOS 其实不懂 GPT，还得要透过 GPT 提供兼容模式才能够读写这个磁盘装置。而且 BIOS 仅为 16 位的程序，在与现阶段新的操作系统接轨方面有点弱掉了！为了解决这个问题，因此就有了 UEFI（Unified Extensible Firmware Interface）
:::
::::

::: note 整个开机流程到操作系统之前的动作

- BIOS 是计算机开机时会主动执行的第一个程序
- BIOS 会去分析计算机里面有哪些储存设备，以硬盘为例，BIOS 会依据使用者的设定去取得能够开机的硬盘，并且到该硬盘里面读取第一个扇区的 MBR 位置。MBR 这个仅有 446bytes 的硬盘容量里面会放置最基本的开机管理程序，此时 BIOS 就功成圆满，而接下来就是 MBR 内的开机管理程序的工作了
- 这个开机管理程序的目的是加载核心文件，由于开机管理程序是操作系统在安装的时候所提供的，所以它会认识硬盘内的文件系统格式，因此就能够读取核心文件。接下来就是核心文件的工作，也就是将之后的工作交给操作系统

:::

## 补充

现代硬盘寻道都是采用 CHS（Cylinder Head Sector）的方式

- 硬盘读取数据时，读写磁头沿径向移动，移到要读取的扇区所在磁道的上方，这段时间称为**寻道时间(seek time)**
- 因读写磁头的起始位置与目标位置之间的距离不同，寻道时间也不同
- 磁头到达指定磁道后，然后通过盘片的旋转，使得要读取的扇区转到读写磁头的下方，这段时间称为**旋转延迟时间**
- 然后再读写数据，读写数据也需要时间，这段时间称为**传输时间**
