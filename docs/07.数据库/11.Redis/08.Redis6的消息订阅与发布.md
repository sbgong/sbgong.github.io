## 订阅与发布

进程间的一种消息通信模式：发送者(pub)发送消息，订阅者(sub)接收消息。先订阅后发布后才能收到消息

![订阅/发布消息图](https://cdn.jsdelivr.net/gh/sbgong/static/blog/database/redis/202206070127324.bmp)

## 相关命令介绍

|                    命令                    | 说明                             |
| :----------------------------------------: | -------------------------------- |
| `pubsub subcommand [argument [argument…]]` | 查看订阅与发布系统状态           |
|         `publish channel message`          | 将信息发送到指定的频道           |
|      `psubscribe pattern [pattern…]`       | 订阅一个或多个符合给定模式的频道 |
|    `punsubscribe [pattern [pattern…]]`     | 退订所有给定模式的频道           |
|       `subscribe channel [channel…]`       | 订阅给定的一个或多个频道的信息   |
|     `unsubscribe [channel [channel…]]`     | 指退订给定的频道                 |

```shell
# 命令行一
SUBSCRIBE c1 c2 c3  # 可以一次性订阅多个频道
PSUBSCRIBE new*     # 订阅多个，通配符*

# 命令行二
PUBLISH c2 hello-redis # 消息发布
PUBLISH new1 redis2015 # 收取消息
```
