字体

CSS1 规范写于 1996 年，"Font Properties"（字体属性）一节的开头是这样说的，『设置字体属性是样式表最常见的用途之一』。这么多年过去了，这句话仍无法推翻。

CSS2 开始支持使用 @font-face 下载指定的自定义字体，不过直到 2009 年前后这个功能才以一致的方式被广泛支持。然而要注意，这并不表明你能完全控制字体。如果字体下载失败，或者字体的格式不被用户的浏览器支持，那么文字将使用后备字体显示

# 字体族

我们通常说的『字体』通常包含多个变体，分别针对粗体、斜体等。例如你可能熟悉的 Times 字体，其实它有多种变体，包括 TimesRegular、TimesBold、TimesItalic、TimesBoldItalic 等。Times 的这些变体各自都是一个字型（font face），我们通常说的 Times，其实是这些不同字型的统称。也就是说，Times 其实是一个字体族（font family），而不是一个字体

为了覆盖所有情况，CSS 定义了五种通用字体族：

字体族 中文 说明
serif 衬线字体 这种字体中的字形宽度各异，而且有衬线
（在每个字母的边缘都有一个小的笔触）
sans-serif 无衬线字体 这种字体中的字形宽度各异，而且无衬线
（字体线条简洁，没有小笔画）
monospace 等宽字体 等宽字体中的字形宽度一样，一般用于显示编程代码或表格数据
这种字体可以有衬线，也可以无衬线
cursive 草书字体 这种字体尝试模仿人类笔迹或手写体
通常这种字体在笔画末端有较大的花饰，而且比衬线字体华丽
fantasy 幻想字体 这种字体有时也叫装饰字体或展示字体，没有什么统一的特征
不过可以确定的是，无法将其划分到其他类别中的字体都是奇幻字体

    在 font-family 声明中，如果字体名称中有一个或多个空格（例如："New York"），或者字体名称中有符号（例如：# 或 $），建议使用引号（单引号或双引号都行）。如果没有引号，用户代理可能会忽略那个字体名称，然后接着处理规则后面的部分。注意：这只是推荐做法，不是强制要求

    # @font-face 使用自定义字体

    @font-face 是让你在设计中使用自定义的字体。假设你想使用的字体没有广泛安装，而是个十分特别的字体，借助 @font-face，你可以自定义一个专门的字体族名称，对应于服务器上的一个字体文件。用户代理将下载那个文件，使用它渲染页面中的文本，就好像用户的设备中安装了那个字体一样

    @font-face 是惰性加载字型的；这表明，仅当需要使用指定的字型渲染文本时，才会加载，否则不加载。其实，浏览器不管是否需要，都会现行下载声明的全部字型，这是浏览器的缺陷

    @font-face {
  font-family: "SwitzeraADF";
  src: url("SwitzeraADF-Regular.otf");
}

h1 {
  font-family: SwitzeraADF, sans-serif;
}

    定义字体的全部参数都在 @font-face {} 结构中编写，这些参数称为描述符，与属性十分相似，格式是 descriptor: value;

      描述符 说明
      font-family 定义字体族描述符，供 font-family 属性使用 必需
      src 为定义的字型提供一个或多个源。如果有多个源，之间以逗号分隔 必需
      字型的源可以指向任何 URI，不过有个限制：字型必需与样式表同源，因此不能把 src 指向别人的网站下载别人的字体。当然同源限制有个例外，就是使用 HTTP 首部 Access-Control-Allow-Origin 设定服务器，允许跨站加载
      （1）format()：告诉用户代理所用的字体是什么格式
      （2）local()：指定已经安装在用户设备中的字体族名称（可以是多个）
      font-style 区分常规、斜体和倾斜字型 默认值：normal
      font-weight 区分不同的字重（例如加粗） 默认值：normal
      font-stretch 区分不同的字符宽度（例如紧缩和加宽） 默认值：normal
      font-variant 区分众多字形变体（例如小号大写字母） 默认值：normal
      font-feature-settings 直接访问 OpenType 的低层特性（例如启用连字） 默认值：normal
      unicode-range 定义指定字体中可用的字符范围；默认覆盖全部 Unicode 字符 默认值：U+0-10FFFF
      值 说明
      embedded-opentype EOF 格式（Embedded OpenType）
      opentype OTF 格式（OpenType）
      svg SVG 格式（Scalable Vector Graphics）
      truetype TTF 格式（TrueType）
      woff WOFF 格式（Web Open Font Format）

      # 字重：font-weight

      font-weight 指代字体的粗细程度。自重越大，字体越黑越粗

      功能 说明
      取值 normal / bold / bolder / lighter / 100 / 200 / 300 / 400 / 500 / 600 / 700 / 800 / 900
      初始值 normal
      适用于 所有元素
      继承性 是
      动画性 否
      • 一般来说，字重越大，字体越黑、越粗
      • 在九级字重中，100 是最细的，900 是最粗的；400 对应于normal，700 对应于bold
      • 如果给定的字体族中字重的等级少于 9 个，用户代理必须填补既定方式的空缺
      • 在字体的变体中，以"Normal"、"Regular"等表示的可以分配给数字 400，以"Medium"表示的可以分配给数字 500。然而，只有一个变体，而且以"Medium"表示，那么应该分配给 400，而非 500
      字形 分配的关键字 分配的数字
      SwitzeraADF Light 100, 200, 300
      SwitzeraADF Regular normal 400
      SwitzeraADF Medium 500
      SwitzeraADF Blod blod 600, 700
      SwitzeraADF Extra Bold 800, 900

      # 字号：font-size

      font-size 取值：

      • 七个绝对大小值：xx-small、x-small、small、medium、large、x-large、xx-large
      • 两个相对大小值：smaller、larger
      • 长度和百分比值：<length>、<percentage>
      功能 说明
      初始值 medium
      适用于 所有元素
      百分数 根据父元素的字号计算
      计算值 一个绝对长度
      继承性 是
      动画性 是

      字体是如何确定的：font-size 属性与渲染结果之间的关系由字体设计者决定。这个关系在字体中通过em方框（或em 盒子）表示。em 方框与字体中字符的边界没有关系，其实它指的是在没有行距（CSS 中的 line-height）的情况下两条基线之间的距离。字体中完全有可能存在高度超过基线之间距离的字符，鉴于此，设计字体时要确保所有字符都比 em 方框小。因此，font-size 的作用是为字体的 em 方框提供一个尺寸，所以显示出来的字符不可能完全是指定的大小

      p.one { font-size: small; }
p.two { font-size: larger; }
p.three { font-size: 1.6em; }
p.four { font-size: 166%; }

      # 自动调整字号：font-size-adjust

      有两个因素影响字体是否清晰易辨：字号和 X 高度

      X 高度除以字号得到的结果称为高宽比值（aspect value）。随着字号的减小，高宽比值越高，字体越清晰易辨；相反，高宽比值较低的字体，很容易变得模糊难辨，CSS 提供的font-size-adjust属性用于改变字体族之间的高宽比值

      功能 说明
      取值 <number> / none / auto
      初始值 none
      适用于 所有元素
      继承性 是
      动画性 是

      注意：截至 2017 年年末，唯一支持 font-size-adjust 属性的用户代理是 Geeko(Firfox)系列

      # 字形：font-style

      font-style用于设置字形：normal（常规）、italic（斜体）、oblique（倾斜体）

      斜体和倾斜体的区别：

      • 斜体：是一种单独的字型，各字母的构造有些改动（除了字符有点斜之外，字符本身还可能会做调整）
      • 倾斜体：只是竖直体的倾斜版本
      功能 说明
      取值 normal / italic / oblique
      初始值 normal
      适用于 所有元素
      计算值 指定的值
      继承性 是
      动画性 否

      其实，不是每款字体都复杂到同时提供斜体和倾斜体，也很少有浏览器复杂到能区分二者之间的不同

      • 如果没有斜体，而有倾斜体，可以使用后者代替后者
      • 如果有斜体，而没有倾斜体，根据规范，用户代理可能不会使用前者代替后者。用户代理可以直接倾斜竖直体，得到倾斜体

      # 字体拉伸：font-stretch

      font-stretch 定义元素的文字是否横向拉伸变形（文字的拉伸是相对于浏览器显示的字体的正常宽度）

      注意：仅当使用的字体族中有宽体和窄体时，这个属性才起作用，而一般的字体族并没有这样的变体（如果有，字体族的价格往往不菲）

      属性值 说明
      normal 正常文字宽度
      ultra-condensed 比正常文字宽度窄 4 个基数
      extra-condensed 比正常文字宽度窄 3 个基数
      condensed 比正常文字宽度窄 2 个基数
      semi-condensed 比正常文字宽度窄 1 个基数
      semi-expanded 比正常文字宽度宽 1 个基数
      expanded 比正常文字宽度宽 2 个基数
      extra-expanded 比正常文字宽度宽 3 个基数
      ultra-expanded 比正常文字宽度宽 4 个基数

      # 字距调整：font-kerning

      有些字体定义了字符之间相对位置的数据，即字距。不同的字符组合，字据是不一样的。字距可以使用font-kerning 属性呼出或禁止

      功能 说明
      取值 auto / normal 或 none
      初始值 auto
      适用于 所有元素
      继承性 是
      动画性 否
      • auto：把决定权交给用户代理，让用户代理选择最合适的处理方式
      • normal：让用户代理正常处理字距，即使用字体中的字距数据
      • none：让用户代理忽略字体中的字距信息

      注意：如果既调整字距，又应用 letter-spacing 属性，正确的顺序是先调整字距，然后再根据 letter-spacing 属性的值调整字符间距，而不是反过来

      # 字体变形：font-variant

      除了字重、字形等之外，字体还有变形。变形信息内嵌在字型中，包括旧时的各种连写、小号大写字母、小数的表示形式、数字之间的间距、零有没有贯穿线等。如果字型中有这些变形信息，可以通过 CSS 的font-variant 属性调用

      功能 说明
      取值 normal / small-caps CSS2.1
      初始值 normal
      适用于 所有元素
      继承性 是
      动画性 否
      • normal：普通形式
      • small-caps：小号大写字母

      注意事项：

      • CSS3 新增的值取值繁杂，具体百度吧。截至 2017 年年末，字体变形还没有得到浏览器的广泛支持。当然，你可以使用 CSS2.1 定义的值。不过，Level3 新增的值中多数只被 Gecko 和 Webkit 支持
      • 即便字体的一个变形可在特定的浏览器中使用，但是这并不代表其他变形也能使用。而且，字体中有某个变形，也不表示在所有浏览器中都能调用。可以看出，情况很复杂，而且没有多少详细的指南可供参考

      # 字体特性：font-feature-settings

      font-feature-settings 属性从低层控制 OpenType 字体的哪些特性可以使用

      功能 说明
      取值 normal / <feature-tag-value>#
      初始值 normal

      <feature-tag-value> 值的具体格式为：<string> [ <integer> | on | off ]?

      多数特性的值是整数 0 和 1，相当于 off 和 on。不过有些特性的值可以是其他数，大于 1 时既表示启用特性，也用于定义选择的索引。如果列出了某个特性，但是没有提供数字，假定为 1（启用），因此以下规则都是等效的。还有请注意所有 <string> 值都必须放在引号里，没有放在引号里面的将会被忽略

      font-feature-settings: "liga";
font-feature-settings: "liga" 1;
font-feature-settings: "liga" on;

# 忽略 dlig
font-feature-settings: "liga", dlig;

      # 字体合成：font-synthesis

      • 背景：有时，指定的字体族中可能缺少某种字型，例如粗体或斜体。此时，用户代理可能会尝试使用可用的字型合成所需的字型，不过这样得到的效果不是特别好。为了解决这个问题，CSS 提供了font-synthesis 属性，让创作人员控制合成哪种字型，或者在渲染页面时禁止合成
      • 注意：截至 2017 年年末，只有 Firfox 支持 font-synthesis 属性
      功能 说明
      取值 none / weight、style
      初始值 weight style
      适用于 所有元素
      继承性 是
      动画性 否

      在支持的用户代理中，font-synthesis: none 声明将阻止用户代理在对应的元素上做这种合成

      # font 属性

      font 简写属性在一个声明中设置所有字体属性

      功能 说明
      取值 <font-style>,<font-variant>,<font-weight>,<font-size>/<line-height>,<font-family>
      百分数 <font-size>基于父元素计算；<line-height>基于当前元素的<font-size> 计算
      继承性 是
      • <font-style>、<font-variant>、<font-weight> 这前三个属性的出现顺序可以任意排列；此外，如果其中某个属性的值是 normal，甚至不用写出
      • <font-size>/<line-height>、<font-family> 这后两个属性必须出现在 font 声明中，且其顺序必须不能变动，否则整个规则无效，很可能被用户代理彻底忽略
      • line-height 是文本属性，增不增加 line-height 值完全是可选的。如果加入 line-height，它始终在 font-size 前面，不能反过来，而且两个值之间要以斜线分隔

      # 字体匹配机制

        1. 用户代理创建或访问字体属性数据库。这个数据库中有用户代理能访问的全部字体的各个 CSS 属性。通常，设备中的所有字体都在这里，不过可能还有其他字体（例如：用户代理可能内置了字体）。如果用户代理遇到两个一样的字体，将忽略其中一个
        2. 用户代理把应用了字体属性的元素摘出来，构建显示元素所需的字体属性列表。首先，用户代理根据这个列表选择使用哪个字体族显示元素。如果能找到完全匹配的字体，用户代理就使用那个字体；否则，还要做些额外工作
        3. 匹配字体时先看font-stretch 属性
        4. 然后再看 font-style 属性。任何以“italic” 或 “oblique” 表示的字体都能匹配 italic 关键字。如果没有这样的字体，匹配失败
        5. 接下来匹配 font-weight ，鉴于 CSS 对 font-weight 的处理方式，这一匹配绝不会失败
        6. 然后处理 font-size。匹配字号时要有一定容差，不过这个容差由用户代理定义。因此，指定的字号和实际使用的字号在一个用户代理中可能允许存在 20%的容差，而在另一个用户代理中可能只允许 10%的容差
        7. 如果第二步没有找到匹配的字体，用户代理在同一个字体族中选择替代字体。找到后，回到第二步
        8. 假设找到一个基本匹配的字体，但是字体没有显示元素所需的全部信息，比如字体缺少版权符号，那么用户代理将回到第三步，搜索替代字体，然后再执行第二步
        9. 最后，如果找不到匹配的字体，而且所有替代字体都试过了，用户代理将选择指定字体族中的默认字体，力争正确显示元素
        1. 查看默认启用的字体特性，包括指定文本所需的特性。默认启用的特性有“calt” “ccmp” “clig” “liga” “locl” “mark” “mkmk” 和 “rlig”
        2. 如果是 @font-face 规则定义的字体，检查 @font-face 规则中 font-variant 描述符对应的特性。然后检查 @font-face 规则中 font-feature-settings 描述符对应的特性
        3. 检查由 font-variant 或 font-feature-settings 之外的属性确定的特性设置（例如，把 letter-spacing 属性设为默认值之外的值时将禁用连字）
        4. 检查 font-variant 属性及其子属性（例如 font-variant-ligatures），以及其他可能会调用 OpenType 特性的属性（例如 font-kerning）的值对应的特性
        5. 检查 font-feature-settings 属性的值对应的特性
