::: tab "vi 编辑器"

- 所有的 UNIX-like 系统都会内置 vi 文本编辑器，其他的文本编辑器则不一定会存在
- 许多软件的编辑接口都会主动调用 vi（例如：crontab、visudo、edauota 等）

:::
::: tab "vim 编辑器"

- vim 视为 vi 的高级版本，可以使用颜色或下划线的方式来显示一些特殊信息
- vim 具有程序编辑能力，所以 vim 也主要作为程序开发的工具
- vim 具有程序编辑的能力，并且可以主动地识别字体颜色判别语法的正确性、方便程序设计
  - 例如：使用 vim 打开 C 语言文件与 HTML 文件，由于语法不同，字体的颜色也会不同
- vim 还支持其他很多功能，例如：正则表达式的查找方式、多文件编辑、区块复制等

:::
::::

## vi 的使用

- Ubuntu 中默认为 vi 编译器，没有 vim 编译器，需要自行安装
- centos 中默认为 vim 编译器，有 vi 编译器，但是 vi 编译器使用需要加上绝对路径（/bin/vi）

### 三种模式

基本上 vi 共有三种模式：**一般命令模式**（command mode）、**编辑模式**（insert mode）与**命令行模式**（command-line mode）

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab 一般命令模式
以 vi 打开一个文件就直接进入一般命令模式（这也是默认模式）。在这个模式中，可以使用『上下左右』按键来移动光标，也可以使用『删除字符』『删除整行』来处理文件内容，也可以使用『复制』『粘贴』来处理你的内容。但是无法编辑文件的内容
:::
::: tab 编辑模式
一般命令模式无法编辑文件的内容，当你按下 `i`、`a`、`o` 等相应字母时，就会进入编辑模式。如果按下『Esc』就会退出编辑模式，进入一般命令模式
:::
::: tab 命令行模式
在一般模式当中，输入 `:`、`/`、`?` 三个中任意一个按钮，就可以将光标移动到最下面那一行，此时为命令行模式。在这个模式中，可以提供你『查找数据』的操作，读取、保存、批量替换字符、退出 vi、显示行号等操作也是在此模式下完成的
:::
::::

**指令语法**：`/bin/vi 文件名` 注意：vi 后面一定要加文件名，不管该文件名存在与否

如果你开启的文件已经存在，则在文件状态栏显示的信息如『"/etc/man_db.conf" [readonly] 131L,5171C』代表的意思就是：当前打开的文件名是/etc/man_db.conf；由于文件启动者的身份缘故，目前文件为只读状态；文件内有 131 行，5171 个字符

### vim 缓存文件

如果你使用 vim 在编辑一个文件时，突然电脑死机或者种种原因导致文件未保存或处理就关闭了 vim 编辑器，那么系统此时就会将文件这些数据保存为一个缓存文件

- 缓存文件名为：`.原文件名.swp`，并且它是一个隐藏文件
- 缓存文件只有当编辑器不正常退出时才会建立
- 因为 vim 非正常退出，所以文件数据尚未保存。但是数据保存在缓存文件中，因此可以通过缓存文件来恢复数据

```shell
# 模拟系统，让vim编辑器非正常退出：在vim的一般指令模式下，按下ctrl+z的组合按键即可
ls -al # 可以看到有个缓存文件产生：.man_db.conf.swp
# 再使用vim编辑器打开原文件，系统会提示你一些错误提示信息，并且在最底下可以让你输入一些命令来进行下一步操作
vim man_db.conf
```

系统提示你两个错误提示信息：

- (1)的意思是：可能有其他人或程序同时在编辑你这个文件，有以下解决方法
  - 找到另外一方，让其停止编辑操作
  - 在最底下输入 O，进入只读状态来读取查看文件
- (2)的意思是：在前一个 vim 环境中，因为某些不知名的原因导致 vim 中断，有以下解决方法
  - 输入 R 来进行缓存文件恢复数据
  - 如果缓存文件无用，输入 D 命令删除缓存文件并打开这个文件

|  最下方的操作选项   | 说明                                                                                                                             |
| :-----------------: | -------------------------------------------------------------------------------------------------------------------------------- |
| O（open read-only） | 打开此文件进入只读状态，不能够进行编辑。可用于一个文件正在被他人编辑时来进行读取                                                 |
|  E（edit anyway）   | 还是使用正常的方式打开你这个要编辑的文件，并不会加载缓存文件的内容。不过可能会出现多个用户同时更改文件数据的问题，缓存文件还存在 |
|    R（recover）     | 以缓存文件的内容打开这个文件，此时就可以将以前那些未保存的数据再读取进来。不过当你退出 vim 后，缓存文件还存在，需要手动删除      |
|   D（delete it）    | 确定这个缓存文件是无用的，然后打开原文件并且删除这个缓存文件                                                                     |
|      Q（quit）      | 直接退出 vim，不存在任何操作                                                                                                     |
|     A（abort）      | 忽略这个编辑操作，与 Q 的功能类似                                                                                                |

## 注意事项

### 中文编码问题

::: warning 编码问题请注意

- 你的 Linux 系统默认支持的语系数据，这与 `/etc/locale.conf` 文件有关
- 你的终端（bash）的语系：这与 LANG,LC_ALL 这几个变数有关
- 文件原本的编码
- 开启终端机的软件，例如 GNOME 底下的窗口接口

:::

### 文件语系编码转换：iconv

> 通过 `iconv` 这个命令可以将一个文件的语系编码进行转换，这个命令是针对于文件而言的

- `iconv --list`
- `iconv -f 原本编码 -t 新编码 原文件名 [-o 新文件名]`

|     选项参数     | 说明                                                                   |
| :--------------: | ---------------------------------------------------------------------- |
| `--list` 或 `-l` | 列出 iconv 支持的语系编码数据                                          |
|  `-f encoding`   | 后面跟这个文件的原本编码                                               |
|  `-t encoding`   | 后面跟你要转换的编码                                                   |
|    `-o file`     | 指定输出文件。如果你想将转换后的内容保存到一个文件中，可以使用这个选项 |
|       `-c`       | 忽略输出的非法字符                                                     |
|       `-s`       | 禁止警告信息，但不是错误信息                                           |
|   `--verbose`    | 显示进度信息                                                           |

```shell
iconv --list # 列出iconv支持的语系编码
# 将UTF-16编码的文件转换为UTF-8格式的文件并且保存到UTF-8.txt文件中
iconv -f UTF-16 -t UTF-8 UTF-16.txt -o UTF-8.txt
```

### DOS 与 Linux 的断行文字

::: tip 换行符种类

- DOS/Windows 系统使用：CRLF（\r\n）
- Unix 系统使用：LF（\n）

:::

我们知道 Linux 中输入命令开始执行的依据是『回车』，而 Linux 中回车为 LF。而 DOS 与 Windows 换行符为 CRLF，多了一个 CR 出来，如果是一个 shell 脚本的程序文件，将可能造成程序无法执行的状态

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab "dos2unix 和 unix2dos"
**指令语法**

- dos 转 Linux：`dos2unix [-kn] 文件 [新文件]`
- Linux 转 dos：`unix2dos [-kn] 文件 [新文件]`

| 选项参数 | 说明                                                                  |
| :------: | --------------------------------------------------------------------- |
|   `-k`   | 保留该文件原本的 mtime 时间格式（不更新文件上次内容经过自定义的时间） |
|   `-n`   | 保留原本的旧文件，将转换后的内容输出到新文件                          |

:::
::: tab 代码案例

```shell
unix2dos -k man_db.conf     # 将LF格式的换行符转换为CRLF格式（不生成新文件）
dos2unix -k -n man_db.conf man_db.conf.linux  # 将CRLF格式的换行符转换为LF格式（生成新文件）
```
