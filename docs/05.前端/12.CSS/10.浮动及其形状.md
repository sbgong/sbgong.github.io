在很长一段时间内，浮动元素是所有 Web 布局方案的基础（很大程度上依赖 clear 属性）。但是浮动并不是为布局而生的，与使用表格布局基本一样，把浮动作为布局工具算是无奈之举，那时别无他选。然而，浮动自身确实相当有趣和有用的，尤其是最近新增浮动形状之后，内容可以沿非矩形边框浮动了

## 浮动：float

| 功能   | 说明                  |
| ------ | --------------------- |
| 取值   | `left / right / none` |
| 初始值 | none                  |
| 适用于 | 所有元素              |
| 继承性 | 否                    |
| 动画性 | 否                    |

CSS 可以浮动任何元素，通过 `float` 属性实现。例如：图像浮动到右侧，其他内容（例如文本）将围绕图像流动。浮动主要用于盒子模型，可以让多个盒子一行展示，注意浮动元素不会压住标准流的文字

在了解浮动之前，一定要建立容纳块的概念。浮动元素的容纳块是最近的块级祖辈元素。在此，不管元素是什么类型，浮动后得到的都是块级框

浮动元素的位置由一系列规则约束，在深入探讨具体行为之前，先来了解一下，这些规则与计算外边距和宽度的方式略有相似之处，而且都有些符合常识的地方，这些规则如下：

- 规则一：浮动元素的左（或右）外边界不能超出容纳块的左（或右）内边界
- 规则二：如果在文档源码中处于前面的元素向左浮动，那后面的浮动元素的左外边界必定在前一个元素右外边界的右侧，除非后一个元素的顶边在前一个元素的底边以下。这条规则能确保所有浮动的内容都可见，不会出现一个浮动元素遮盖另一个浮动元素的情况，让我们可以放心使用浮动
- 规则三：左浮动元素的右外边界不能在右浮动元素的外边界的右侧；右浮动元素的左外边界不能在左浮动元素的右外边界的左侧。这条规则避免浮动的元素重叠，假设网页正文的宽度是 500 像素，内容里有两个图像，都是 300 像素宽。第一个图像向左浮动，第二个图像向右浮动。根据这条规则，第二个图像不会与第一个图像出现 100 像素宽的重叠区域。事实上，第二个图像的顶边将正好在左浮动图像的底边以下
- 规则四：浮动元素的顶边不能比父元素的内顶边高。如果浮动元素位于两个折叠的外边距之间，在两个元素之间放置它的位置时，将视其有个块级父元素。这条规则的前半部分避免浮动元素一直向文档的顶端浮动。后半部分进一步规定某些情况下的对齐方式。例如：有三个段落，中间那个段落是浮动的，此时中间那个段落在浮动时好似有一个块级父元素，这样能避免浮动的段落一直向上移动，超出三个段落共有的父元素
- 规则五：浮动元素的顶边不能比前方任何一个浮动元素或块级元素的顶边高。与规则四一样，这条规则避免浮动的元素一直向上移动，超出父元素的顶边。而且，浮动元素的顶边不能比前方任何一个浮动元素的顶边高
- 规则六：浮动元素的顶边不能高于文档源码中出现在浮动元素之前的元素生成的框体所在的行框的顶边。这条规则和规则 4,5 类似，这条规则进一步限制浮动元素上移的幅度，以免浮动元素超出前方内容所在的行框顶边。假如一个段落中有个浮动的图像，那么图像顶边能到达的最高位置是图像原来所在的那一行行框的顶边，这样能避免图像向上浮动太远
- 规则七：左浮动元素的左边如果还有一个向左浮动的元素，那么它的右外边界不能在容纳块右边界的右侧。类似，右浮动元素的右边如果还有一个向右浮动的元素，那么它的右外边界不能在容纳块左边界的左侧
- 规则八：浮动元素必须放在尽可能高的位置上
- 规则九：左浮动元素必须尽量向左移动，右浮动元素必须尽量向右移动。位置越高，向左或向右移动的距离越远

注意：

- 浮动的盒子不再保留原先的位置
- 浮动元素四周的外边距不折叠
- 如果浮动的是非置换元素，要为元素设定宽度。否则，元素的宽度趋近于零，因此浮动元素的宽可能只有一个字符宽
- 不管元素是 什么类型，浮动后得到的都是块级框
- 浮动元素的后代也浮动，浮动的范围扩大，涵盖浮动的后代元素。因此若想让元素随父元素一起浮动，可以浮动父元素
- 负外边距对浮动的影响：负外边距能导致元素比父元素宽，父元素也能让浮动的元素从父元素中突出来。还有一种情况能导致浮动的元素超出父元素的左右外边界，即浮动元素比父元素宽时

当浮动元素与内容重叠时：

- 行内框与浮动元素重叠时，其边框、背景和内容都在浮动元素"之上"渲染
- 块级框与浮动元素重叠时，其边框和背景在浮动元素"背后"渲染，而内容在浮动元素"之上"渲染

## 清除浮动：clear

由于父级盒子很多情况下，不方便给高度（因为不清楚高度到底会是多少），但是子盒子浮动又不占有位置，最后父级盒子高度为 0 时，就会影响下面的标准流盒子（由于浮动元素不再占用原文档流的位置，所以它会对后面的元素排版产生影响），所以就需要清除浮动。清除浮动的本质是清除浮动元素造成的影响；如果父盒子本身有高度，则不需要清除浮动。清除浮动之后，父级就会根据浮动的子盒子自动检测高度。父级有了高度，就不会影响下面的标准流了

| 功能   | 说明                         |
| ------ | ---------------------------- |
| 取值   | `left / right / both / none` |
| 初始值 | none                         |
| 适用于 | 块级元素                     |
| 继承性 | 否                           |
| 动画性 | 否                           |

- left：清除左侧浮动的影响
- right：清除右侧浮动的影响
- both：清除左右两侧浮动的影响
- none：允许元素向另一个元素的任何一边浮动，即浮动元素可以在元素的两边都可以出现

在 CSS1 和 CSS2 中，clear 其作用的方式是增加元素的上外边距，把元素移到浮动的元素下方。因此，为元素声明的上外边距其实会被忽略。然而，CSS2.1 引入了间距（clearance）这个概念，间距是为了把元素向下移动，确保显示在浮动元素的下方而在元素上外边距上方增加的额外空白。这意味着，清除了浮动的元素，其上外边距不再受到影响。现在，元素下移是因为有这一块间距

## 浮动形状：shape-outside

| 功能   | 说明                                              |
| ------ | ------------------------------------------------- |
| 取值   | `none / [<basic-shape> // <shape-box>] / <image>` |
| 初始值 | none                                              |
| 适用于 | 浮动元素                                          |
| 继承性 | 否                                                |
| 动画性 | `<basic-shape>`                                   |

- none：除了浮动元素自身的外边距框，没有形状，与常规情况下一样
- image：内容将流入与之"直接接触"那一侧的透明部分。内容不会同时流入图像的左右两边。注意：图像必须有透明区域才能定义浮动形状。如果图像格式为 JPEG，或者是没有 alpha 通道的 GIF 或 PNG，得到的形状将是矩形，这与 `shape-outside: none` 的效果没有区别
- basic-shape：注意，无论如何，形状不会超出浮动框
  - `inset()`：内凹形状，在计算向内偏移时可以为形状定义圆角
  - `circle()`：圆形；注意要定义半径和中心点的位置
  - `ellipse()`：椭圆；注意要定义两个半径和中心点的位置
  - `polygon()`：多边形；多边形形状由一系列逗号分隔的 x-y 坐标定义，值可以使用长度或百分数，相对形状框的左上角计算。一对 x-y 坐标指定多边形的一个顶点。如果第一个顶点和最后一个顶点的坐标不同，浏览器会把两点连接起来，封闭多边形。多边形还可以调整填充规则，默认的填充规则是 `nonzero`，得到一个完全填充的多边形；此外还可以使用 `evenodd`，多边形的某些部分有填充，而其他部分则没有。注意：截止 2017 年年末，唯一支持 CSS shapes 的浏览器 Chrome 尚不支持填充方式，所有多边形都以 nonzero 方式填充
- shape-box：指明形状最外边的界限。有四个值：`margin-box`、`border-box`、`padding-box`、`content-box`

## 使用透明图像定义形状：shape-image-threshold

可以使用有透明区域的图像定义浮动形状，之前图像中任何不完全透明的部分都是形状的一部分，这是默认行为。不过可以使用 `shape-image-threshold` 属性修改。这个属性用于指定透明度为多少时允许内容流入。反过来说就是，不透明度为多少时算在浮动形状内

| 功能   | 说明       |
| ------ | ---------- |
| 取值   | `<number>` |
| 初始值 | 0.0        |
| 适用于 | 浮动元素   |
| 继承性 | 否         |
| 动画性 | 是         |

- 对 `shape-image-threshold: 0.5` 来说，图像中透明度超过 50%的部分将允许内容流入，而透明度小于 50%的部分则构成浮动形状
- 对 `shape-image-threshold: 1` 来说，整个图像都不在浮动形状中，也就是没有浮动形状，内容将沿浮动元素流动
- 对 `shape-image-threshold: 0` 来说，图像中所有不透明的部分都在浮动形状中，即只有完全透明的区域才允许内容流入

此外，小于零的值将重置为 0.0，大于 1 的值将被重置为 1.0

## 为形状添加外边距：shape-margin

定义好浮动形状之后，可以使用 `shape-margin` 属性为形状添加"外边距"。与普通元素的外边距类似，形状的外边距把内容推开一定长度或百分数距离；百分数相对元素容纳块的宽度计算，这一点也与常规的外边距一样

| 功能   | 说明                      |
| ------ | ------------------------- |
| 取值   | `<length> / <percentage>` |
| 初始值 | 0                         |
| 适用于 | 浮动元素                  |
| 继承性 | 否                        |
| 动画性 | 是                        |

不过要记住，形状绝不会超出形状框。因此默认情况下，形状不会比没有形状的浮动元素的外边距框大。因为 shape-margin 会增加形状的尺寸，所以放大后的形状如果超出形状框，将被裁剪。若想确保间隔距离始终被保留下来，要设定常规的外边距，而且要大于或等于 shape-margin

## 清除浮动的策略：闭合浮动

闭合浮动. 只让浮动在父盒子内部影响，不影响父盒子外面的其他盒子

- 额外标签法，也称为隔墙法，是 W3C 推荐的做法，但自己不推荐，因为它会增加一个结构
- 父级添加 overflow 属性
- 父级添加 after 伪元素；代表网站：百度、淘宝网、网易等
- 父级添加双伪元素；代表网站：小米、腾讯等

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab "额外标签法"
额外标签法就是在最后一个浮动的子元素后面添加一个额外标签。注意：这个额外新增的标签，必须是块级元素

例如 `<div style="clear:both"></div>`，或者其他标签（如`<br />`等）

- 优点：通俗易懂，书写方便
- 缺点：添加许多无意义的标签，结构化较差

:::
::: tab "父级添加 overflow 属性"
可以给**父级**添加 overflow 属性，将其属性值设置为 hidden、 auto 或 scroll

- 优点：代码简洁
- 缺点：无法显示溢出的部分

:::
::: tab "父级添加 after 伪元素"
:after 方式是额外标签法的升级版。也是给**父元素**添加

- 优点：没有增加标签，结构更简单
- 缺点：照顾低版本浏览器

```css
.clearfix:after {
  content: "";
  display: block;
  height: 0;
  clear: both;
  visibility: hidden;
}
.clearfix {
  /* IE6、7 专有 */
  *zoom: 1;
}
```

:::
::: tab "父级添加双伪元素"
也是给给**父元素**添加

- 优点：代码更简洁
- 缺点：照顾低版本浏览器

```css
.clearfix:before,
.clearfix:after {
  content: "";
  display: table;
}
.clearfix:after {
  clear: both;
}
.clearfix {
  *zoom: 1;
}
```
