# X window 与文本模式的切换

  • 通常我们也称文本模式为终端机接口（Terminal 或 Console），Linux 预设的情况下会提供 6 个 Terminal 来让使用者登入，切换的方式为使用：Ctrl + Alt + [F1]~[F6] 的组合按键
  • 注销 Linux 直接输入 exit 即可离开

系统会将[F1]~[F6]命名为 tty1 ~ tty6的操作接口环境。在 CentOS7 环境下，当开机完成之后，默认系统只会提供给你一个 tty 而已，因此无论是文字界面还是图形界面，都是会出现在 tty1 中。tty2 ~ tty6 其实一开始是不存在的，但是当你要切换时（按下切换的组合键），系统才产生出额外的 tty2,tty3……

若你在纯文本环境中启动 X window，那么图形界面就会出现在当时的那个 tty 上面。举例来说，当你在 tty3 登入系统，然后输入 startx 启动个人的图形界面，那么这个图形界面就会产生在 tty3 上面

# 在终端界面登入 Linux

按下Ctrl + Alt + [F2]进入 tty2 登录界面（如果你没有启动图形窗口界面的话默认会来到 tty1 这个环境中）

# 文本模式下的指令格式

指令详细说明

  • 一行指令中第一个输入的部分就是『指令 command』或『可执行文件，例如批次脚本 script』
  • command 为指令的名称，例如切换工作目录的指令为 cd 等
  • 中括号并不存在于实际的指令中
    • 加入选项设定时，通常选项前会带 -，例如：-h
    • 如果要带选项的完整全名，则选项前会带 --，例如：--help
  • parameter1 parameter2 为依附在选项后面的参数，或者是 command 的参数
  • 指令、选项、参数这几个内容中间以空格分隔，不论空几格 shell 都视为一格
  • 按下『Enter』键后，该指令就立即执行
  • 指令太长的时候，可以使用反斜杠『\』来跳脱『Enter』符号，使指令连续到下一行
  • 在 Linux 系统中，英文大小写字母敏感，cd 和 CD 不同
  • 指令后面的选项除了前面带有减号『-』之外，某些特殊情况下，选项或参数前面也会带有正号『+』的情况

# 重要的几个热键

    『Tab』键具有命令补全、文件补齐 和 参数补齐的功能

    • 命令补全：如果按『Tab』键的地方在指令后面，就代表着命令补全。例如 cal 这个指令，如果在指令处输入 ca，然后再按两次『Tab』，就会列出所有以 ca 开头的指令
    • 文件补齐：如果按『Tab』键的地方在指令的第二个位置以后，就会代表文件补齐了。例如输入『ls -al ~/.bash』然后再加上两个『Tab』，就会列出用户家目录下所有以.bash 开头的文件名
    • 参数补齐：例如输入『date --』然后再加上两个『Tab』，系统就会列出 date 这个指令可以使用的选项有哪些

    『Ctrl+C』可以将正在运行中的指令中断。如果你在 Linux 中输入了错误的指令或者参数，可以使用这个组合键让当前跑着的程序停掉

    注意：如果你正在运行比较重要的指令，可别急着使用这个组合键

    『Ctrl+D』通常代表着键盘输入结束的意思。另外它也可以用来取代 exit 的输入，例如你想要直接离开文字接口，可以直接按下 Ctrl+D 就能直接离开了

    如果你在纯文本的画面中执行某些命令，这个指令的输出信息相当长，所以导致前面的部分已经不在目前的屏幕画面中了。如果你想要回头看一下输出的信息，就可以使用『Shift+PageUp/PageDown』的翻页功能

    # --help

    --help 用在协助你查询你曾经用过的指令所具备的选项和参数

    事实上，几乎 Linux 上面的指令，在开发的时候，开发者就将可以使用的指令语法与参数写入指令操作过程中了。你只需要使用『--help』这个选项，就能够将该指令的用法做一个大致的了解

    # man page

    man 就是 manual（操作说明）的简称，命令使用，例如：man date

      在输入『man date』后，数据第一行显示的是『DATE(1)』，它表示一般用户可使用的指令

      第一行数字 代表内容
      1 用户在 shell 环境中可以操作的指令或可执行文件
      2 系统调用的函数与工具等
      3 一些常用的函数与函式库，大部分为 C 的函式库
      4 装置文件的说明，通常在/dev 下的文件
      5 配置文件或者是某些文件的格式
      6 游戏（games）
      7 管理与协议等，例如 Linux 文件系统、网络协议、ASCII code 等的说明
      8 系统管理员可用的管理指令
      9 跟 Kernel 相关的文件
      man page 子部分 内容说明
      name 简短的指令、数据名称说明（列出这个指令的完整全名，并且简单说明用途）
      synopsis 简短的指令下达语法简介（列出这个指令的基本语法）
      description 较为完整的说明，这部分最好仔细看看（详细说明这个指令的选项和参数用法）
      options 针对 synopsis 部分中，有列举的所有可用的选项说明
      commands 当这个程序（软件）在执行的时候，可以在此程序（软件）中下达的指令
      files 这个程序或数据所使用或参考或连结到的某些文件
      see also 可以参考的，跟这个指令或数据有相关的其他说明
      example 一些可以参考的范例
      environment 与这个指令相关的环境参数说明
      author 这个指令的作者
      copyright 版权声明
      快捷键 说明
      空格键 向下翻一页
      PageUp 向上翻一页
      PageDown 向下翻一页
      Home 去到第一页
      End 去到最后一页
      /string 向下搜索 string 这个字符串
      ?string 向上搜索 string 这个字符串
      n,N 利用/或？来搜寻字符串时，可以用 n 来继续下一个搜寻，可以用 N 来进行反向搜寻
      q 结束这次的 man page

      既然有 man page，自然就是因为有一些文件数据，所以才能够以 man page 读出来。那么这些 man page 的数据根据不同的 distribution，存放的地址会有些许差异，不过通常放在 /usr/share/man 这个目录中。当然这个路径也是可修改的：修改 /etc/man_db.conf即可（有的版本是 man.conf 或 manpath.conf 或 man.config 等）

      『man -f Command』可以找到更多有关这个指令的说明文件

      • 从输出的结果来看：
        • 左边部分：指令（或文件）以及该指令所代表的意义（就是那个数字）
        • 右边部分：这个指令建议说明
      • 当使用该指令时，man 只会找数据中的左边那个指令（或文件）的完整名称，有一点不同都不行

      案例说明——例如 man -f man，便可查出 man 指令的多个说明文件

      • 使用 man 1 man 即可查看 man(1)的文件数据
      • 使用 man 7 man 即可查看 man(7)的文件数据

      如果只输入 man man，那么取出的数据就跟搜寻的顺序有关了，搜寻的顺序是记录在 /etc/man_db.conf这个配置文件中的，先搜寻到的那么说明文档就会先被显示出来。一般来说，通常会先找到数字较小的那个（因为排序的原因）。所以 man man会跟 man 1 man结果相同

      『man -k Command』通过关键词来搜索更多有关这个指令的说明文件

      • 不同于『man -f Command』指令的全部匹配，『man -k Command』实现的是关键词的搜索
      • 只要含有该指令那个字眼的（不见得是完整字符串）就都会显示出来
      • whatis 是 man -f的缩写
      • apropos 是 man -k的缩写

      注意：这两个特殊指令要使用的前提是必须要有建立 whatis 数据库才行，这个数据库的建立需要以 root 的身份输入mandb指令，旧版则是makewhatis

      # info page

        在所有的 Unix Like 系统当中，都可以利用 man 来查询指令或者是相关文件的用法。但是在 Linux 里则又提供了一种在线求助的方法，那就是利用 info 这个命令

        基本上 info 与 man 的用途差不多，都是用来查询指令的用法或者文件的格式。但是与 man page 一口气输出一堆信息不同，info page 则是将文件数据拆分成一个一个的段落，每个段落用自己的页面来编写，并且在各个页面中还有类似网页的超链接来调到各不同的页面中，每个独立的页面也被称为一个节点（node）。所以，你可以将 info page 想成是文本模式的网页显示数据

        # 其他有用的文件

        一般而言，指令或者软件制作者都会将自己的指令或者软件的说明制作成『联机帮助文件』。但毕竟不是每个指令都需要作为联机帮助文件的，还有很多的说明需要额外的文件（某些软件不仅告诉你如何做，还会有一些相关的原理会说明）。那么这些说明文件就存放在 /usr/share/doc 这个目录里。另外很多原版软件放出的时候，都会有一些安装须知、预计工作事项、未来工作规划等东西，还有包括可安装的程序等，这些文件也都会放置在这个目录当中

        # 文本编辑器：nano

          基本语法是：nano [选项] [文件名]

          • 创建或打开文件：nano 路径/文件名（如果已有的文件就直接打开，如果没有就新建）
          • 复制一整行：Alt+6 复制一整行
          • 剪贴一整行：Ctrl+K
          • 粘贴：Ctrl+U
          • 翻页：Ctrl+Y 上一页，Ctrl+V 下一页

          注意：nano 默认是自动换行的，使用-w 就可以避免由于自动换行产生空白区域，从而导致无法运行的尴尬局面

          快捷键 说明
          Ctrl+G 取得联机帮助
          Ctrl+X 离开 nano 编辑器：如果你修改了文件，下面会询问你是否需要保存修改。输入 Y 确认保存，输入 N 不保存，按 Ctrl+C 取消返回。如果确认保存，下一步会让你输入想要保存的文件名。如果不需要修改文件名直接回车就行；若想要保存成别的名字则输入新名称然后确定。这个时候也可用 Ctrl+C 来取消返回
          Ctrl+O 保存文件，若你有权限的话就能够保存文件了
          Ctrl+R 从其他文件读入资料，可以将某个文件的内容贴在本文件中
          Ctrl+W 搜索字符串：输入你要搜索的关键字，按 enter。会定位到第一个匹配的文本。Alt+W 定位下一个
          Ctrl+C 说明目前光标所在处的行数与列数等信息
          Ctrl+_ 可以直接输入行号，让光标快速移动到该行

          # 开机和关机

          Windows 是单人多任务操作系统，而 Linux 则是多人多任务。因此在你看不到的屏幕背后其实可能有相当多人同时在一台主机上面工作。所以正常情况下，要关机需要注意以下几件事

          • 观察系统的使用状态，通过这些信息判断是否可以关机了
            • 如果要查看目前有谁在线，可以使用 who 这个指令
            • 如果要查看网络的联机状态，可以使用 netstat -a 这个指令
            • 如果要查看进程情况，可以使用 ps -aux 这个指令
          • 通知在线使用者关机的时刻：要关机总得给在线的使用者一些时间来结束他们的工作，这个时候可以使用 shutdown 的特别指令来达到这一功能
          • 正确的关机指令使用：例如 shutdown 和 reboot 这两个指令

          开机/关机相关指令

          • 将数据同步写入磁盘的指令：sync
          • 惯用的关机指令：shutdown
          • 重新启动、关机：reboot、halt、poweroff

          # 数据同步写入磁盘：sync

          由于所有的数据都得要被读入内存后才能够被 CPU 所处理，但是数据又常常需要由内存写回硬盘当中（例如储存的动作）。由于硬盘速度太慢（相对内存来说），如果常常让数据在内存与硬盘中来回写入/读出，系统的效能就不会太好。 因此在 Linux 系统中，为了加快数据的读取速度，某些已经加载到内存中的数据将不会直接被写回硬盘，而是先暂存在内存当中。如此一来，如果一个数据被你重复的改写，那么由于它尚未被写入硬盘当中，因此可以直接由内存当中读取出来，在速度上一定是快上很多 不过这也会造成些许困扰，那就是万一你的系统因为某些特殊情况造成不正常关机，由于数据尚未被写入硬盘当中，所以就会造成数据的更新不正常。这个时候就需要 sync 这个指令来进行数据的写入动作了。直接在 Terminal 下输入 sync，那么在内存中尚未被更新的数据，就会被写入硬盘中。所以这个指令在系统关机或重新启动之前多执行几次，虽然目前的 shutdown/reboot/halt 等指令均已经在关机前进行了 sync 的执行了，不过多执行几遍，比较放心

          一般情况下，一般账户使用 sync 指令只会更新自己账户的硬盘数据，root 账户则会更新整个系统的数据。因此最好使用 root 账户做同步更新

          # 关机指令：shutdown

            • 如果是在主机前面以实体终端机（tty1~tty7）来登入系统时，不论用什么身份都能够关机
            • 如果是使用远程管理工具来登入主机，那关机就只有 root 有权力

            shutdown 指令会通知系统内的各个程序，并且将通知系统中的一些服务来关闭。shutdown 可以达成如下工作：

            • 可以自由选择关机模式：是要关机还是重新启动
            • 可以设定关机时间：可以设定成现在立刻关机，也可以设定某一个特定的时间才关机
            • 可以自定义关机讯息：在关机前，可以将自己设定的信息传送给在线的 user
            • 可以仅发出警告讯息：有时有可能你要进行一些测试，而不想让其他的使用者干扰，或者是明白的告诉使用者某段时间要注意以下。这个时候可以使用 shutdown 来吓一吓使用者，并不是真的要关机

            指令语法：/sbin/shutdown [-krhc] [时间] [警告讯息]

            选项与参数 说明
            -k 不是真的要关机，只是发送警告讯息出去
            -r 在将系统的服务停掉之后，重新启动（常用）
            -h 将系统的服务停掉之后，立即关机（常用）
            -c 取消已经在进行的 shutdown 指令内容
            时间 指定系统的关机时间，若没有指定时间，则默认 1 分钟后自动进行

            案例：/sbin/shutdown -h 10 'I will shutdown after 10 mins'

            • 在执行 shutdown 之后，系统会告诉大家，这部机器将会在 10 分钟之后关机，这段信息将会显式在目前登入者的屏幕前方
            • 你可以输入 shutdown -c 来取消这次的关机指令
            • 而如果你什么参数也没有添加，单纯执行 shutdown 之后，系统默认会在 1 分钟后进行关机动作
            选项与参数 说明
            shutdown -h now 立即关机，其中 now 相当于时间为 0 的状态
            shutdown -h 20:25 系统在今天的 20:25 关机，如果在 21:25 才下达此命令，则隔天才关机
            shutdown -h +10 系统再过 10 分钟后自动关机
            shutdown -r row 系统立即重新启动
            shutdown -r +30 'message' 再过 30 分钟会重新启动，并显示后面的讯息给所有在线的使用者
            shutdown -k now 'message' 仅发出警告信息，系统并不会关机

            # 重新启动、关机：reboot,halt,poweroff

            基本上，在预设的情况下，这几个指令都会完成一样的工作（全部的动作都是去调用 systemctl这个重要的管理命令），所以你只要记得其中一个就好了

            • reboot：重新启动
            • halt：进入系统停止的模式，屏幕可能会保留系统已经停止的信息，这与你的电源管理模式有关
            • poweroff：系统关机，所以没有提供额外的电力，屏幕空白
            • suspend：进入休眠模式

            # 管理所有服务的指令：systemctl

            目前系统中所有服务的管理指令是 systemctl，上面谈到的 reboot,halt,poweroff,shutdown 等，其实都是调用了这个 systemctl 指令

            systemctl 指令基础语法：systemctl [指令]

            例如：

            • 系统重新启动：systemctl reboot
            • 系统直接关机：systemctl poweroff
