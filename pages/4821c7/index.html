正则表达式

正则表达式（RE,regular expression）就是处理字符串的方法，它是以行为单位来进行字符串的处理行为。正则表达式通过一些特殊符号，可以轻易达到『搜索/删除/替换』某特定字符串的处理程序

正则表达式的字符串表示方式依照不同的严谨度分为：基础正则表达式（BRE）和扩展正则表达式（ERE）

  • 基础正则：^、$、^$、.、*、.*、\、[]、[-]、[^]、{}、{n}、{n,}、{n,m}
  • 扩展正则：+、?、|、()、()+

正则和通配符并不一样

  • 正则（RE）是用来宫高级语言进行过滤匹配字符的，它支持 grep、sed、awk、find 等命令
  • 通配符（glob）是用来匹配文件名的，Linux 下面大部分命令都支持通配符

注意

  • 由于正则表达式是处理字符串的一种表示方式，那么对字符排序有影响的语系数据就会对正则表达式的结果有影响。所以使用正则表达式时，需要特别留意当时环境的语系，否则可能会发现与别人不相同的选取结果
    • en_US.UTF-8（大部分情况没问题），如果出现问题则修改字符集为 C export LANG=C
  • grep : 最早的文本匹配程序，使用基本正则表达式来匹配文本（alias grep='grep --color=auto'）
  • egrep : 扩展式 grep，使用扩展正规表达式来匹配文本（egrep 是 alias egrep='egrep --color=auto'）

# 基础正则

    匹配次数 说明
    . 匹配任意单个字符
    * 代表『重复前一个字符，0 到无穷次』的意思
    例如『oo*』的含义是匹配至少 1 个 o 以上的字符串
    .* 任意长度的任意字符
    [] 匹配指定范围内的任意单个字符；注意：[]里面的内容去掉了特殊含义
    [-] 若有连字符（-）在括号内时，指的是连续字符的范围
    [^] 取反，排除；不在指定范围的字符
    {} 连续的字符范围
    {n} 匹配前面的字符 n 次
    {n,} 匹配前面的字符最少 n 次
    {,n} 匹配前面的字符最多 n 次
    {n,m} 匹配前面的字符最少 n 次，最多 m 次
    \? 匹配前面的字符 0 或 1 次
    \+ 匹配前面的字符至少 1 次
    位置锚定 说明
    ^ 行首字符
    $ 行尾字符
    ^PATTERN$ 用户模式匹配整行
    ^$ 空行
    \< 或 \b 词首锚定，用于单词模式的左侧
    \> 或 \b 词尾锚定，用于单词模式的右侧
    \<PATTERN\> 匹配整个单词
    字符匹配 说明
    \ 转义字符
    [:lower:] 代表小写字符，即 a-z
    [:upper:] 代表大写字符，即 A-Z
    [:digit:] 代表数字，即 0-9
    [:alpha:] 代表任意英文大小写字符，即 A-Z，a-z
    [:alnum:] 代表英文大小写字符及数字，即 0-9，A-Z，a-z
    [:blank:] 代表 Space 空格键与 Tab 键两者
    [:graph:] 除了空格键与 Tab 按键外的其他所有键
    [:cntrl:] 代表键盘上面的控制按键，即包括 CR、LF、Tab、Del……
    [:punct:] 代表标点符号，即 "、'、?、!、;、:、#、$……
    [:space:] 任何会产生空白的字符，包括空格键、Tab、CR……
    [:xdigit:] 代表 16 进制的数字类型，包括 0-9、A-F、a-f 的数字与字符
    [:print:] 代表任何可以被打印出来的字符

    # 扩展正则

    特殊符号 说明
    + 前一个字符连续出现 1 次或 1 次以上
    ? 前一个字符连续出现 0 次或 1 次
    \| 匹配 1 个字符或多个字符，常用在匹配单词的功能上
    () 被括起来的内容表示一个整体

    # 文本处理：grep/sed/awk

    • grep：主要用来过滤内容；grep 命令是过滤速度最快的
    • sed：主要用来替换、修改文件内容，取行
    • awd：主要用来取列，对比/比较、统计计算

    # 过滤文件内容：grep

    指令语法：grep [选项] '搜寻字符串' 数据来源文件名

    选项参数 说明
    -a 将 binary 文件以 text 文件的方式检索数据
    -c 计算找到『搜寻字符串』的次数，不显示搜索结果
    -i 忽略大小写
    -n 显示匹配的行号
    -o 仅显示匹配到的字符串
    -w 匹配整个单词
    -v 反向搜索，输出没有匹配到的字符串的那些行
    -A n 显示搜索行及其向下相邻的 n 行
    -B n 显示搜索行及其向上相邻的 n 行
    -C n 显示搜索行及其向上下相邻的各 n 行
    -E egrep 支持扩展正则
    --color=auto 对匹配到的文本着色显示（CentOS7 里默认显示）

    # 文件内容修改工具：sed

      指令语法：sed [选项] [动作] 数据来源文件名

      选项参数 说明
      -n 使用 silent 模式，在一般的 sed 用法中，所有来自 STDIN 的数据一般都会被列出在屏幕上。但如果加上-n 参数后，则只有在经过 sed 特殊处理的那一行（或动作）才会被列出来
      -e 直接在命令行模式上进行 sed 的动作编辑
      -f 直接将 sed 的动作写在一个文件内。-f filename 则可以执行 filename 内的 sed 动作
      -r sed 的动作支持的是扩展正则表达式的语法（默认是基础正则表达式语法）
      -i 直接修改读取的文件内容，而不是由屏幕输出

      动作语法：[n1[,n2]]功能

      • n1,n2：不见得会存在，一般代表『选择进行的动作的行数』。例如：我的动作是需要在 10 到 20 行之间进行的
      功能 说明
      s 替换：可以直接进行替换的工作；通常这个 s 的动作可以搭配正则表达式
      d 删除：因为是删除，所以 d 后面通常不接任何内容
      p 打印：将某个选择的数据打印出来。通常 p 会与参数 sed -n 一起运作
      c 取代（replace 替代这行的内容）：c 的后面可以接字符串，这些字符串可以取代 n1,n2 之间的行
      a 新增（append 追加，向指定的行货每一行追加内容，行后面）：a 的后面可以接字符串，而这些字符串会在目前的下一行出现
      i 插入（insert 插入，向指定的行或每一行插入内容，行前面）：i 的后面可以接字符串，而这些字符串会在目前的上一行出现

      增加格式（cai）

      查找格式（p）：

      • 2p：指定行号进行查找
      • 1,5p：指定行号范围进行查找
      • /sbgong/p：类似于 grep 过滤，//里面可以写正则
      • /10:00/,/11:00/p：表示范围的过滤

      删除格式（d）：

      • 2d：指定行号进行删除
      • 1,5d：指定行号范围进行删除
      • /sbgong/d：类似于 grep 过滤删除，//里面可以写正则

      替换格式（s）：

      • s#要匹配的内容#要替换的内容#g、s@要匹配的内容@要替换的内容@g、s/要匹配的内容/要替换的内容/g 均可
      • s：substitute 表明是替换的意思
      • g：global 全局替换，意味着 sed 会替换每行所有匹配的内容；不加 g 时，默认只会替换每行第一个匹配的内容

      sed 本身是一个管线命令，可以将数据进行增删改查等功能

      我们知道，Vim 采用的是交互式文本编辑模式，你可以用键盘命令来交互性地插入、删除或替换数据中的文本。但 sed 命令不同，它采用的是流编辑模式，最明显的特点是，在 sed 处理数据之前，需要预先提供一组规则，sed 会按照此规则来编辑数据

      sed 会根据脚本命令来处理文本文件中的数据，这些命令要么从命令行中输入，要么存储在一个文本文件中，此命令执行数据的顺序如下：

      • 每次仅读取一行内容
      • 根据提供的规则命令匹配并修改数据
        • 注意，sed 默认不会直接修改源文件数据，而是会将数据复制到缓冲区中，修改也仅限于缓冲区中的数据
      • 将执行结果输出 当一行数据匹配完成后，它会继续读取下一行数据，并重复这个过程，直到将文件中所有数据处理完毕

      # 数据处理工具：awk

      它的运作模式是『预处理+逐行处理+最终处理』：预处理来定义变量，逐行处理来修改变量，最终处理来打印变量

        指令语法：awk [选项] 'BEGIN{commands}匹配规则{执行命令}END{commands}' 文件名

        • BEGIN{ commands }开始块就是在程序启动的时候执行的代码部分，并且它在整个过程中只执行一次。一般情况下，我们可以在开始块中初始化一些变量。注意：开始块部分是可选的，你的程序可以没有开始块部分
        • 匹配规则{执行命令}部分匹配该行内容成功后，才会执行 commands 的内容
        • END{ commands }结束块是在程序结束时执行的代码。注意：结束块部分也是可选的
        选项参数 说明
        -F fs 指定以 fs 作为输入行的分隔符，awk 命令默认分隔符为空格或制表符
        -f file 从脚本文件中读取 awk 脚本指令，以取代直接在命令行中输入指令
        -v var=val 在执行处理过程之前，设置一个变量 var，并给其设备初始值为 val

        awk 的脚本命令由 2 部分组成，分别为匹配规则和执行命令：匹配规则{执行命令}

        • 这里的匹配规则，用来指定脚本命令可以作用到文本内容中的具体行，可以使用字符串（比如 /demo/，表示查看含有 demo 字符串的行）或者正则表达式指定
        • 如果没有指定执行命令，则默认会把匹配的行输出；如果不指定匹配规则，则默认匹配文本中所有的行

        取出行的匹配规则

        • NR==1：取出第一行
        • NR>=1 && NR<=5：取出 1 到 5 行范围
        • /sbgong/：使用正则匹配到行

        取出列的匹配规则：{print xxx}

        • -F：指定分隔符；指定每一列的结束标志（默认是空格或者 tab 键）
        • $数字：取出某一列
        • $0：取出整行的内容
        内置变量 说明
        NR 行号，从 1 开始，多文件时候也是连续接着计数
        FNR 各文件分别计数的行号，多文件的时候会和 NR 不同，它会重新计数
        NF 一行中字段数量，最后一个字段内容可以用$NF 取出
        FS 每个字段的结束标记
        OFS 输出字段的分隔符（默认是空格）
        RS 行分隔符（默认以\n 作为一行的结尾），单行分割成多行用到

        # 格式化打印：printf

          指令语法：printf '打印格式' 实际内容

          修饰符 说明

          控制字母 说明
          d 十进制整数
          i 十进制整数
          u 不带正负号的十进制值
          x 十六进制整数（a-f）
          X 十六进制整数（A-F）
          o 八进制整数
          s 字符串
          f 浮点数
          e 用科学计数法表示一个数
          g 用科学计数法或浮点数表示一个数（选择较短的格式）
          c ASCII 字符，即显示对应参数的第一个字符
          b 退格键（backspace）
          n 换行符
          r 回车符
          t 水平制表符
          v 垂直制表符
          a 警告声音输出
          f 换页

          # 文件对比工具：diff/cmp

            diff 是用来比较两个纯文本文件之间的差异的，并且以行为单位来比对的。

            指令语法：diff [选项] from-file to-file

            • from-file：作为原始文件的文件名
            • to-file：作为目的比对文件的文件名

            注意：from-file 或 to-file 可以使用 - 取代，它代表着 STDIN 的意思

            选项参数 说明
            -b 忽略一行当中多个空白的差异
            -B 忽略空白行的差异
            -i 忽略大小写的不同

            cmp 是用来比较两个非纯文本文件之间的差异的，它主要利用字节单位去做对比

            指令语法：cmp [-l] file1 file2

            • -l：将所有的不同点的字节处都列出来，因为 cmp 默认仅会输出第一个发现的不同点
