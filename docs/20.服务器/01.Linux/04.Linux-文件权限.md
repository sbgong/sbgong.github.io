## 文件的归属

Linux 比较强大的一点就是它的多人多任务环境。而各个用户可能具有保密的文件数据，因此需要根据具体的使用情况设置不同的文件权限。Linux 中为了解决这个问题提出了用户和用户组的概念

> Linux 中将文件的身份分为三个类别，分别是文件拥有者(owner)，所属群组(group)，其他人(others)，每个文件都可单独地对不同的身份执行不同的权限(read,write,execute)。在用户和用户组之外还有一个超级用户，该用户就是 root，这个用户类似于 windows 中的管理员，可以针对系统中的任何文件做出更改，在具体操作时要谨慎使用

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab owner 
文件拥有者确定文件的归属，并且文件拥有者也可以对自己的文件设置权限，允许或者拒绝某些用户的访问
:::
::: tab group 
群组在进行团队协作的时候会非常有用。当执行协作任务的时候，某些文件可能需要让多个用户进行共用，此时需要设置用户组的相应权限，以确保在用户组中的所有用户都有权限去访问对应的文件
:::
::: tab others 
others 就可以根据字面意思来理解。对于一个文件，如果该用户并不是文件的拥有者，又被排除在用户组之外，那么这个用户就被赋予了其他人的身份，因此不会享有用户拥有者和用户组成员的待遇。如果想要访问某个文件，则需要等待该文件针对其他人的权限被设置之后才能够进行访问
:::
::::

::: warning
默认情况下，Linux 中所有的系统上的帐号和一般身份身份，以及 root 的相关信息，都记录在 `/etc/passwd` 文件中。个人的密码则是记录在 `/etc/shadow` 文件中。所有的组名都记录在 `/etc/group` 中。因此这三个文件的重要性就不言而喻了，具体操作时一定要慎重
:::

### 文件的类型与权限

**第一部分：代表这个文件的类型与权限；共 10 个字符**

::: info 第一个字符代表文件类型

- `d` 表示目录
- `-` 表示普通文件
- `l` 表示链接文件（软连接，即快捷方式）
- `b` 表示块设备文件，就是存储数据以供系统存取的接口设备，简单而言就是硬盘
- `c` 表示字符设备文件，串行端口的接口设备，例如键盘、鼠标等
- `s` 表示网络套接字（socket），通常用在网络数据连接中
- `p` 表示管道（pipe）

:::

![rwx](https://cdn.jsdelivr.net/gh/sbgong/static/blog/server/linux/202205021442106.png)

::: info 接下来的字符中，以三个为一组，且均为『rwx』的三个参数的组合

- 前三位是文件拥有者的权限
- 中间三位是群组权限
- 后三位其他人权限

其中『r』代表可读(read)-4，『w』代表可写(write)-2，『x』代表可执行(execute)-1。要注意的是，这三个权限的位置不会改变，如果没有权限，就会出现减号『-』
:::

### 硬链接的数量

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab 硬链接 
一般情况下，文件名和 inode 号码是一一对应的，每个 inode 号码对应一个文件名。但是 Linux 系统允许一个 inode 值对应多个文件名。在同一个分区内，inode 号相同的文件就是硬链接文件

- 硬链接的文件，inode 相同，属性也一致
- 只能在同一个磁盘分区下，同一个文件系统下创建硬链接
- 可以对已存在的文件做硬链接，该文件的硬链接数，至少是 1，为 0 就表示文件不存在
- 删除一个硬链接，不影响其他相同 inode 号的文件
- 不能对文件夹创建硬链接，只有文件可以
- 文件夹的硬链接，默认是 2 个，每个目录下面都有一个硬链接"."，和对应上级目录的硬链接".."
  - 如果在该空目录下创建新的空目录，硬链接数：3，因为新创建的子目录有".."
  - 在父目录里创建一个子目录，父目录的链接数增加 1。所以文件夹的硬链接数量=2+（第一层子目录总数）
  - 对于目录，不可以创建硬链接，但可以创建软链接（目录是一个特殊的文件，目录保存的是当前目录下的文件名字）

**总结：第三部分的『硬链接个数』就是表示有多少文件名链接到此 inode 节点上**
:::
::: tab 软链接 
Linux 系统中的软链接可以理解为 Windows 下的快捷方式。软链接文件储存的是源文件的路径

- 软链接文件的 inode 号和源文件不同
- 删除普通软链接，不影响源文件
- 删除源文件，软链接找不到目标，报错提示
- 可以操作软链接完成对源文件的修改
:::
::::

操作系统中专门用于管理和存储文件的信息软件称为文件系统。一个新的磁盘格式化文件系统后，就有了两个存储空间：一个是 inode 存储空间，一个是 block 存储空间。inode 数量和 block 数量由磁盘的容量决定

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab inode 
inode 存储空间：存储设备上所有文件名对应的元数据信息（可通过 stat 命令查看文件的属性信息）
- inode 空间内存放文件的属性信息，也存放 block 的指针信息
- 这些元数据信息不包含文件名，文件的名称信息存储在上一级目录 block 中
- 每个 inode 的大小一般是 128 字节或 256 字节，格式化文件系统时确定 inode 的总数，这也是硬盘格式化后可用内存不是全部内存空间的原因

:::
::: tab block 
block 存储空间：存储设备上所有文件的内容和数据
- 默认大小为 4K，可以进行调整；每读取一个 block 就会消耗一次磁盘 IO
- 文件小于 4K 时，一个 block 存不满，剩余空间只能浪费，无法存储其他数据
- 文件大于 4K 时，占用多个 block（即一个文件对应一个 inode 和多个 block）

:::
::::


理解 inode，要从文件储存说起。文件储存在硬盘上，硬盘的最小存储单位叫做扇区。每个扇区储存 512 字节（相当于 0.5KB）。操作系统读取硬盘的时候，不会一个个扇区地读取，这样效率太低，而是一次性连续读取多个扇区，即一次性读取一个块。这种由多个扇区组成的块是文件存取的最小单位。块的大小，最常见的是 4KB，即连续八个扇区组成一个块

文件数据都储存在块中，那么很显然，我们还必须找到一个地方储存文件的元信息，比如文件的创建者、文件的创建日期、文件的大小等等。这种储存文件元信息的区域就叫做 inode(index node)，中文译名为索引节点。inode 号即索引节点号，文件名仅是为了方便大家记忆而使用的，而操作系统则是通过 inode 号寻找正确的文件数据块（Linux 读取文件流程：根据文件名对应的 inode 号码找到 inode 相关信息，然后根据 inode 信息找到文件数据所在的 block 读出数据）

由于 inode 号码与文件名分离，导致一些 UNIX/LINUX 系统具有以下现象

- 当文件名包含特殊字符，可能无法正常删除文件，直接删除 inode，也可以删除文件
- 移动或重命名文件时，只改变文件名，不影响 inode 号码
- 打开一个文件后，系统通过 inode 号码来识别该文件，不再考虑文件名

### 文件所有者与所属用户组

对于文件来说，每个文件都有一个特定的所有者，也就是对该文件具有所有权的用户。同时在 Linux 系统中，用户是按组分类的，一个用户属于一个或多个组。而文件所有者以外的用户又可以分为文件所有者的同组用户和其他用户

- 第三部分：文件所有者的名称
- 第四部分：文件所属用户组的名称

### 文件大小

**第五部分：文件的容量大小，默认单位是字节 bytes**

- 对于普通文件，显示的是文件的大小（字节数）
- 对于目录，是指目录大小（目录内 inode 列表所占空间，而不是目录内文件所占的空间大小）
- 对于设备文件是指主设备号（第 6 列为次设备号）
- 对于符号链接/软链接，是指链接文件的路径名的字节数

### 文件修改日期

**第六部分：是文件的创建日期或者是最近的修改日期（mtime）**

- 这里的时间是该文件最后被更新的时间（更新操作包括文件创建、内容更新、文件名更新等）
- 这里的内容分别为日期（月/日）及时间。如果这个文件被修改的时间距离现在太久了，那么时间部分仅会显示年份而已

具体可参考[文件的三种时间状态](/pages/4fb970/#文件的时间状态)

### 文件名称

如果文件名之前多一个『.』，则代表这个文件为隐藏文件

## 修改文件属性与权限

- `chgrp`：改变文件或目录的所属群组（chang owner）
- `chown`：改变文件或目录的拥有者（change group）
- `chmod`：改变文件或目录的权限（change modify）

### chgrp：改变文件所属群组

**指令语法：chgrp [选项] 用户组 文件或目录**

| 选项参数                        | 说明                                                 |
| ------------------------------- | ---------------------------------------------------- |
| `-R` 或 `--recursive`           | 递归处理，将指定目录下的所有文件都更新为这个群组之意 |
| `-f` 或 `--quiet` 或 `--silent` | 不显示错误信息                                       |
| `-v` 或 `--verbose`             | 显示指令执行过程                                     |
| `-c` 或 `--changes`             | 效果类似 `-v`参数，但仅汇报更改的部分                |


::: warning
要注意，要被改变的组名必须要在 `/etc/group` 这个文件内存在才行，否则就会显式错误
:::

### chown：改变文件拥有者

::: tip chown 指令语法
**chown 的用途很多，它还可以顺便直接修改群组的名称，甚至可以单纯只修改群组 `chown .群组名 文件`**

- chown [选项] 所有者 文件或目录
- chown [选项] 所有者:组 文件或目录

选项 `-R` 可以递归变更连同目录下所有文件的拥有者
:::

::: warning
要注意，要被改变的用户必须要在 `/etc/passwd` 这个文件内存在才行，否则就会显式错误
:::

### chmod：改变文件的权限

chmod 用于修改某个目录或文件的访问权限。权限的设定方式有两种：可以使用**数字**或者是**符号**来进行权限的变更

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab 使用语法—数字形式 
> 数字类型更改文件权限——chmod 指令语法：chmod [选项] xyz 文件

xyz：每一个字母都对应 rwx 属性的数值相加，合起来 xyz 就是 3 个权限数字

- `r` 用数字 4 来代表
- `w` 用数字 2 来代表
- `x` 用数字 1 来代表

**文件的基本权限有 9 个，分别是三种文件身份对应的三种权限方式，每种身份各自的三个权限分数是需要累加的**
:::
::: tab 使用案例—数字 
例如将权限[-rwxrwxrwx]变为[-rwxr-xr--]，需要执行的指令是：`chmod 754 .bashrc`

- owner = rwx = 4+2+1 = 7
- group = r-x = 4+0+1 = 5
- others = r-- = 4+0+0 = 4
:::
::: tab 使用语法—符号形式 
文件的三种身份和三种权限都可以使用其简写字符形式相互组合来表示这九个权限值

- 文件所有者（user）对应于 `u`
- 用户组（group）对应于 `g`
- 其他人（others）对应于 `o`
- 全部的身份（all）对应于 `a`

![符号说明](https://cdn.jsdelivr.net/gh/sbgong/static/blog/server/linux/202205021442353.png)

:::
::: tab 使用案例—符号
```shell
chmod u=rwx,go=rx 文件名      # 等价于 [-rwxr-xr-x]
chmod u=rwx,g=rx,o=r 文件名   # 等价于 [-rwxr-xr--]
chmod a+w 文件名              # 给每个人增加写入的权限
chmod a-x 文件名              # 给每个人去掉可执行的权限
```
:::
::::

## 目录与文件之权限意义

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab 权限对文件的重要性 
文件是实际含有数据的地方，包括一般文本文件、数据库内容文件、二进制可执行文件等

- r（read）：可读取文件的实际内容，如读取文本文件的文字内容
- w（write）：可以编辑、新增或者是修改该文件的内容（但不包含删除该文件）
- x（execute）：该文件具有可以被系统执行的权限
:::
::: tab 权限对目录的重要性
目录存储的是文件名列表，文件名与目录有强烈的关联

- r（read contents in directory）：表示具有读取目录结构列表的权限；可以利用 ls 这个指令将该目录的内容列表显示出来
- w（modify contents of directory）：表示具有变动该目录结构列表的权限
  - 建立新的文件与目录
  - 将已存在的文件或目录进行重命名
  - 移动该目录内的文件、目录位置
  - 删除已经存在的文件与目录（不论该文件的权限为何）
- x（access directory）：表示用户能否进入该目录
:::
::::


## Linux 文件种类与扩展名

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab 普通文件
- 纯文本文件：文件内容可以直接读到数据（数字、字母等）。例如 `cat ~/.bashrc` 就可以看到该文件的内容
- 二进制文件：在 Linux 当中的可执行文件（script）
- 数据格式文件(data)：有些程序在运作的过程中会读取特定格式的文件，那些特定格式的文件被称为数据文件（data file）
  - 例如：Linux 在使用者登入时，都会将登录的数据记录在 `/var/log/wtmp` 那个文件内，该文件时一个 data file，它能通过 last 这个指令读出来，但使用 cat 时，会读出乱码
:::
::: tab 设备与装置文件
设备与装置文件：通常都集中在/dev 这个目录之下，主要分为两类
- 区块设备文件（b:block）：存储数据以提供系统随机存取的接口设备（硬盘与软盘）
- 字符设备文件（c:character）：一些串行端口的接口设备（键盘与鼠标），这些设备特点就是一次性读取，不能够截断输出
:::
::: tab 文件扩展名
一个 Linux 文件能不能被执行，和它的权限有关，与文件名没有一点关系。不过仍然希望可以借扩展名来了解文件是什么东西

- `.sh`：脚本或批处理文件，因为批处理文件是使用 shell 写成的，所以扩展名就写成 .sh
- `.Z` `*.tar` `*.tar.gz` `*.zip` `*.tgz`：经过打包的压缩文件。这是因为压缩软件分别为`gunzip/tar`等，由于不同的压缩软件，而取其相关的扩展名

Linux 系统的文件名只是让人了解该文件可能的用途，真正的执行与否仍然需要权限的规范才行
:::
::: tab 文件名长度的限制
单一文件或目录的最大容许文件名为 255 字节

- 若是以 ASCII 英文占用一个 bytes 来说，则大约可达 255 个字符长度
- 若是以每个中文占用 2bytes 来说，最大文件名大约在 128 个中文字

文件名的限制，最好避免出现以下特殊字符
`*` `?` `;` `&` `!` `|` `\` `'` `"` `>` `<` `[` `]` `{` `}` `(` `)`
:::
