选择符

# 元素选择符

选择符 说明
通配符选择符 *
类型选择符 E
类选择符 E.class
ID 选择符 E#id

    CSS2 引入了一个新的选择符，名为通用选择符，写作星号（*），这个选择符匹配所有元素，很像是通配符

    * {color: red;}
    1
    1

    文档中的元素是最基本的选择符

    • 为 HTML 文档编写样式时，选择符一般是 HTML 预定义的某个元素
    • 在 XML 中，什么都可以作为选择符。因为 XML 允许创建新标记语言，所以什么都可以作为元素名称
    html {color: black;}
    1
    1

    应用样式而不关心所涉及的元素，最常使用的是类选择符

    <p class="warning">hello world</p>
.warning { font-weight: blod; }
    1
    2
    1
    2

    类选择符的使用步骤：

    • 先修改文档的标签，让类选择符起作用。修改方式是设定 class 属性值（注意：class 属性的值是区分大小写的）
    • 然后为 class 属性值设置样式：在 class 属性值的前面加上点号（.）

    多个类选择符的使用：

    • 根据 HTML 规范，class 属性的值可以是多个值，值之间使用空格分隔；且词的顺序无关紧要，结果是完全一样的
    • 多个类选择符可以串在一起，选择的是同时具有多个类名的元素，而且对类名的顺序没有要求

    注意：使用类选择符、ID 选择符、属性选择符、伪类选择符或为元素选择符时，如果没有依附元素选择符，隐式蕴含通用选择符（使用 * 符号表示）

    <p id="lead-para">Hello World</p>
#lead-para { font-weight: blod; }
    1
    2
    1
    2

    ID 选择符的使用步骤：

    • 先修改文档的标签，让 ID 选择符起作用。修改方式是设定 id 属性值
    • 然后为 id 属性值设置样式：在 id 属性值的前面加上点号（#）

    注意：

    • 与类选择符不同，ID 选择符不能串在一起使用，因为 ID 属性的值不能是以空格分隔的列表
    • 根据 HTML 规范，ID 属性的值是区分大小写的
    • 与类选择符一样，ID 选择符中的通用选择符也可以省略

    # 关系选择符

    选择符 说明
    选择符分组/群组选择符 E1,E2…
    后代选择符 E F
    子选择符 E > F
    相邻选择符 E + F
    兄弟选择符 E ~ F

      如果想把同一个样式应用到多个元素上（把多个样式应用到一个元素或一组元素上），请使用群组选择符

      群组选择符：多个选择符之间以逗号分隔；右边的规则应用到前面的选择符应用的元素上

      h2,p {color: gray;}
      1
      1

      后代选择符：规则中的选择符由两个或多个空格分隔的选择符构成。选择符之间的空格是一种连接符。注意：两个元素之间的层级间隔可以是无限的

      ul ol ul em { color: gray; }
      1
      1

      子选择符：只选择一个元素的子元素，要使用子代连接符，即大于号（>），子代连接符两侧的空格是可选的

      h1 > strong { color: red; }
      1
      1

      相邻选择符：选择同一个父元素中紧跟在另一个元素后面的一个元素，要使用紧邻同胞连接符，即一个加号（+）

      /* 选择紧跟在h1元素后面的p元素 */
h1 + p { margin-top: 0; }
      1
      2
      1
      2

      兄弟选择符：选择一个元素后面同属一个父元素的另一个元素。语法：Selectors Level 3 引入了一个新的同胞连接符，使用波浪号（~）表示

      /* 选择和h2同属一个父元素的ol元素文本倾斜 */
h2 ~ ol { font-style: italic; }
      1
      2
      1
      2

      # 属性选择符

      选择符 说明
      E [attr] 只匹配属性，不管属性值；任何属性值都能选取到
      E [attr = "val"] 精确匹配，明确属性和属性值
      E [attr ^= "val"] 选择的元素有 attr 属性，且其值以 val 开头
      E [attr $= "val"] 选择的元素有 attr 属性，且其值以 val 结尾
      E [attr *= "val"] 选择的元素有 attr 属性，且其值包含子串 val
      E [attr ~= "val"] 选择的元素有 attr 属性，且其值是包含 val 这个词的一组词
      E [attr | = "val"] 选择的元素有 attr 属性，且其值以 val 和一个英文破折号（-）开头，或者值就是 val 本身

      不管是类选择符还是 ID 选择符，我们选择的起始都是属性的值，在其他标记语言中，这样编写的类选择符和 ID 选择符可能无法使用（class 和 ID 属性或许根本不存在），为了解决这个问题，CSS2 引入了属性选择符，根据属性及其值选择元素。属性选择符大致可以分为四类：简单属性选择符、精准属性值选择符、部分匹配属性值选择符、起始值属性选择符

        简单属性选择符：如果想选择具有某个属性的元素，而不管属性的值是什么，可以使用简单属性选择符

        /* 若想选择具有class属性的所有h1元素 */
h1[class] {color:silver;}
/* 把具有title属性的元素选中 */
*[title] {font-weight: bold;}
/* 还可以基于多个属性选择：若想把同时具有href和title属性的HTML超链接 */
a[href][title] {font-weight: bold;}

        精准属性值选择符：只选择属性为特定值的元素。与选择属性时一样，可以把多个属性和值选择符串在一起

        plant[moon="1"] {font-weight: bold;}
/* 选取同时满足以下两个属性值的元素 */
a[href="/images"][title="image"] {font-size: 200%;}
/* 注意：属性值必须与指定的值完全一致 */
<plant type="barren rocky">Mercury</plant>
<!-- 正确的形式 -->
plant[type="barren rocky"] {font-weight: bold;}
<!-- 错误的形式 -->
plant[type="barren"] {font-weight: bold;}

        部分匹配属性值选择符：根据属性值的一部分进行选择元素，而不是完整的值

        /* 匹配属性值开头的子串 */

/* 选择class属性的值为 btn 或者以 btn- 开头的元素 */
*[class|="btn"]{}
<button class="btn-small-arrow-active">Click me</button>

        CSS Selectors Level 4 为属性选择符引入了一个不区分大小写的选项。在结束方括号前加上 i，属性选择符便不管文档语言的要求，匹配属性值时不区分大小写

        • 它只针对属性选择符中的值，属性的大小写还是区分的
        • 截至 2017 年年末，Opera Mini、Android 浏览器和 Edge 不支持这个选项
        /* 如此便能匹配href属性的值以.pdf结尾的任何a元素，而不管P、D和F三个字母的大小写 */
a[href $='.PDF' i]

        # 伪类选择符

        伪类选择符：利用这种选择符可以为文档中不一定真实存在的结构指定样式，或者为某些元素的特定状态赋予幽灵类

        • CSS 允许把伪类拼接串联在一起，拼接伪类的顺序其实没什么关系；但注意不要拼接相互排除的伪类，这样匹配不了任何东西
        • 伪类的效果就是把某种幽灵类应用到伪类依附的元素上
        /* 选取有鼠标悬停的未访问链接 */
a:link:hover {
}
        1
        2
        3
        1
        2
        3
          • 动态伪类：它们在页面渲染之后根据页面的变化而变化
          • 伪类顺序是固定的：link → visited → focus → hover → active
          选择符 说明
          :link 指代用作超链接的锚记（即具有 href 属性），而且指向尚未访问的地址
          :visited 指代指向已访问地址的超链接
          :focus 指代当前获得输入焦点的元素，即可以接受键盘输入或以某种方式激活
          :hover 指代鼠标指针放置其上的元素，例如鼠标指针悬停在超链接上
          :active 指代由用户输入激活的元素，例如用户单击超链接时按下鼠标按键的那段时间
          选择符 说明
          :enabled 指代启用的用户界面元素，即接受输入的元素
          :disabled 指代禁用的用户界面元素，即不接受输入的元素
          :checked 指代由用户或文档默认选中的单选按钮或复选框
          :indeterminate 指代既不选中也没有不选中的单选按钮或复选框；这个状态只能由 DOM 脚本设定，不能由用户设定
          :default 指代默认选中的单选按钮、复选框或选项
          :valid 指代满足所有数据有效性语义的输入框；表示用户输入的值满足全部数据验证条件
          :invalid 指代不满足所有数据有效性语义的输入框；表示用户输入的值不满足全部数据验证条件
          :in-range 指代输入的值在最小值和最大值之间的输入框
          :out-of-range 指代输入的值小于控件允许的最小值或大于控件允许的最大值的输入框
          :required 指代必须输入值的输入框
          :optional 指代无需一定输入值的输入框；匹配没有 required 属性的表单控件，或者 required 属性值为 false 的控件
          :read-write 指代可由用户编辑的输入框
          :read-only 指代不能由用户编辑的输入框
          选择符 说明
          :root 选择文档的根元素，在 HTML 中，根元素始终是 html
          :empty 选择空元素；选择没有任何子代的元素，甚至连文本节点都没有（包括文本和空白）
          :only-child 匹配完全没有同胞的元素
          :only-of-type 匹配同胞中唯一的那种类型的元素
          :first-child 选择一个元素的第一个子元素
          :last-child 选择一个元素的最后一个子元素
          :first-of-type 选择第一个元素中某种元素的第一个
          :last-of-type 选择第一个元素中某种元素的最后一个
          :nth-child(an+b) 选择第 n 个子元素
          :nth-last-child(an+b) 从一组同胞的最后一个元素开始，从后向前计算，选择第 n 个子元素
          :nth-of-type(an+b) 选择第 n 个某种子元素
          :nth-last-of-type(an+b) 从一组同胞的最后一个元素开始，从后向前计算，选择第 n 个某种子元素

          括号中使用简单的代数式定义公式

          • 代数式的形式为：an+b 或 an-b
          • 其中 a 和 b 是具体的整数，n 原封不动
          • 也可以使用两个特殊的关键字：even（偶数）和 odd（基数）
          选择符 说明
          :target 匹配文档中特定"id"点击后的效果
          :lang(fr) 语言选择元素
          :not(简单选择符) 否定伪类；匹配不符合当前选择器的任何元素
          *:target {}
*:lang(fr) {}
li:note(.moreinfo) {}

          # 伪元素选择符

          伪元素选择符：为了实现特定的效果，它在文档中插入虚构的元素。所有伪元素只能出现在选择符的最后，这也表明一个选择符中只能有一个伪元素

          选择符 说明
          ::first-letter 装饰元素的首字母
          ::first-line 装饰元素的首行
          ::before 创建和装饰“前置”内容
          ::after 创建和装饰“后置”内容
          h2::before { content: "]]"; color: silver; }
body::after { content: "The End"; }
