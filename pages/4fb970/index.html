# 目录与路径

# 绝对路径与相对路径

路径包括绝对路径（absolute）和相对路径（relative）

  • 绝对路径：由根路径(/)开始写起的文件名或路径名称；例如：/home/sbgong
  • 相对路径：相对于当前路径的文件名写法；例如：./home/sbgong 或 ../../home/sbgong。反正开头不是/就是相对路径

    cd 切换目录（change directory）

    # 执行文件路径的变量：$PATH

    指令 ls的完整文件名为 /bin/ls（这是绝对路径）『为什么用户可以在任何地方执行/bin/ls 这个指令呢？』这都是因为环境变量 PATH 的帮助。当我们在执行一个指令的时候，系统会依照 PATH 的设定去每个 PATH 定义的目录下搜寻文件名为 ls的可执行文件，如果在 PATH 定义的目录中含有多个文件名为 ls 的可执行文件，那么先搜寻到的同名指令先被执行

    PATH（一定要大写）这个变量的内容是由一堆目录所组成的，每个目录中间用冒号（:）来隔开，每个目录是有顺序之分的

    echo $PATH            # 也可使用 echo ${PATH}
PATH="${PATH}:/root"  # 修改 PATH：将 /root 目录添加到 PATH 当中

    # 查看文件或文件夹：ls

    ls 是『list』的意思，重点在显示文件的文件名与相关属性（默认显式出非隐藏的文件，并以文件名进行排序）

    ls 指令基本语法：ls [选项] 文件名与目录名

    选项参数 说明
    -a 全部的文件，连同隐藏文件一起列出来（--all）
    -A 全部的文件，连同隐藏文件，但不包括.和..这两个目录（--almost-all）
    -c 按修改时间排序，最新的在最前
    -d 仅列出目录本身，而不是列出目录内的文件数据（--directory）
    -f 直接列出结果，而不进行排序（ls 默认会以文件名排序）
    -F 根据文件、目录等信息，给予附加数据结构（--classify）
    * 代表可执行文件
    / 代表目录
    = 代表 socket 文件
    l 代表 FIFO 文件）
    -G 不显示群组名称（--no-group）
    -h 将文件内容以人类易读的方式列出来（例如 KB、MB、GB 等）（--human-readable）
    -i 列出文件和目录的 inode 号码
    -l 使用详细格式列表，包含文件的属性与权限等数据；ls -l 等于 ll
    -L 如遇到性质为软链接的文件或目录，直接列出该链接所指向的原始文件或目录（--dereference）
    -m 用逗号『,』分隔每个文件和目录的名称
    -n 列出 UID 与 GID，而非使用者与群组的名称
    -r 反向排序；例如：原来文件名由小到大，反向则为由大到小（--reverse）
    -R 递归处理；将指定目录下的所有文件及子目录一并处理（--recursive）
    -S 以文件或目录的大小排序，而不是用文件名排序
    -t 用文件和目录的更改时间排序
    -X 以文件和目录的最后一个扩展名排序
    --color=never 不要依据文件特性给予颜色显示
    --color=auto 让系统自行根据设定来判断是否给予颜色
    --color=always 显示颜色
    --full-time 以完整时间模式输出（包含年、月、日、时、分）
    --time={atime,ctime} 输出 ctime 或 atime，而非 mtime

    目录颜色表示

    • 蓝色 → 目录
    • 绿色 → 可执行文件
    • 红色 → 压缩文件
    • 青色 → 链接文件
    • 灰色 → 其他文件

    # 复制文件或目录：cp

      指令语法：cp [选项] 源文件 目标文件 或 cp [选项] 源文件1 源文件2…… 目标目录

      选项参数 说明
      -a 相当于 -dr --preserve=all的意思
      -d 若源文件为链接文件，则复制链接文件属性而非文件本身
      -f 为强制的意思，若目标文件已经存在且无法开启，则移除后再尝试一次
      -i 若目标文件已经存在时，在询问是否要覆盖
      -p 连同文件的属性一起复制过去，而非使用默认属性（备份常用）
      -r 递归持续复制，用于目录的复制行为
      -s 对源文件建立软链接，而非复制文件
      -l 对源文件建立硬连接，而非复制文件
      -u 只会在目标文件比源文件旧才更新复制源文件（或目标文件不存在时）
      --preserve=all 除了-p 的权限相关参数外，还加入 SELinux 的属性，links、xattr 等也复制了
      # 案例一：将/var/log/wtmp复制到/tmp中且观察属性
cd /tmp                     # 切换到tmp目录
cp /var/log/wtmp .          # 复制/var/log/wtmp文件到当前目录下
ls -l /var/log/wtmp wtmp    # 查看两个文件的属性信息（文件的更新时间和文件所属人、用户组不同）
cp -a /var/log/wtmp wtmp_2  # 将文件的所有特性都一起复制过来

# 案例二：复制/etc/这个目录下的所有内容到/tmp底下
cp /etc/ /tmp               # 失败：如果是目录则不能直接复制，需要加上 -r 选项
cp -r /etc/ /tmp            # 成功
cp -a /etc/ /tmp            # 在复制目录的时候，文件和目录的权限可能会被改变，此时可以使用 -a 来做备份

# 案例三：建立一个软链接
cp -s bashrc bashrc_slink   # 建立软链接
cp -l bashrc bashrc_hlink   # 建立硬链接

# 案例四：将多个文件复制到指定目录下
cp ~/.bashrc ~/.bash_history /tmp # 将多个文件一次复制到同一个目录中去，注意一定要是目录

      # 删除文件或目录：rm

        remove 指令语法：rm [选项] 文件或目录

        注意：在 Linux 系统下，为了怕文件被 root 误杀，所以很多 distributions 都已经默认加入 -i 这个选项了

        选项参数 说明
        -f 就是强制 force 的意思，忽略不存在的文件，不会出现警告讯息
        -i 互动模式，会询问使用者是否删除该文件
        -r 递归删除，最常用在目录的删除（这是非常危险的选项）
        rmdir /tmp/etc    # 删不掉，因为这不是空目录
rm -r /tmp/etc    # 递归删除该目录（默认使用了 -i 选项，这是一种保护动作）
\rm -r /tmp/etc   # 如果不想以保护模式递归删除非空目录的话，可在指令前加上反斜杠，忽略掉alias的指定选项
rm ./-aaa-        # 如果文件名是以 - 开头的话，为避免系统误判(-是选项嘛)，可以使用这种形式进行删除

          # 文件内容查看

          # 直接查看文件内容：cat/tac/nl

            指令语法： cat [选项] 文件名

            选项参数 说明
            -A 相当于 -vET 的整合选项，可列出一些特殊字符而不是空白而已
            -b 列出行号，仅针对非空白行做出行号，空白行不标行号
            -E 将结尾的断行字符 $ 显式出来
            -n 打印出行号，连同空白行也会有行号
            -T 将『tab』按键以『^I』显式出来
            -v 列出一些看不出来的特殊字符

            主要功能是将一个文件的内容连续的打印在屏幕上面（从第一行到最后一行连续显示在屏幕上）
            cat 是 Concatenate（连续）的简写

            指令语法：tac [选项] 文件名

            tac：由最后一行到第一行反向在屏幕上显式出来，tac 是 cat 倒着写的

            指令语法：nl [选项] 文件名

            选项参数 说明
            -b a 表示不论是否为空行，都要列出行号（类似 cat -n）
            -b t 如果有空行，空的那一行不要列出行号（默认值）
            -n ln 行号在屏幕的最左方显式
            -n rn 行号在自己字段的最右方显示，且不加 0
            -n rz 行号在自己字段的最右方显式，且加 0
            -w 行号字段的占用字符数

            nl 可以将文件内容自动加上行号，且比 cat 能做更多的显示设计，包括位数与是否自动补齐 0 等功能

            nl /etc/issue                 # 默认空行不显示行号
nl -b a /etc/issue            # 不论空行，都要列出行号
nl -b a -n rz /etc/issue      # 在自己字段的地方补上0
nl -b a -n rz -w 3 /etc/issue # 默认字段是6位数，这是想改为3位数的方式

            # 翻页查看文件内容：more/less

              more：一页一页的显示文件内容

              指令语法：more 文件名 在 more 这个程序的运作过程中，你有几个按键可以按的

              按键 说明
              空格键 向下翻一页
              Enter 向下翻一行
              /字符串 向下搜寻『字符串』这个关键词
              :f 立刻显示出文件名以及目前显示的行数
              q 退出 more
              b或ctrl+b 代表往回翻页，不过这个动作只对文件有用，对管线无用

              less 的用法比 more 更丰富，在 more 的时候，没办法向前面翻页，只能往后翻页。但是 less 就可以使用『pageup』『pagedown』等按键功能来往前往后翻看文件；也可以在 less 里头向下搜寻字符串、向上搜寻字符串

              指令语法：less 文件名

              按键 说明
              空格键 向下翻一页
              pagedown 向下翻一页
              pageup 向上翻一页
              /字符串 向下搜寻『字符串』
              ?字符串 向上搜寻『字符串』
              n 重复前一个搜寻（与/或?有关）
              N 反向的重复前一个搜寻（与/或?有关）
              g 前进到这个文件的第一行去
              G 前进到这个文件的最后一行去
              q 退出 less

                head /etc/man_db.conf         # 默认显式前面10行
head -n 20 /etc/man_db.conf   # 显式文件的前20行

                # 读取非文本文件内容：od

                od 读取非文本文件内容：以二进制的方式读取文件内容

                指令语法：od [-t TYPE] 文件名

                TYPE 的取值 说明
                a 利用默认的字符来输出
                c 使用 ASCII 字符来输出
                d[size] 利用十进制来输出数据，每个整数占用 size bytes
                f[size] 利用浮点数来输出数据，每个整数占用 size bytes
                o[size] 利用八进制来输出数据，每个整数占用 size bytes
                x[size] 利用十六进制来输出数据，每个整数占用 size bytes
                od -t c /usr/bin/passwd   # 将文件的内容以ASCII方式来展现
od -t oCc /etc/issue      # 将文件的内容以8进制数列出储存值与ASCII的对照表

                # 文件的时间状态

                一个文件有三个主要的变动时间

                • mtime：当文件的『内容数据』发生变化的时候，就会更新这个时间。内容数据指的是文件的内容，而不是文件的属性或者权限（modification time）
                • atime：当『文件的内容被取用』时，就会更新这个时间。例如用cat去读取文件，就会更新文件的 atime（access time）
                • ctime：当文件的『状态』发生变化的时候，就会更新这个时间。状态指的是权限和属性被更改了（status time）

                # 修改文件或目录的时间/创建文件：touch

                指令语法：touch [选项] 文件名

                选项参数 说明
                a 只修改文件的访问时间（atime）
                c 只修改文件的 ctime，若该文件不存在则不建立新文件
                m 只修改 mtime
                d 使用指定字符串表示时间，从而代替当前时间（修改了 mtime 和 atime，ctime 没有改变）
                t 使用格式为[YYYYMMDDhhmm]的字符串代替当前时间（修改了 mtime 和 atime，ctime 没有改变）
                touch filename  # 默认该文件的三个时间都会更新为目前的时间，若该文件不存在，则会主动创建一个新的空文件
touch -d "2 days ago" bashrc    # 将文件日期调整为2天前
touch -t 202204280238 bashrc    # 将文件日期改为2022/04/28 02:38

                # 文件与目录的默认权限与隐藏权限

                # 文件默认权限：umask

                  文件最终权限 = 文件默认权限 - umask 设置的数字分数权限

                  目录最终权限 = 目录默认权限 - umask 设置的数字分数权限

                  • 文件默认权限（-rw-rw-rw-）：666
                  • 目录默认权限（drwxrwxrwx）：777
                  • umask：以数字形态的权限设定分数
                    • 第一种查看出来会显示 4 个数字，第 1 个数字先不用管，默认权限与后面 3 位数有关
                    • 后三位分别代表：拥有者、用户组、其他人的权限（r：4，w：2，x：1）
                  • umask -S：以符号类型的方式显示权限

                  修改 umask 数值直接在指令后面添加数字即可，例如：umask 002

                  一般情况下：root 用户的 umask 是 022，普通用户的 umask 是 002

                  # 文件隐藏属性：chattr/lsattr

                  注意

                  chattr和lsattr指令只在 ext2/ext3/ext4 的 linux 文件系统上面完整生效，xfs 文件系统仅支持部分参数

                    指令语法： chattr [+-=][参数] 文件名或目录名

                    选项参数 说明
                    + 增加某一个特殊参数，其他原本存在的参数则不动
                    - 移除某一个特殊参数，其他原本存在的参数则不动
                    = 有且仅有后面接的参数
                    a 文件将只能增加数据，而不能删除也不能修改数据，只有 root 才能设定这个属性
                    i 它可以让一个文件不能被删除、改名、无法写入或新增数据，只有 root 才能设定这个属性
                    A 若你有存取此文件或目录时，它的访问时间 atime 将不会被修改
                    S 一般文件是异步写入磁盘的，如果加上这个属性，当你进行任何文件的修改，数据更改会同步写入磁盘
                    c 会自动将文件进行压缩，在读取文件的时候自动进行解压缩，在保存的时候会先进行压缩后再保存（对于大文件很有用）
                    d 当 dump 程序被执行的时候，设置该属性可将文件/目录不会被 dump 备份
                    s 如果设定了改属性，假设该文件被删除，它将会被完全移除出这个硬盘空间；所以如果误删了，则无法补救
                    u 与 s 相反，假设该文件被删除了，数据内容其实还存在磁盘中；如果是误删，可以补救

                    指令语法： lsattr [选项] 文件名或目录名

                    选项参数 说明
                    -a 将隐藏文件的属性也显示出来
                    -d 如果接的是目录，仅列出目录本身的属性而非目录内的文件名
                    -R 连同子目录的数据也一并列出来
                    touch /tmp/attrtest # 创建一个空文件
chattr +i attrtest  # 赋予 i 的属性
rm attrtest         # 尝试删除时会提示删不掉
chattr -i attrtest  # 取消该文件的 i 属性

                    # 文件特殊权限：SUID/SGID/SBIT

                      SUID：当 s 这个标志出现在文件拥有者的 x 权限上时，此时就被称为 Set UID，简称 SUID 的特殊权限

                      SUID 对一个文件的特殊功能

                      • SUID 权限仅对二进制程序有效
                      • 执行者对于该程序需要具有 x 的可执行权限
                      • 本权限仅在执行该程序的过程中有效
                      • 执行者将具有该程序拥有者的权限

                      SGID：当 s 这个标志出现在群组的 x 权限上时，此时就被称为 Set GID，简称 SGID 的特殊权限

                      SGID 可以对文件或目录做设置

                      • 对文件来说，SGID 有如下的功能：
                        • SGID 对二进制程序有效
                        • 执行者对于该程序需要具有 x 的可执行权限
                        • 执行者在执行的过程中将会获得该程序群组的权限
                      • 对文件来说，SGID 有如下的功能：
                        • 用户若对于此目录具有 r 与 x 的权限时，该用户能够进入此目录
                        • 用户在此目录下的有效群组将会变成该目录的群组
                        • 用途：若用户在此目录下具有 w 的权限（可以新建文件），则使用者所建立的新文件，该文件的群组与此目录的群组相同

                      SBIT：目前只针对目录有效，对于文件已经没有效果了（Sticky Bit）

                      SBIT 对目录的作用

                      • 当用户对于此目录具有 w 和 x 权限，亦即具有写入权限时
                      • 当用户在该目录下建立文件或目录时，仅有自己与 root 才有权利删除改文件

                      # 文件名的搜索：whereis/locate/find

                      一般文件名的搜索是先使用 whereis 或 locate 来检查，如果真的找不到了才使用 find 来搜索（因为 find 效率不高）

                        whereis：只在一些特定的目录中寻找文件文件名，并没有全系统去查询，所以执行效率高（需要完整文件名）

                        指令语法： whereis [选项] 文件或目录名

                        选项参数 说明
                        -l 可以列出 whereis 会去查询的几个主要目录
                        -b 只找二进制格式的文件
                        -m 只找在说明文件 manual 路径下的文件
                        -s 只找 source 来源文件
                        -u 搜索不在上述三个项目当中的其他特殊文件

                        如果想知道 whereis 到底查了多少目录，可以使用 whereis -l 来确认一下

                        locate 是利用数据库来搜索文件名，没有实际搜索硬盘内的文件系统状态（需要部分文件名即可）。locate 寻找的数据是由已建立的数据库 /var/lib/mlocate 里面的数据所搜索到的，而数据库的建立默认是每天执行一次，所以当你新建立文件，而且在数据库更新之前搜索改文件，那么 locate 会告诉你找不到，因为必须更新数据库。手动更新 locate 数据库的方法非常贱，直接输入 updatedb 即可，该指令会去读取 /etc/updatedb.conf 这个配置文件的设置，然后再去硬盘里面进行搜索文件名的动作，最后就更新整个数据库文件。因为 updatedb 会去搜索硬盘，所以当你执行 updatedb 时，可能会等待数分钟的时间

                        指令语法： locate [选项] 关键词

                        选项参数 说明
                        -i 忽略大小写的差异
                        -c 不输出文件名，仅计算找到的文件数量
                        -l 仅输出几行的意思，例如输出五行则是 -l 5
                        -S 输出 locate 所使用的的数据库文件的相关信息，包括该数据库记录的文件/目录数量等
                        -r 后面可接正规表示法的显示方式
                        locate -l 5 passwd  # 找出系统中所有与passwd相关的文件名，且只列出5个
locate -S           # 列出locate查询所用的数据库文件信息——文件名与各数据数量

                        find：如果你要找的文件具有特殊属性，例如 SUID、文件拥有者、文件大小等，那么可以利用 find 进行查找。但 find 在查找数据的时候很费硬盘，所以没事情不要使用 find

                        指令语法： find [路径] [option] [action]

                        选项参数 说明
                        -mtime n n 为数字，意义为在 n 天之前的『一天之内』被更动过内容的文件
                        -mtime +n 列出在 n 天之前（不含 n 天本身）被更动过内容的文件名
                        -mtime -n 列出在 n 天之内（含 n 天本身）被更动过内容的文件名
                        -newer file file 为一个存在的文件，列出比 file 还要新的文件名
                        -uid n n 为数字，这个数字是用户的账号 ID（UID），这个 UID 记录在/etc/passwd
                        -gid n n 为数字，这个数字是组名的 ID（GID），这个 GID 记录在/etc/group
                        -user name name 为使用者账号名称
                        -group name name 为组名
                        -nouser 寻找文件的拥有者不存在于/etc/passwd 的人
                        -nogroup 寻找文件的拥有群组不存在于/etc/group 的文件
                        -name filename 搜索文件名为 filename 的文件
                        -size [+-]SIZE 搜索比 SIZE 还要大(+)或小(-)的文件(SIZE 单位:c 代表 byte,k 代表 1024bytes)
                        -type TYPE 搜索文件的类型为 TYPE(类型:f 为一般文件,b/c 为装置文件,d 为目录,l 为软链接,s 为 socket,p 为管道文件)
                        -perm mode 搜索文件权限刚好等于 mode 的文件;mode 类似于 chmod 的属性值
                        -perm -mode 搜索文件权限必须要全部囊括 mode 的权限的文件
                        -perm /mode 搜索文件权限包含任一 mode 的权限
                        -exec command command 为其他指令，-exec 后面可再接额外的指令来处理搜索到的结果
                        -print 将结果打印到屏幕上，这个动作是默认动作
                        # 与时间有关的选项案例
find / -mtime 0               # 将过去系统上面24小时内有更动过内容的文件列出
find /etc -newer /etc/passwd  # 寻找/etc目录下的文件，如果文件日期比/etc/passwd新就列出
# 与使用者或组名有关的选项案例
find /home -user sbgong       # 在/home文件夹下搜索属于sbgong的文件
find / -nouser                # 搜索系统中不属于任何人的文件
# 与文件权限及名称有关的选项案例
find / -name passwd           # 找出文件名为passwd这个文件
find / -name "*passwd"        # 找出文件名包含了passwd这个关键词的文件
find /run -type s             # 找到/run目录下，文件类型为socket的文件名
find / -perm /7000            # 找到文件当中含有SGID或SUID或SBIT的属性
find / -size +1M              # 找出系统中大于1MB的文件
# 额外可进行的动作
find /usr/bin /usr/bin -perm /7000 -exec ls -l {} \;  # 注意这里不支持命令别名，所以此处不可使用ll
# {}代表由find找到的内容会被放置到{}位置中；(-exec)是开始的标志，(\;)是结束的标志
