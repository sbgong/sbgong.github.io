文本是网页的重要元素，因此 CSS 提供了很多相关的属性，以不同的方式控制文本。那么，文本和字体之间有什么区别呢？简单而言，文本就是内容，而字体是用于显示内容的。借助文本属性，可以影响文本相对一行中其他内容的位置，可以把文本设为上标，可以为文本添加下划线，可以改变文本的大小写，甚至还能在一定程度上模拟打字机的 Tab 键

在此之前要先明确『行内』和『块级』两个术语，因为在本章将多次用到

- 块级方向：指当前书写模式放置块级元素的方向。比如说，在英语中，块级方向从上到下，纵向的
- 行级方向：指块级元素中行内元素的书写方向。比如说，在英语中，行内方向是从左至右，横向的

## 缩进文本：text-indent

使用西方语言撰写的书，多数在排版时会缩进一段的第一行，而且段落之间没有空行

CSS 为缩进文本提供了 `text-indent` 属性

| 功能   | 说明                       |
| ------ | -------------------------- |
| 取值   | `<length> 或 <percentage>` |
| 初始值 | 0                          |
| 适用于 | 块级元素                   |
| 百分数 | 相对于所在块级元素的宽度   |
| 继承性 | 是                         |
| 动画性 | 是                         |

- text-indent 属性把元素的第一行文本缩进指定特定的长度，缩进的长度可以是负值。这个属性通常用于缩进段落的第一行。使用负值时要小心，以防前几个词被浏览器窗口的边缘吃掉，为避免出现这种显示问题，建议加上外边距或内边距
- text-indent 属性可以用在任何块级元素上，缩进将沿着行内方向展开；text-indent 属性不能用于行内元素或置换元素（如图像）
- text-indent 属性的值可以使用任何长度单位，包括百分数
- text-indent 属性会继承，因此可能会出现意料之外的效果
- 文本缩进只影响元素的第一行，即便有换行也是如此

```
# 把段落的第一行缩进3em
p { text-indent: 3em; padding-left: 4em; }
```

## 文本对齐：text-align

`text-align` 属性控制元素中各文本行的对齐方式（不控制元素的对齐方式，只影响元素中的内容）

| 功能   | 说明                                                                       |
| ------ | -------------------------------------------------------------------------- |
| 取值   | `start / end / left / right / center / justify / match-parent / start end` |
| 初始值 | 在 CSS3 中是 start，在 CSS2.1 中由用户代理指定，有可能根据书写方向而定     |
| 适用于 | 块级元素                                                                   |
| 继承性 | 是                                                                         |
| 动画性 | 否                                                                         |

- start：意思是文本与元素所在行框的起边对齐。在从左至右书写的语言中，起边是左边；在从右至左书写的语言中，起边是右边；在竖写语言中，根据具体的书写方向，可能是顶边或底边。这样规定的最终效果是，默认值支持更多语言书写方向，而在目前已知的情况中，大多数时候默认的行为都是一致的
- end：把文本与边框的终边对齐。在从左至右书写的语言中，终边是右边；在从右至左书写的语言中，终边是左边；以此类推
- justify： 两端对齐文本。一行的两端都与父元素的边界对齐，单词和字母之间的空白会做调整，从而保证每一行的长度完全一致。印刷品中经常使用两端对齐。两端对齐的文本要经过拉伸，正好占满父元素左右边界之间的空间。至于如何拉伸，是由用户代理决定的，而跟 CSS 没有关系（至少截至 2017 年年末是这样的）
- match-parent：浏览器不支持这个值，它的作用基本上被`inherit` 涵盖了。它的作用是让元素的对齐方式与父元素保持一致。不过二者是有区别的：如果父元素的对齐方式设为 start 或 end， match-parent 将计算得到的 left 或 right 赋予元素； inherit 则不然，应用到元素上的就是 start 或 end
- start end：截止 2017 年末，start end 尚未实现

## 对齐最后一行：text-align-last

元素的最后一行与其他内容的对齐方式不同，具体如何对齐由 `text-align-last` 属性决定

- 其实只要一行后面有强制换行，不管是不是在元素的末尾，都受 text-align-last 属性的控制；当然块级元素的最后一行也受这个属性控制
- 如果元素的第一行也是最后一行，text-align-last 的优先级比 text-align 高

| 功能   | 说明                                                   |
| ------ | ------------------------------------------------------ |
| 取值   | `auto / start / end / left / right / center / justify` |
| 初始值 | auto                                                   |
| 适用于 | 块级元素                                               |
| 继承性 | 是                                                     |
| 动画性 | 否                                                     |

**注意**：截至 2017 年年末，Safari 和 Opera Mini 不支持 text-align-last，IE 和 Edge 只支持 left、right 和 center 三个值

## 行的高度：line-height

行之间的距离受行的『高度』影响，这里所说的高度是相对文本行而言的

`line-height` 属性指行的基线之间的距离，与字号无关，决定着元素所在方框的高度是增还是减。line-height 控制的是行距，是除字体高度之外在文本行上方的额外空间。也就是说，line-height 的值与字体高度之差就是**行距**

| 功能   | 说明                                          |
| ------ | --------------------------------------------- |
| 取值   | `<number> / <length> / <percentage> / normal` |
| 初始值 | normal                                        |
| 适用于 | 所有元素                                      |
| 百分数 | 相对于元素的字号                              |
| 继承性 | 是                                            |
| 动画性 | 是                                            |

- normal：行之间的空间由用户代理计算，不同的用户代理计算出来的值可能不同，不过一般都是字体高度的 1.2 倍左右
- length：行高 line-height 的值可以是长度值（如 18px 或 2em），不过多数情况下首选纯数字，此数字会与当前的字体尺寸相乘来设置行间距
- percentage：基于当前字体尺寸的百分比行间距

## 行的构成

![行](https://cdn.jsdelivr.net/gh/sbgong/static/blog/frontend/css/202207121054644.webp)

文本行中的每个元素构成一个内容区（content area），其高度由字体的高度决定。随内容区出现的是一个行内框（inline box），如果不考虑其他因素，其高度和内容区完全相等，line-height 导致的行距是影响行内框高度的因素之一

元素的行距（leading）等于 font-size 的计算结果减去 line-height 的计算结果，这个值是行距的总值。注意，行距可能为负数。行距分为两半，分别放到内容区的上部和下部。算上行距，得到的就是元素的行内框。确定一行内容的全部行内框之后，行框（line box）也就确定了。行框恰好包围最高那个行内框的顶端和最低那个行内框的底端

- 一行文字被放在一个 line box 里，而 line box 又由一个或多个 inline box(行内框)组成
- inline box 中的文本内容组成了 content area(虚线框所示)，content area 的大小跟文字大小有关，而决定文字大小的是 font-size，因此 font-size 作用于 content area。在未设置 line-height 的情况下，inline box 的高度由 content area 决定
- 而 line-height 则是作用于 inline box 高度，line-height 为 20px;font-size 为 16px;的情况下，会在 content area 的上下设定 2px 的间隔。这样上下两行基线间距离也就增加了 4px
- line box 的高度由 inline boxs 中最高的那一个决定
- height 设定的是容器的高度，并非 line box 高度，如果容器高度正好与 line box 高度相等，那么行文本就在容器内垂直居中了

![行的构成](https://cdn.jsdelivr.net/gh/sbgong/static/blog/frontend/css/202207121058717.webp)

举个例子，假设 font-size 是 14 像素（因此内容区也就是这么高），line-height 的计算结果为 18 像素。二者之差（4 像素）分成两半，分别放到内容区的上部和下部。因此，行内框的高度为 18 像素

## 垂直对齐文本：vertical-align

`vertical-align` 定义行内元素在行框内的垂直对齐方式；CSS 中的 vertical-align 属性不继承，它只能用于行内元素和置换元素（例如图像和表单输入框）

| 功能   | 说明                                                                                           |
| ------ | ---------------------------------------------------------------------------------------------- |
| 取值   | `baseline / sub / super / top / text-top / middle / bottom / text-bottom / length /percentage` |
| 初始值 | baseline                                                                                       |
| 适用于 | 行内元素和单元格                                                                               |
| 百分数 | 相对元素的 line-height 值                                                                      |
| 继承性 | 否                                                                                             |
| 动画性 | `length`，`<percentage>`                                                                       |

| 属性值      | 说明                                                                                                                                                                    |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| baseline    | 基线对齐；强制元素的基线与父元素基线对齐；如果目标元素没有基线，例如图像、表单输入框或其他置换元素，元素的底端与父元素的基线对齐                                        |
| sub         | 把元素放在下标处，即元素的基线（对置换元素来说是底边线）低于父元素的基线。低处多少距离由用户代理决定。注意 sub 不改变元素的字号，因此下标中的文本不会变小或变大         |
| super       | 让元素的基线（或置换元素的底边线）高于父元素的基线。同样高出多少距离由用户代理决定。注意 super 不改变元素的字号，因此上标中的文本不会变小或变大                         |
| bottom      | 把元素所在行内框的底边与行框的底边对齐                                                                                                                                  |
| text-bottom | 元素行内框的底边与父元素内容区的底边对齐                                                                                                                                |
| top         | 把元素所在行内框的顶边与行框的顶边对齐                                                                                                                                  |
| text-top    | 元素行内框的顶边与父元素内容区的顶边对齐                                                                                                                                |
| middle      | 中线对齐；通常用在图像上，把行内元素所在方框的中线与父元素基线向上偏移 0.5ex 处的线对齐（1ex 等于父元素的 font-size）                                                   |
| length      | 长度值；把元素抬升或下沉声明的距离                                                                                                                                      |
| percentage  | 百分数；把元素的基线（或置换元素的底边）相对父元素的基线抬升或下沉指定的量（指定的百分数相对元素自身而非父元素的 line-height 计算）；正百分数抬升元素，负百分数下沉元素 |
| inherit     | 继承父元素的相应属性值                                                                                                                                                  |

## 单词间距：word-spacing

`word-spacing` 属性的值为长度，可正可负。指定的长度值追加到单词的标准间距上。其实，word-spacing 属性用于修改单词之间的距离。因此，默认值 normal 等同于把值设为零

- 设为正长度值时，单词之间的距离增加；设为负值时，单词之间靠得更近
- 该属性可以将指定的额外间隙添加到每个单词之后，最后一个单词不添加

| 功能   | 说明                |
| ------ | ------------------- |
| 取值   | `<length> / normal` |
| 初始值 | normal              |
| 适用于 | 所有元素            |
| 继承性 | 是                  |
| 动画性 | 是                  |

## 字符间距：letter-spacing

`letter-spacing` 属性修改的是字符或字母之间的距离。该属性值也支持取任何长度值，默认值 normal 等同于 0

| 功能   | 说明                |
| ------ | ------------------- |
| 取值   | `<length> / normal` |
| 初始值 | normal              |
| 适用于 | 所有元素            |
| 继承性 | 是                  |
| 动画性 | 是                  |

**注意**：如果使用的字体带连字特性，而且启用了，调整字符间距或单词间距可能导致这个特性失效。比如说，调整字符间距后，浏览器不会重新计算连字或其他连接

## 文本转换：text-transform

`text-transform` 属性转变文本的大小写

| 功能   | 说明                                        |
| ------ | ------------------------------------------- |
| 取值   | `uppercase / lowercase / capitalize / none` |
| 初始值 | none                                        |
| 适用于 | 所有元素                                    |
| 继承性 | 是                                          |
| 动画性 | 否                                          |

- uppercase：把文本转换成大写
- lowercase：把文本转换成小写
- capitalize：只把各单词的首字母变成大写

## 文本装饰：text-decoration

| 功能   | 说明                                         |
| ------ | -------------------------------------------- |
| 取值   | `none / underline / overline / line-through` |
| 初始值 | none                                         |
| 适用于 | 所有元素                                     |
| 继承性 | 否                                           |
| 动画性 | 否                                           |

- underline：为元素添加下划线
- overline：在文本上方绘制一条线
- line-through：绘制一条贯穿文本中部的线
- none：把应用到元素上的装饰效果去掉

## 文本渲染效果：text-rendering

`text-rendering` 是 CSS 新增的属性，它其实是一个 SVG 属性，支持它的用户代理不会把它视作 CSS 属性。这个属性的作用是让用户代理在显示文本时应该优先考虑什么方面

| 功能   | 说明                                                             |
| ------ | ---------------------------------------------------------------- |
| 取值   | `auto / optimizeSpeed / optimizeLegibility / geometricPrecision` |
| 初始值 | auto                                                             |
| 适用于 | 所有元素                                                         |
| 继承性 | 是                                                               |
| 动画性 | 是                                                               |

- optimizeSpeed 和 optimizeLegibility 它们指明首先考虑绘制速度，而不是清晰性特性。如紧排和连字（optimizeSpeed），或者反过来（optimizeLegibility）
- optimizeLegibility 针对的清晰性特性没有明确定义，而且文本渲染效果通常取决于运行用户代理的操作系统，因此具体效果视情况而定。客观来说，这两种优化方式之间的区别很小，不过对可读性可能会产生重大影响
- geometricPrecision 的作用不太明显，它其实是让用户代理尽量精准地绘制文本，确保能无损缩放。别以为情况始终是这样，现实并非如此。比如说，有些字体会随着字号的变化而改变紧排或连字效果，在字号小时提供较大的字距，字号变大之后又紧缩字距。使用 geometricPrecision 可以让用户代理忽略随字号增减而出现的这些变化。如果不理解的话，可以这样想：geometricPrecision 的作用相当于让用户代理把文本绘制成一系列 SVG 路径，而不是字形
- auto：用户代理自行决定合适的方法，只要向清晰性倾斜

## 文本阴影：text-shadow

| 功能   | 说明                                    |
| ------ | --------------------------------------- |
| 取值   | `none / length // length length length` |
| 初始值 | none                                    |
| 适用于 | 所有元素                                |
| 继承性 | 否                                      |
| 动画性 | 是                                      |

- 默认文本没有阴影。如果想，可以为文本定义一个或多个阴影。每个阴影由一个可选的颜色和三个长度值定义，最后一个长度值也是可选的
- 颜色设定的是阴影的颜色，因此可以定义绿色、紫色，甚至是白色的阴影。如果省略颜色，阴影的颜色将与文本的颜色相同
- 前两个长度值是阴影离文本的距离。第一个长度值设定横向偏移，第二个长度值设定纵向偏移。负的长度值把阴影向文本的左侧和上部偏移
- 可选的第三个长度值定义阴影的模糊半径,不允许负值。具体的模糊方法没有规定，因此不同的用户代理可能会采取不同的模糊效果

注意：大量阴影或模糊半径较大的文本阴影会损耗性能，尤其是在低功耗和 CPU 能力有限的情况下，例如移动设备。建议创作人员进行充分的测试之后再投放使用阴影的设计

## 处理空白：white-space

`white-space` 属性，它影响用户代理对文档源码中空格、换行符和制表符的处理方法。这个属性影响浏览器对待单词之间及文本行之间空白的方式。XHTML 在某种程度上已经做了处理，即把空白压缩成一个空格。CSS2.1 引入 pre-wrap 和 pre-line 两个值

| 功能   | 说明                                          |
| ------ | --------------------------------------------- |
| 取值   | `normal / pre / nowrap / pre-wrap / pre-line` |
| 初始值 | normal                                        |
| 适用于 | 全部元素（CSS2.1）；块级元素（CSS1 和 CSS2）  |
| 继承性 | 否                                            |
| 动画性 | 否                                            |

- normal：默认处理方式。让浏览器采用当前的方式处理空白，即丢掉多余的空白。换行符（回车）变成空白，而且连续的多个空格变成一个空格
- pre：原封不动的保留你输入时的状态，空格、换行都会保留，并且当文字超出边界时不换行。等同 pre 元素效果
- nowrap：禁止元素中的文本換行，除非使用的是 br 元素
- pre-wrap：文本中的空白序列得以保留，但是文本行将正常换行
- pre-line：空白序列将像常规文本中那样折叠，但是保留换行

|    值    | 空白 | 换行符 | 自动换行 |
| :------: | :--: | :----: | :------: |
|  normal  | 折叠 |  忽略  |   禁止   |
|   pre    | 保留 |  保留  |   禁止   |
|  nowrap  | 折叠 |  忽略  |   允许   |
| pre-wrap | 保留 |  保留  |   允许   |
| pre-line | 折叠 |  保留  |   允许   |

## 设置制表符的宽度：tab-size

既然 white-space 取某些值时空白会保留下来，那么制表符（即 Unicode 码位 0009）也就将显示成制表符了。但是一个制表符等于多少个空格呢？这就要问 `tab-size` 属性了

| 功能   | 说明                   |
| ------ | ---------------------- |
| 取值   | `<length> / <integer>` |
| 初始值 | 8                      |
| 适用于 | 块级元素               |
| 继承性 | 是                     |
| 动画性 | 是                     |

- 默认情况下，一个制表符相当于八个连续的空格。不过，可以使用 tab-size 属性设为其他整数值。例如：`tab-size:4;` 将把一个制表符渲染成四个连续空格
- 如果提供的是长度值，那么制表符将渲染成指定的长度。例如：`tab-size: 10px;` 将把三个连续的制表符渲染成 30 像素宽的空白

**注意**：目前 Webkit 和 Geeko 支持 tab-size 属性。不过，只支持整数值，不支持长度值

## 断字：hyphens

| 功能   | 说明                   |
| ------ | ---------------------- |
| 取值   | `manual / auto / none` |
| 初始值 | manual                 |
| 适用于 | 所有元素               |
| 继承性 | 是                     |
| 动画性 | 否                     |

- manual：默认值，只在文档中手动插入的连字符处断字（即 `U+00AD` 和 `&shy;` ）；否则，不断字
- auto：即使没有手动插入连字符，浏览器也会在单词中“合适的”位置断字，把单词分开。这就带来了一些有趣的问题，比如怎样定义“单词”，以及何为断字的“合适”位置。其实这两个问题完全由所用的语言决定。用户代理应该优先在手动插入的连字符处断字，但情况并非总是如此
- none：即使有手动插入的连字符，也不断字；即 `U+00AD` 和 `&shy;` 将被忽略

因为断字与所用的语言有很大关系，而且 CSS 规范没有准确定义用户代理应该如何断字（连模糊说明都没有），所以在不同的浏览器中断字的方式极有可能不同

## 处理文本软换行：word-break

`word-break` 这个属性的作用是控制不同语言处理文本软换行的方式。如果一串文本太长，一行放不下，就会软换行。这个概念与换行符和 br 元素导致的硬换行是相对的。文本在何处软换行由用户代理（或操作系统）决定，但是创作人员可以使用 word-break 属性影响这一决定

| 功能   | 说明                            |
| ------ | ------------------------------- |
| 取值   | `normal / break-all / keep-all` |
| 初始值 | normal                          |
| 适用于 | 所有元素                        |
| 继承性 | 是                              |
| 动画性 | 是                              |

- normal：默认值；按正常的方式换行，通俗地说，就是文本在单词之间换行。不过不同语言中单词的定义有所不同
- break-all：软换行可能出现在任何字符之间，即使是在一个词的内部，使用这个值的时候，连字符不显示，即使换行出现在断字的位置
- keep-all：禁止在字符之间软换行，这个行为类似于 `white-space: pre;`

## 处理文本软换行：line-break

line-break 可以用来处理如何断开带有标点符号的中日韩三国的文本行

| 功能   | 说明                             |
| ------ | -------------------------------- |
| 取值   | `auto / loose / normal / strict` |
| 初始值 | auto                             |
| 适用于 | 所有元素                         |
| 继承性 | 是                               |
| 动画性 | 是                               |

- auto：默认值，指示用户代理按正常行为软换行，而更为重要的是，让用户代理根据情况变更换行方式。其实，auto 的作用是让用户代理根据需求在 loose、normal 和 strict 之间切换，甚至可以在一个元素的不同行之间切换
- loose：采用『最宽松的』换行规则，主要用于文本行较短的情况，例如报纸
- normal：采用『最常规的』换行规则。什么是最常规的规则，没有准确定义，不过前面说过，规范中有建议的行为
- strict：采用『最严格的』换行规则。同样，何为最严格的，也没有明确定义

## 文本换行：overflow-wrap

了解了断字和软换行之后，我们不禁会想，如果文本超出了所在容器怎么办？这正是 `overflow-wrap` 属性要解决的问题

| 功能   | 说明                  |
| ------ | --------------------- |
| 取值   | `normal / break-word` |
| 初始值 | normal                |
| 适用于 | 所有元素              |
| 继承性 | 是                    |
| 动画性 | 是                    |

- normal：默认值，按正常方式换行，即在单词之间换行，或者由所用的语言决定
- break-word：可以在单词的内部换行

**注意**：只有 white-space 属性的值允许换行时，overflow-wrap 才会起作用。如果 white-space 禁止换行（例如设为 pre），那么 overflow-wrap 没有任何效果

## 设置书写模式：writing-mode

| ------ | ------------------------------------------------------------------------ |
| 取值   | `horizontal-tb / vertical-rl / vertical-lr`                              |
| 初始值 | horizontal-tb                                                            |

## 控制文本方向：direction <Badge text="CSS2：目前不建议使用" type="error" />

它会改变行内基线的方向，**注意**：CSS 规范明确指出，不建议把 CSS 的 direction 和 unicode-bidi 属性用在 HTML 文档上。原话是这样的：“因为 HTML（或用户代理）能够禁用 CSS 样式，所以我们建议使用 HTML 的 dir 属性和 `<bdo>` 元素，确保在缺少样式表的情况下能得到正确的双向布局”。在这里介绍这两个属性是因为在旧样式表中可能还有它们的身影

`direction` 属性影响块级元素中文本的书写方向，表格列布局的方向，内容在横向上溢出元素框的方向，以及两端对齐的元素中最后一行的位置。对行内元素来说，仅当 unicode-bidi 属性设为 embed 或 bidi-override 时，direction 才起作用

| 功能   | 说明        |
| ------ | ----------- |
| 取值   | `ltr / rtl` |
| 初始值 | ltr         |
| 适用于 | 所有元素    |
| 继承性 | 是          |
| 动画性 | 是          |

- lrt：默认值，文本流从左到右
- rtl：文本流从右到左

## 控制文本方向：unicode-bidi <Badge text="CSS2：目前不建议使用" type="error" />

它会改变行内基线的方向，**注意**：CSS 规范明确指出，不建议把 CSS 的 direction 和 unicode-bidi 属性用在 HTML 文档上。原话是这样的：“因为 HTML（或用户代理）能够禁用 CSS 样式，所以我们建议使用 HTML 的 dir 属性和 `<bdo>` 元素，确保在缺少样式表的情况下能得到正确的双向布局”。在这里介绍这两个属性是因为在旧样式表中可能还有它们的身影

`unicode-bidi` 属性，用于同一个页面里存在从不同方向读进的文本显示，与 `direction` 属性一起使用

| 功能   | 说明                             |
| ------ | -------------------------------- |
| 取值   | `normal / embed / bidi-override` |
| 初始值 | ltr                              |
| 适用于 | 所有元素                         |
| 继承性 | 否                               |
| 动画性 | 是                               |

- normal： 根据双向算法，目标元素不打开新的嵌套层级。对于行内元素，跨元素边界隐式重排
- embed： 对于行内元素，根据双向算法，打开新的嵌套层级。让一级嵌套的方向由 direction 属性指定。元素内部隐式重排，在元素开头添加一个 LRE 字符（U+202A，针对 direction:ltr）或一个 RLE 字符（U+202B；针对 direction:rtl），在元素末尾添加一个 PDF 字符（U+202C）
- bidi-override： 严格按照 <' direction '> 属性的值重排序。忽略隐式双向运算规则
