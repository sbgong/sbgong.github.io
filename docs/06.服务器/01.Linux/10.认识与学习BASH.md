## 认识 BASH 这个 Shell

> shell 是命令解释器，用于解释用户对操作系统的操作

- 操作系统内核（Kernel）能控制计算机各种硬件资源，还能管理内存、控制进程等，非常的重要，但如果操作系统被用户随意操作的话，若使用者应用不当，将会使得整个系统崩溃。所以在用户和操作系统之间加上了一层壳（Shell 壳程序）进行过渡，通过 Shell 过滤掉用户危险和错误的操作，保护操作系统的正常运行
- Shell 是用来提供给用户操作系统的一个接口，因此这个壳程序可以调用其他的软件指令（例如：chmod/fdisk 等）：它接收用户命令，然后调用相应的应用程序
- 基本上 shell 分两大类：`图形界面shell，例如：GNOME` 和 `命令行式shell，例如：BASH`
  - 作为命令语言，它交互式解释和执行用户输入的命令或者自动地解释和执行预先设定好的一连串的命令
  - 作为程序设计语言，它定义了各种变量和参数，并提供了许多在高级语言中才具有的控制结构，包括循环和分支

::: tip shell 种类

- sh：第一个流行的 shell 由 Steven Bourne 发展出来的，为了纪念就称其为 Bourne Shell，简称 sh
- csh：由伯克利大学的 Bill Joy 设计依附于 BSD 版的 UNIX 系统中的 shell，这个 shell 的语法有点类似 C 语言，所以才叫 C Shell，简称 csh
- tcsh：整合了 csh，提供了更多的功能
- bash：当前 Linux 版本的 shell。这个 shell 是 Bourne shell 的增强版本，也是基于 GNU 的架构下的发展出来的

:::

- Linux 系统下的 `/etc/shells` 文件记录了你当前 Linux 系统中所支持的 shell 版本
- 系统上合法的 shell 都要写入到这个文件中：因为当系统某些服务在运行过程中，会去检查使用者能够使用的 shells，而这些 shell 的查询就是由 /etc/shells 这个文件
- `/etc/passwd` 文件：不同的用户登录系统时，系统就会给我们一个默认 shell 来进行工作，那这个不同使用者默认的 shell 分别是什么哪？这些记录都在/etc/passwd 文件中

### Bash Shell 的功能

bash 是 GNU 计划中重要的工具软件之一，也是 Linux distribution 的标准 shell。bash 主要兼容于 sh，并且依据一些使用者的需求而加强的 shell 版本

- **记录历史命令**：只要在命令行按『上下键』就可以找到前/后一个输入的命令，而在很多 distribution 里面默认的指令记忆功能可以达到 1000 个，也就是说，你曾经下达过的指令几乎都被记录下来了
  - `~/.bash_history` 文件：bash 记录了你曾经使用过的命令，不同的用户使用过的历史命令也不同，这些历史命令都记录在自己家目录下的.bash_history 文件中
  - ~/.bash_history 记录的是你上一次登录系统所执行过的命令，而至于这一次所执行过的命令都被缓冲在内存中，当你系统关机/重启后才会更新到~/.bash_history 文件中
- **命令与文件补齐功能**（tab 键）：这个按键的功能就是在 bash 里头才有的
  - [Tab]接在一串指令的第一个字的后面，则为命令补全
  - [Tab]接在一串指令的第二个字以后时，则为文件补齐
  - 若安装 `bash-completion` 软件后，则在某些指令后面使用[Tab]键时，可以进行选项/参数的补齐
- **命令别名设置功能**（alias）：例如想要查看目录下面的所有文件（包括隐藏文件）及所有文件的属性，那就需要下达『ls -al』，别名就是『ll』。直接在输出 `alias` 即可查看目前系统中的别名了
- **程序化脚本**（shell scripts）：从前 DOS 年代需要将一堆命令写在一起来形成批处理文件（编程语言书写），而在 Linux 系统中，可以通过 shell 提供的环境变量以及相关命令来设计 shell 脚本，就能够实现相应的功能
- **任务管理、前台、后台控制**
- **通配符**

### 查看 bash 内置命令：type

bash 中已经内置了很多命令，例如：cd、umask、等等很多，type 命令就可以查看 bash 中有哪些内置的命令

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab 指令语法
**指令语法**：`type [选项] 命令`

|   选项参数   | 说明                                                                                                                                                                   |
| :----------: | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不加任何参数 | type 会显示出『命令』是外部指令还是 bash 内建指令                                                                                                                      |
|     `-t`     | 查询『命令』时只会列出下面的其中一项<br>（1）file：表示为外部命令<br>（2）alias：表示该命令为命令别名 alias 所设置的名称<br>（3）builtin：表示该命令为 bash 的内置命令 |

:::

## Shell 的变量功能

**变量的概念**：变量就是一个字符串，可以指向一个内容，该内容可以是路径、命令、数字、字符串等...

::: note 变量的分类（变量名称只能是英文字母和数字的组合，且不能以数字开头）

- 环境变量：或者叫做全局变量（通常以大写字符来表示）
- 自定义变量：或者叫做局部变量（建议使用小写字符）

:::

### 变量的基本使用：echo

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab 变量的定义与赋值
**定义**：直接使用变量名指向一个内容，该变量就即时创建/赋值：`变量名=变量所指内容`

- 等号两边不能有空格
- 变量的内容有空格时，使用双引号或者单引号括起来
- 变量所指的内容使用""（双引号）括起来，双引号内的特殊字符如&等，可以保持原有的特性
- 变量所指的内容使用''（单引号）括起来，单引号内的特殊字符仅为一般字符（纯文本）
- 可以使用\转义字符将特殊字符（ENTER、$、空格、'、\等）变为一般字符，类似于 C 语言
- 在一串指令的执行中，还需要借由其他额外的指令所提供的的信息时，可以使用反引号『\`指令\`』或『$(指令)』
- 若该变量为扩增变量内容时，可用『$变量名称』或『${变量名称}』累加内容
- 数组变量类型的定义：`数组名[索引]=内容`，数字从 0 开始，可以随意取，但建议有顺序
- 删除变量：`unset 变量名`

:::
::: tab "查看变量内容：echo"

- 第一种：`echo $变量名`
- 第二种(推荐)：`echo ${变量名}`
- 查看多个变量：`echo "${变量1},${变量2}..."`

echo 命令相关选项：

- `-e`：激活转义字符
- `-n`：显示的内容不换行

一个变量没有指向内容时，使用 echo 查看时会显示为空或出现错误信息
:::
::: tab "环境变量的定义与设置：export"
环境变量语法：`export 变量名`

- 使用 export 可以使该变量成为环境变量，使变量在子进程下也可以执行
- 普通变量类似于局部变量。环境变量类似于全局变量

**父进程和子进程的关系概念**：当我们登录 Linux 取得一个 bash 之后，这个 bash 就是一个独立的进程(拥有独特的 PID)，在这个 bash 下执行的任何命令都是由这个 bash 衍生出来的子进程

- 两者之间的关系：当我们在命令行执行 `bash` 这个命令之后，就会进入一个子进程。此时父进程处于等待状态，所执行的操作都在子进程中操作，输入 exit 或者 logout 可以退出这个子进程，恢复来到父进程当中
- 特点：子进程只会继承父进程的环境变量，而不会继承父进程的自定义变量。所以环境变量在子进程中可以使用，自定义变量不可以使用。此时就可以使用 export 命令设置环境变量
- 为什么环境变量可以被子进程所使用：当启动一个 shell 时，操作系统会分配一内存区给 shell 使用，使用 export 可以使自定义变量的内容写入上述的内存区域当中。当加载子进程时，子 shell 可以将父 shell 的环境变量所在的内存区域导入自己的环境变量区块当中

:::
::: tab "键盘读取变量：read"
要读取来自键盘输入的变量，就使用 `read` 这个指令。这个指令最常用在 shell script 的编写中

**指令语法**：`read [选项] 变量`

| 选项参数 | 说明                                           |
| :------: | ---------------------------------------------- |
|   `-p`   | 后面可以接提示字符串，交互功能                 |
|   `-t`   | 后面接用户输入字符串的等待的时间（以秒为单位） |

:::
::: tab 代码案例

```shell
var="lang is $LANG" # 双引号案例，输出为：lang is en_US.UTF-8
var='lang is $LANG' # 单引号案例，输出为：lang is $LANG
var=lang\\sbgong       # 转义案例，输出为：lang\sbgong
# 额外指令参数案例，相当于：/lib/modules/4.4.0-31-generic/kernel
cd /lib/modules/$(uname -r)/kerne
PATH="$PATH:/home/bin"  # 扩增变量内容案例
PATH="${PATH}:/home/bin"
var[0]="sbgong"     # 定义数组变量
var[1]="神臂弓"
echo "${var[0]},${var[1]}" # 查看定义的数组变量
unset var           # 删除数组变量
read atest # 让用户由键盘输入一内容，该内容变成 atest 的变量；可以使用echo ${atest} 查看
read -p "please write your name:" -t 30 myname  # 提示信息并只等待30秒用户输入
```

::: tab "declare"

- 单独使用 declare 时，与 set 命令功能相同可以查看所有的变量（含环境变量/与操作界面有关的变量/自定义变量）
  - 使用时直接输入 `declare`
- 可以定义一个变量的类型，以及更改变量的各种属性
  - 定义变量类型语法：`declare [选项] 变量名`

| 选项参数 | 说明                                                                         |
| :------: | ---------------------------------------------------------------------------- |
|   `-a`   | 将后面的变量定义成数组类型（array）                                          |
|   `-i`   | 将后面的变量定义为整数类型（integer）                                        |
|   `-x`   | 将后面的变量变为环境变量。与 export 命令功能相同                             |
|   `+x`   | 如果该变量为环境变量，可以将变量变为普通变量。与-x 相反                      |
|   `-r`   | 设置后面的变量为 readonly 类型，该变量不能被更改内容，也不能被 unset（删除） |
|   `-p`   | 可以列出该变量的类型                                                         |

注意事项：

- 变量的默认类型为字符串。所以不加任何选项，定义的变量为字符串
  - 即若不指定变量类型，『1+2』是一个字符串而不是计算式
- bash 环境中的数值运算，默认最多仅能达到整数形态。所以：1/3=0
- 如果使用-r 选项不小心把变量变为 readonly 模式，则需要注销再登录才能恢复该变量的类型

```shell
declare -i sum=1+1 # 将一个变量变为整型。如果不加declare -i定义的变量为字符串
echo ${sum}        # 输出：2
declare -x sum     # 将一个变量变为环境变量
export | grep sum  # 输出：declare -ix sum="2"
declare -r sum     # 将一个变量变为readonly类型
sum=sbgong         # 报错：因为该变量是一个只读变量
declare -p sum     # 查看sum变量的类型，输出：declare -irx sum="2"
declare | more     # 查看所有的变量（包含环境变量、与操作界面有关的变量与自定义变量）
```

### 语系变量：locale

- `locale -a`：查看当前 Linux 系统支持的语系
  - zh_CN.utf8：Unicode 的简体中文编码
- `locale`：可以查看各种语系变量所使用的语系
  - LANG 变量：代表主语言环境所使用的语系
  - LC_TYPE 变量：代表字符（文字）辨识的编码
  - LC_NUMERIC 变量：数字系统的显示数据

::: tip 语系变量的设置

- **前提**：我们设置语系变量时，一般建议更改 `LANG` 与 `LC_ALL` 这两个变量，因为设置这两个变量，其他语系变量都会替换成 LANG 与 LC_ALL 这两个变量的值。所以更改语系，一般建议修改 LANG 与 LC_ALL
- **注意事项**：设置语系环境变量，只在此次命令行执行中有效，退出后失效，又恢复成原样
- **方法**：LANG 直接用=号赋值即可。LC_ALL 需要使用 export 设置。我们设置 LANG=zh_CN.UTF-8，发现其他的变量都变成 zh_CN.UTF-8

:::

```shell
locale
LANG=en.US.utf8
export LC_ALL=en_US.utf8
```

### 限制用户的某些系统资源：ulimit

**ulimit 属 Shell 内置命令，用于查看或设置 Shell 以及通过 Shell 启动的进程使用系统资源的上限**

- 限制分为软限制（当前限制）和硬限制
  - 软限制是内核对相应资源强制执行的值，其中硬限制作为软限制的上限，即软限制值无法超过硬限制的值
- 如果应用程序在运行过程中使用的系统资源超过相应的软限制，将导致进程的终止

ulimit 可以限制用户的某些系统资源，包括可以开启的文件数量、可以使用的 CPU 时间、可以使用的内存总量等等。使用 ulimit 命令用于临时修改资源限制，如果需要永久修改需要将设置写入配置文件 `/etc/security/limits.conf`

**指令语法**：`ulimit [选项] [配额]`

| 选项参数 | 说明                                                                                                                                          |
| :------: | --------------------------------------------------------------------------------------------------------------------------------------------- |
|   `-H`   | 设定资源的硬限制，超出设定的值会报错                                                                                                          |
|   `-S`   | 设置资源的软限制，超出设定的值会告警                                                                                                          |
|   `-a`   | 显示目前所有资源设定的限制                                                                                                                    |
|   `-c`   | 当某些程序发生错误时，系统可能会将该程序在内存中的信息写成文件（排错用）。这种文件就被称为核心文件(core file)。此为限制每个核心文件的最大容量 |
|   `-d`   | 每个进程数据段的最大值                                                                                                                        |
|   `-f`   | 当前 shell 可创建的最大文件容量（单位 KB）                                                                                                    |
|   `-l`   | 可以锁定的物理内存的最大值（单位 KB）                                                                                                         |
|   `-m`   | 可以使用的常驻内存的最大值                                                                                                                    |
|   `-n`   | 每个进程可以同时打开的最大文件句柄数                                                                                                          |
|   `-p`   | 管道的最大值                                                                                                                                  |
|   `-s`   | 堆栈的最大值                                                                                                                                  |
|   `-t`   | 每个进程可以使用 CPU 的最大时间（单位为秒）                                                                                                   |
|   `-u`   | 每个用户运行的最大进程并发数                                                                                                                  |

### login shell 和 no-login shell

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab "login shell"

> login shell：取得 bash 时需要完整的登入流程的，就是 login shell。案例：由 tty1~tty6 登入，需要输入用户的账户和密码，此时取得的 bash 就称为 login shell

login shell 只会读取两个配置文件

- `/etc/profile`：这是系统整体的设置，你最好不要修改这个文件；不仅如此，该文件还能去调用外部的设定数据，在 CentOS7 默认的情况下，/etc/profile.d/\*.sh、/etc/locale.conf 和 /usr/share/bash-completion/completions/\* 都会被依次调用
  - `/etc/profile.d/*.sh`：这些文件规范了 bash 操作接口的颜色、语系、ll 与 ls 指令的命令别名、vi 的命令别名、which 的命令别名等。如果你需要帮所有的使用者设定一些共享的命令别名时，可以在这个目录底下自行建立扩展名为.sh 的文件，并将所需要的数据写入即可
  - `/etc/locale.conf`：这个文件是由 /etc/profile.d/lang.sh 调用的，这也是决定 bash 默认使用哪种语系的重要配置文件
  - `/usr/share/bash-completion/completions/*`：命令补齐、文件名补齐、指令选项参数补齐功能就是从这个目录里面找到相对应的指令来处理的。其实这个目录底下的内容是由 /etc/profile.d/bash_completion.sh 文件调用的
- `~/.bash_profile` 或 `~/.bash_login` 或 `~/.profile`：属于使用者个人设置，你要改自己的数据，就写入这里的文件内
  - bash 在读完了整体环境设置的/etc/profile 并借此调用其他配置文件之后，接下来就会读取自定义的个人配置文件。在 login shell 的 bash 环境中，所读取的个人偏好设置文件其实主要有三个，依序分别是：`~/.bash_profile`、`~/.bash_login`、`~/.profile`
  - 其实 bash 的 login shell 只会读取这三个文件中的一个，而读取顺序就是依照上面的顺序

| /etc/profile 文件内容 | 说明                                                    |
| :-------------------: | ------------------------------------------------------- |
|        `PATH`         | 会根据 UID 决定 PATH 变量要不要含有 sbin 的系统指令目录 |
|        `MAIL`         | 依据账号设定好使用者的 mailbox 到/var/spool/mail/账号名 |
|        `USER`         | 根据用户的账号设定此变量内容                            |
|      `HOSTNAME`       | 根据主机的 hostname 指令决定此变量的内容                |
|      `HISTSIZE`       | 历史命令记录笔数                                        |
|        `umask`        | 包括 root 默认为 022，一般用户为 002                    |

:::
::: tab "no-login shell"

no-login shell：取得 bash 接口的方法不需要重复登入的举动，就是 no-login shell。当你取得 no-login shell 时，该 bash 配置文件仅会读取 ~/.bashrc 文件

- 案例一：以 X window 登入 Linux 后，再以 X 的图形化接口启动终端机，此时那个终端接口并没有需要在此输入账号和密码，那个 bash 的环境就称为 no-login shell
- 案例二：在原来的 bash 环境下再次下达 bash 这个指令，同样的也没有输入账号密码，那第二个 bash（子程序）也是 no-login shell

另外 CentOS 7 还会主动调用 /etc/bashrc（CentOS/Red Hat 系统特有的），这个文件会帮 bash 定义出以下的数据：

- 依据不同的 UID 规范出 umask 的值
- 依据不同的 UID 规范出提示字符（就是 PS1 的变量）
- 调用 /etc/profile.d/\*.sh 的设定

如果你不小心删掉了 `~/.bashrc`，你会发现你的 bash 提示字符发生了变化。这是因为你没有 ~/.bashrc 调用 /etc/bashrc 来规范 PS1 变量。如果想要将命令提示字符变回来，可以复制 /etc/skel/.bashrc 到你的家目录，再修改成你想要的内容，然后使用 source 重新读入环境配置文件，那你的命令提示符就会回来了
:::
::: tab "其他配置文件"
还有一些配置文件可能会影响到你的 bash 操作

- `/etc/man_db.conf`：这个文件记录了『使用 man 的时候，man page 的路径到哪里去找』
- `~/.bash_history`：默认情况下，历史命令就记录在这里，而这个文件能记录几笔数据则跟 HISTFILESIZE 有关。每次登入 bash 后，bash 会先读取这个文件，将所有的历史指令都读入内存，因此当登入 bash 后就可以查询到上次使用过哪些指令
- `~/.bash_logout`：这个文件记录了『当用户注销后，系统再帮我做完什么动作后才离开』。默认情况下，注销时 bash 只是帮我们清掉屏幕的信息而已

:::
::: tab "读入环境配置文件：source"
由于 /etc/profile 与 ~/.bash_profile 都是在取得 login shell 的时候才会读取配置文件，所以如果你将自定义的设置写入上述文件之后，通常都得注销再登入后，该设定才能生效。如果想直接读取配置文件而不注销登录的话，就需要使用 `source` 这个指令了

## 数据流重定向

::: tip Linux 中有 3 种数据流

- 标准输入：输入信息会从 STDIN 中读取（STDIN,Standard Input）
- 标准输出：指令执行所回传的正确信息，会输出到 STDOUT（STDOUT,Standard Output）
- 标准错误输出：指令执行失败后，所回传的错误信息，会输出到 STDOUT（STDERR,Standard Error Output）

:::

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab 数据流重定向
Linux 中标准输入设备默认指键盘/鼠标，标准输出设备默认指屏幕/文件。重定向是『把数据重新传输到其他地方去』

|     流类型     | 操作符 |                                        用途                                        |
| :------------: | :----: | :--------------------------------------------------------------------------------: |
| 重定向标准输入 |  `<`   |                          输入的途径由键盘更改为指定的文件                          |
|                |  `<<`  |      操作符后面接结束输入的字符串；当由键盘输入该字符串时，就代表该次输入结束      |
| 重定向标准输出 |  `>`   | 将执行命令的正确结果输出到指定的文件中<br>如果文件不存在则新建；如果文件存在则覆盖 |
|                |  `>>`  | 将执行命令的正确结果输出到指定的文件中<br>如果文件不存在则新建；如果文件存在则追加 |
| 重定向标准错误 |  `2>`  | 将执行命令的错误结果输出到指定的文件中<br>如果文件不存在则新建；如果文件存在则覆盖 |
|                | `2>>`  | 将执行命令的错误结果输出到指定的文件中<br>如果文件不存在则新建；如果文件存在则追加 |

:::
::: tab 垃圾桶装置
垃圾桶装置 `/dev/null`：它可以吃掉任何导向这个装置的信息；例如可以将错误信息重定向到这里丢弃而不保存错误信息

```shell
find /home -name .bashrc 2> /dev/null # 将错误的信息丢弃，屏幕上显示正确的信息
find /home -name .bashrc > list 2>&1  # 将正确的信息和错误的信息全都写入一个文件内的写法；写法一
find /home -name .bashrc &> list      # 写法二
```

:::
::: tab 多指令执行的判断依据

多命令执行的判断依据：`;`、`&&`、`||`

|        选项参数        | 说明                                                                                                                     |
| :--------------------: | ------------------------------------------------------------------------------------------------------------------------ | --------- | ------------------------------------------------------------------------------------------------------------------------- |
|          `;`           | 分号前的指令执行完后就会立即执行后面的指令                                                                               |
| `command1 && command2` | 若 command1 执行完毕且正确执行（$?=0），则开始执行 command2<br>若 command1 执行完毕且执行错误（$?≠0），则不执行 command2 |
|       `command1        |                                                                                                                          | command2` | 若 command1 执行完毕且正确执行（$?=0），则 command2 不执行<br>若 command1 执行完毕且执行错误（$?≠0），则开始执行 command2 |

## 管道命令：pipe

管道命令使用的是 `|` 这个界定符号

- 管道命令只能处理前一个指令传来的正确信息，对于错误输出信息并没有直接处理的能力，它会忽略 STDERR 的信息
- 管道命令右边的命令，必须能够接收标准输入流命令才行
  - 作用接收标准输入的命令才可以用作管道右边。否则传递过程中数据会抛弃
  - 常用来作为接收数据管道命令有：sed,awk,cut,head,top,less,more,wc,join,sort,split 等等，都是些文本处理命令

### 内容截取命令：cut/grep

> 内容截取命令就是将一段数据经过分析后提取出我们想要的那些数据。一般情况下提取信息通常是针对一行一行来分析的

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab "cut"
cut 的意思就是切，它的用处就是将一段信息给它切出来。其处理的信息是以行为单位的

**指令语法**：

- `cut -d'分割字符' -f fields`
  - 注意：–d 和 –f 必须一起使用，否则命令将无法正确执行
  - 在对截取的数据显示的时候，各段数据之间默认用-d 指定的字符进行连接
- `cut -c 字符区间索引` 例如：cut -c 9-10

| 选项参数 | 说明                                                        |
| :------: | ----------------------------------------------------------- |
|   `-d`   | 后面接分割字符：用处是将一行数据以什么字符分割              |
|   `-f`   | 依据-d 的分隔符将一段信息分割成数段，用该选项指定取出第几段 |
|   `-c`   | 以字符的单位取出固定字符区间的数据                          |

-f 后面接的字符分一下几种情况

| 字符  | 说明                               |
| :---: | ---------------------------------- |
|  `N`  | 选取第 N 段数据输出，从 1 开始计数 |
| `N-`  | 从第 N 段数据开始直至最后一段      |
| `N,M` | 截取 N 和 M 两个字段的数据         |
| `N-M` | 从第 N 段数据开始直到第 M 段       |
| `-M`  | 从第 1 段直到第 M 段数据           |

:::
::: tab "grep"
grep 也是对一行数据进行分析，从分析的数据中取出我们想要的数据。grep 的功能要比 cut 强大的多了，grep 检索的条件是多种多样的，甚至还可以和正则表达式合作来检索

**指令语法**：`grep [选项] '搜寻字符串' 数据来源文件名`

|    选项参数    | 说明                                                               |
| :------------: | ------------------------------------------------------------------ |
|      `-a`      | 将 binary 文件以 text 文件的方式检索数据                           |
|      `-c`      | 计算找到『搜寻字符串』的次数                                       |
|      `-i`      | 对检索的的字符串不区分大小写                                       |
|      `-n`      | 顺便输出行号                                                       |
|      `-v`      | 反向选择，输出没有匹配到的字符串的那些行                           |
|      `-A`      | 后面可加数字，为 after 的意思；除了列出该行外，后续的 n 行也列出来 |
|      `-B`      | 后面可加数字，为 after 的意思；除了列出该行外，前面的 n 行也列出来 |
| `--color=auto` | 将找到的关键词部分加上颜色的显示                                   |

:::
::: tab 代码案例

```shell
# cut 案例
echo ${PATH} | cut -d ':' -f 5    # 取出PATH变量中的第五个路径
echo ${PATH} | cut -d ':' -f 3,5  # 取出PATH变量中的第三个和第五个路径
export | cut -c 12  # 将export输出的信息，截取第12个字符以后的所有字符串
# grep 案例
last | grep 'root'  # 将last当中，有出现root的那一行就截取出来
last | grep -v 'root'  # 与上面案例相反，只要没有root的就取出来
last | grep 'root' | cut -d ' ' -f 1  # 在last输出信息中，只要有root就取出，并且仅取第一栏
```

### 排序统计命令：sort/uniq/wc

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab "sort"
sort 可以依据不同的数据型态进行排序，例如数字与文字的排序就不一样。此外，排序的字符与语系编码有关，因此在需要排序时，建议使用 LANG=C，来让语系统一，数据排序比较好一点

**指令语法**：`sort [选项] [file or stdin]`

| 选项参数 | 说明                                                                                  |
| :------: | ------------------------------------------------------------------------------------- |
|   `-f`   | 忽略大小写，将所有的小写字母转换成大写字母进行比较                                    |
|   `-b`   | 排序时忽略前面的空格，默认情况下空格也会进行排序的                                    |
|   `-M`   | 以月份的名字来排序，例如：JAN、DEC 等排序方法                                         |
|   `-n`   | 使用『纯数字』进行排序，默认是以文字形态排序的                                        |
|   `-r`   | 将排序结果逆序                                                                        |
|   `-o`   | 将排序结果输出到文件中                                                                |
|   `-u`   | 去除重复项                                                                            |
|   `-t`   | 分隔符，将每一行数据按照该选项指定的分隔符分成多个域。默认情况下是使用 tab 键来分割的 |
|   `-k`   | 指定用哪个域/区间（field）的数据来进行排序                                            |

:::
::: tab "uniq"
uniq 命令，其主要功能是去除重复项

**指令语法**：`uniq [选项]`

| 选项参数 | 说明                   |
| :------: | ---------------------- |
|   `-i`   | 忽略大小写字符的不同   |
|   `-c`   | 进行计数，统计重复次数 |

:::
::: tab "wc"
wc 是一个统计命令，它可以计算一个文件里面有多少字，多少行，多少字符等

**指令语法**：`wc [选项] [file]`

- 在没有任何选项的情况下，wc 会统计三个数据：一是文件内容的行数，二是打印单词数量（单词是以 tab 键或者空格符为界定），三是字节数

| 选项参数 | 说明             |
| :------: | ---------------- |
|   `-l`   | 只统计行数       |
|   `-w`   | 只统计单词数量   |
|   `-m`   | 统计有多少个字符 |
|   `-c`   | 只统计字节数     |

:::
::: tab 代码案例

```shell
# sort 排序案例
cat /etc/passwd | sort -t ':' -k 3  # 以文件第三栏来进行排序
last | cut -d ' ' -f 1 | sort       # 利用last，截取出第一栏用户信息，然后加以排序
# uniq 去重案例
last | cut -d ' ' -f 1 | sort | uniq # 仅取出账号栏数据，进行排序之后去除重复数据
last | cut -d ' ' -f 1 | sort | uniq -c # 接上一个案例，统计重复次数
# wc 统计案例
cat /etc/man_db.conf | wc         # 输出的三个数字，分别代表：行、字数、字符数
```

### 双向重定向：tree

:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab "tree 指令语法"
tee 命令的功能是将数据重定向到文件中。我们知道数据重定向直接使用 > 就可以了。没错，>可以实现数据流重定向。但是它并不能使数据再导向标准输出。而 tee 命令可以，不仅可以将数据流重定向到文件中，还可以使数据再标准输出中输出到屏幕上

**指令语法**：`tree [-a] file`

- `-a`：以累加（append）的方式，将数据加入 file 当中

:::
::: tab 代码案例

```shell
last | tree last.list | cut -d ' ' -f 1 # 将 last 信息输出到 last.list 文件中
ls -l /home | tree ~/homefile | more    # 将数据保存到 ~/homefile 文件中，同时屏幕也有输出信息
ls -l /home | tree -a ~/homefile | more # -a 参数能够将信息累加（不加则意味着覆盖）
```

:::
::::

::: tip 重定向和管道命令的区别
简单地说，重定向操作符 `>` 将命令与文件连接起来，用文件来接收命令的输出；而管道符 `|` 将命令与命令连接起来，用第二个命令来接收第一个命令的输出
:::

### 参数代换：xargs


:::: tabs cache-lifetime="5" :options="{ useUrlFragment: false }"
::: tab 指令语法
**指令语法**：`xargs [选项] command`

- 当 xargs 后面没有接任何的指令时，默认是以 echo 来进行输出

| 选项参数 | 说明                                                                                                                                               |
| :------: | -------------------------------------------------------------------------------------------------------------------------------------------------- |
|   `-d`   | 更改分隔符（默认情况下，xargs 将换行符和空格作为分隔符，把标准输入分解成一个个命令行参数）                                                         |
|   `-0`   | 如果输入的 STDIN 含有特殊字符，例如`，\，空格键等字符时，这个-0 参数可以将它还原成一般字符，这个参数可用于特殊状态                                 |
|   `-e`   | 这个是 EOF（end of file）的意思，后面可以接一个字符串。当 xargs 分析到这个字符串时，就会停止继续工作                                               |
|   `-p`   | 打印出要执行的命令，询问用户是否要执行，用户输入 y 以后才会真正执行（由于使用 xargs 命令存在转换参数的过程，有时需要确认以下到底执行的是什么命令） |
|   `-t`   | 打印出最终要执行的命令，然后直接执行，不需要用户确认                                                                                               |
|   `-l`   | 如果标准输入包含多行，该参数可以指定多少行作为一个命令行参数                                                                                       |
|   `-n`   | 将同一行的几个命令行参数作为一次命令的输入参数（类似与-l）                                                                                         |

:::
::: tab "xargs指令说明"
> xargs 命令的作用是把标准输入转换为命令行参数

管道前面的命令的输出作为后面命令的标准输入。既然是标准输入，所以这里就有一个要求是后面的命令可以在标准输入中读取数据。这个条件还是比较苛刻的，在 linux 中并不是所有的命令都可以在标准输入设备中读取数据的，能读取的命令数量不多，除了在前面介绍的 cut、grep、wc 等命令之外，xargs 也是一个可以在标准输入中读取数据的命令。xargs 在标准输入中读取出数据，然后将这些数据作为那些不能从标准输入中读取数据的命令的输入。所以 xargs 命令，说白了，就是用来传递数据的

例如：`echo 'hello world' | xargs echo`。它是将管道左侧的标准输入，转换为命令行参数 hello world，然后传给第二个 echo 命令。真正执行的命令，紧跟在 xargs 后面，接受 xargs 传来的参数。再举一个例子 `echo 'one two three' | xargs mkdir` 等同于 `mkdir one two three`

xargs 后面的命令默认是 echo。大多数情况下，xargs 命令都是跟管道一起使用的，但它也可以单独使用。输入 xargs 按下回车以后，命令行就会等待用户输入，作为标准输入。你可以输入任意内容，然后按下 Ctrl + d，表示输入结束，这时 echo 命令就会把前面的输入打印出来

:::

### 关于减号（-）的用途

- 就是代表标准输出/标准输入, 视命令而定

`tar -cvf - /home | tar -xvf -` 这句的意思就是把/home 拷贝到当前目录下

- tar 命令后面跟着的 -cvf 这个减号是表示后面的字母是 tar 命令的选项
  - 其中 c 表示打包、v 表示显示执行过程、f 表示打包后生成的文件
- 然后紧跟着 f 后面的减号，在这里是指标准输出流
- 前半句的意思是把 /home 打包放到输出流中
- 接着用管道知符号 `|` 连接后半部分。前半部分输出的流做为后半部分输入的参数，就是那个流，这个流在这里用减号表示
- 后半部分也是 tar 命令，这次的选项 x 是解包。后半句完整的意思是把输入流-里的内容解包到当前目道录
- 整句的效果和 cp /home . 一样
