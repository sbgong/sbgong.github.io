滤镜、混合、裁剪和遮罩

# 滤镜：filter

滤镜函数可以有多个，之间以空格分隔，各滤镜按列出的顺序应用。因此对 filter: opacity(0.5) blur(1px) 来说，先应用不透明度，得到半透明效果，然后再做模糊处理。如果调换两个函数的位置，应用的顺序随之变化，即先对完全不透明的元素做模糊处理，然后再变成半透明

[none | blur() | opacity() | drop-shadow() |
grayscale() | sepia() | invert() | hue-rotate() |
brightness() | contrast() | saturate() | url() ]#

    滤镜 说明
    blur(<length>) 模糊滤镜；使用高斯模糊对元素的内容做模糊处理，标准偏差由 length 值定义，设为 0 时不对元素做模糊处理。标准偏差不能设为负值
    opacity([<number> / <percentage>]) 透明度滤镜；把透明度滤镜应用到元素上，与 opacity 属性十分相似，0 表示完全透明，1 或 100%则不对元素做任何改动。不允许使用负值。值可以大于 1 或 100%，不过计算最终值时将取整为 1 或 100%
    drap-shadow(<length>{2,3} <color>?) 投影滤镜；创建与元素的 alpha 通道形状一致的投影，带模糊效果，而且可以指定颜色。长度和颜色的处理方式与 box-shadow 属性一样，因此前两个 length 值可以为负数，而第三个（定义模糊半径）不可以。如未提供 color 值，使用的颜色与元素的 color 属性的计算值一样
    filter: blur(3px) drop-shadow(0.5em 0.7em 30px gray) opacity(0.5);
    1
    1
    颜色滤镜 说明
    grayscale([<number> / <percentage>]) 灰阶；把元素的颜色变成指定的灰阶。值为 0 时，元素没有任何变化，值为 1 或 100%时，元素完全变成灰度
    sepia([<number> / <percentage>]) 墨色调；把元素的原色变成指定的墨色调（sepia 是红褐色，等价于 sRGB 色彩空间中的 #704214 或 rgba(112,66,20)），值为 0 时，元素没有任何变化，值为 1 或 100%时，元素完全变成褐色
    invert([<number> / <percentage>]) 反相；把元素的所有颜色做反相处理。反相的方法是，拿 255 或 100%减去颜色的 R、G、B 通道的值，得到 0~255 或 0%~100%之间的值。值为 0 时，元素没有任何变化；值为 1 或 100%时，颜色完全反相；值为 0.5 或 50%时，在色彩空间的中点停止反相，得到均匀的灰色
    hue-rotate(<angle>) 旋转色相；在色轮上旋转色相，而饱和度和明度保持不变。值为 0deg 时，元素没有任何变化；值为 360deg 时，元素基本上也没有什么变化
    滤镜 说明
    brightness([<number> / <percentage>]) 调整元素上颜色的亮度。值为 0 时，元素为纯黑色；值为 1 或 100%时，元素没有任何变化；大于 1 和 100%的值得到的颜色比元素原有的颜色更亮，最终有可能变成纯白色
    contrast([<number> / <percentage>]) 调整元素上颜色的对比度。对比度越高，越容易区分颜色；对比度越小，颜色越接近。值为 0 时，元素变成纯灰色；值为 1 或 100%时，元素没有任何变化；大于 1 和 100%的值得到的颜色比元素原有的对比度大
    saturate([<number> / <percentage>]) 调整元素上颜色的饱和度。饱和度越高，颜色越鲜艳；饱和度越低，颜色越暗淡。值为 0 时，元素完全没有饱和度，得到灰度效果；值为 1 或 100%时，元素没有任何变化；值大于 1 或 100%的值，得到的结果是过渡饱和

    url(<uri>) uri 指向一个使用 SVG 句法定义的滤镜（具体而言是 <filter> 元素）

    • 引用的目标可以是只包含一个滤镜的 SVG 图像，例如：url(wavy.svg)，也可以是 SVG 图像中某个标识符对应的滤镜，例如：url(filters.svg#wavy)。后一种方式的优点是，可以在一个 SVG 中定义多个滤镜，这样便于加载、缓存和引用
    • 如果 url 函数指向不存在的文件，或者指向的 SVG 片段不是 <filter> 元素，那个这个函数不起作用，整个 filter 声明都将无效
    img.logo {filter: url(/assets/filters.svg#spotlight) grayscale(100%);}

    # 合成和混合：mix-blend-mode

    除了滤镜，CSS 还提供有确定如何把元素合成在一起的功能。比如说，由于定位，两个元素可能出现部分重叠。根据目前所学的知识，我们知道前面的部分会遮盖后面的部分。这种处理方式有时称为简单的 alpha 合成，如果前面的元素某些部分有 alpha 通道，而且值小于 1，我们就能看到后面的元素。熟悉 PS 的人可能知道，重叠的图层有多种混合模式，CSS 也提供了这样的功能

    在 CSS 中，混合策略有两种（截止 2017 年年末）：整个元素与背后的内容混合；把元素的多个背景层混合在一起

    重叠元素的混合方式可以使用 mix-blend-mode 属性设定

    normal | multiply | screen | overlay | darken | lighten | color-dodge | color-burn | hard-light | soft-light | difference | exclusion | hue | saturation | color | luminosity
    1
    1
    混合元素 说明
    初始值 normal
    适用于 所有元素
    继承性 否
    动画性 否
      混合模式 说明
      darken 前景中的每个像素与背着物中相应位置上的像素比较，比较 R、G、B 的值，保留较小的值。因此，如果前景中某个像素的颜色为 rgb(91,164,22)，背着物中对应位置的颜色为 rgb(102,104,255)，得到的结果是 rgb(91,104,22)
      lighten 比较前景与背着物相同位置上像素的 R、G、B 分量时，保留较大的值。因此，如果前景中某个像素的颜色为 rgb(91,164,22)，背着物中对应位置的颜色为 rgb(102,104,255)，得到的结果是 rgb(102,164,255)
      difference 比较前景与背着物相同位置上像素的 R、G、B 分量时，取两者之差的绝对值。因此，如果前景中某个像素的颜色为 rgb(91,164,22)，背着物中对应位置的颜色为 rgb(102,104,255)，得到的结果是 rgb(11,60,233)
      exclusion 这个模式是 difference 的温和版本。所用的公式不是 |back-fore|，而是 back+fore-(2*back*fore)，其中 back 和 fore 是介于 0~1 之间的值
      混合模式 说明
      multiply 前景中每个像素分量与背着物中对应位置上的像素分量相乘。得到的结果与前景相比更暗，这是由下面那一层修改的结果。这个混合模式是对称的，调换前景和背着物之后得到的结果完全一样
      screen 对前景和背着物同一位置上的像素分量做反相处理，相乘后再反相。得到的结果与前景相比更亮，这是由下面那一层修改的结果，与 multiply 一样，screen 是对称的
      overlay 这个混合模式是对 multiply 和 screen 的综合应用。如果前景中的像素分量比 0.5（50%）暗，执行 multiply 操作；如果大于 0.5，则使用 screen。可见，较暗的区域变得更暗，而较亮的区域变得更亮。这个混合模式不是对称的，因为调换前景和背着物之后，明暗度变了，应用正片叠底和滤色的方式也变了
      混合模式 说明
      hard-light 这个混合模式的作用与 overlay 相反。与 overlay 类似，这个混合模式也是对 multiply 和 screen 的综合应用，不过起决定作用的是背着物层。因此，背着物中比 0.5（50%）暗的像素分量使用 multiply 模式，而比 0.5 亮的则使用 screen 模式。这样得到的结果就像是投影仪使用刺眼的光线把前景投射到背着物上一样
      soft-light 这个混合模式是 hard-light 的柔和版本。也就是说，执行的操作是相同的，只不过效果有所缓和。这样得到的结果就是投影仪使用朦胧的光线把前景投射到背着物上一样

      颜色加深和颜色减淡是在尽量不改变颜色的前提下把相片变量和变暗

      混合模式 说明
      color-dodge 前景中各像素分量做反相处理，再拿背着物中对应的像素分量除以反相后的前景值。得到的结果是背着物变得更亮，除非前景值为 0，此时背着物没有任何变化
      color-burn 这个混合模式与 color-dodge 的作用相反，背着物中个像素分量做反相处理，反相后的背着物值除以没做任何修改的前景值，然后再反相。最终的结果是，背着物中越暗的像素，由前景中像素加深的程度越大
      混合模式 说明
      hue 把背着物中每一像素的明度和饱和度与前景中对应像素的色相角度合并
      saturation 把背着物中每一像素的色相角度和明度与前景中对应像素的饱和度合并
      color 把背着物中每一像素的明度与前景中对应像素的色相角度和饱和度合并
      luminosity 把背着物中每一像素的色相角度和饱和度与前景中对应像素的明度合并

      # 裁剪：clip-path

      如果想把元素的一部分剪掉，可以使用 clip-path 属性

      none | <url> | [[inset() | circle() | ellipse() | polygon()] ||
[border-box | padding-box | content-box | margin-box | fill-box | stroke-box | view-box]]
      1
      2
      1
      2
      独立混合 说明
      初始值 none
      适用于 所有元素
      继承性 否
      动画性 inset()、circle()、ellipse()、polygon()等
      p.cliped {clip-path: url(shapes.svg#cloud02);}
      1
      1
        裁剪形状 说明
        inset() 值为一到四个长度值或百分数，定义距范围框各边的偏移量。可以使用 round 关键字，或者另一组一到四个长度值或百分数定义圆角
        circle() 只接受一个值，为长度、百分数或关键字，定义圆的半径。在一或两个长度或百分数后可以使用 at 关键字指定圆心的位置
        ellipse() 值必须是两个长度、百分数或关键字，定义椭圆在纵轴和横轴上的半径。在一或两个长度或百分数后可以使用 at 关键字指定椭圆中心的位置
        polygon() 一系列以逗号分隔的 x 和 y 坐标值（二者之间以空格分隔），可用长度或百分数。在坐标值前面可用关键字定义多边形的填充规则
        裁剪框 说明
        margin-box 在外边距的外边界裁剪元素
        border-box 在边框的外边界裁剪元素
        padding-box 在内边距的外边界裁剪元素
        content-box 在内容区的外边界裁剪元素
        fill-box 使用最近的 SVG 视区（即最近的祖辈）作为裁剪框
        stroke-box 使用对象范围框作为裁剪框。对象范围框指能放下元素几何尺寸范围内所有部分的最小框体，包括变形（例如旋转），但不包括外围的描边
        view-box 使用描边范围框作为裁剪框。描边范围框指能放下元素几何尺寸范围内所有部分的最小框体，包括变形（例如旋转），也包括外围的描边

        # 裁剪填充规则：clip-rule

        裁剪填充规则 说明
        取值 nonzero / evenodd
        初始值 nonzero
        适用于 所有元素
        继承性 否
        动画性 否
        • nonzero：填充元素内部的所有区域
        • evenodd：留出一部分不填充

        # 定义蒙版：mask-image

        蒙版：位于形状内部的内容可见，而在形状外部的内容不可见

        应用蒙版的第一步是指定使用什么图像作为蒙版，这一步使用 mask-image 属性完成，其值可以是任何类型的图像

        蒙版图像 说明
        取值 [none / <image> / <mask-source>]#
        初始值 none
        适用于 所有元素
        继承性 否
        动画性 否

        # 改变蒙版的模式：mask-mode

        目前所见的蒙版都是把一个有 alpha 通道的图像应用到另一个元素上，这只是使用图像蒙版的两种方式之一。另一种方式是使用图像各部分的亮度定义蒙版。这两种方式使用 mask-mode 属性切换

        蒙版模式 说明
        取值 [alpha / luminosity / match-source]#
        初始值 match-source
        适用于 所有元素
        继承性 否
        动画性 否

        # 调整蒙版的尺寸：mask-size

        初始值 auto
        适用于 所有元素
        继承性 否
        动画性 <length>,<percentage>

        初始值 repeat
        适用于 所有元素
        继承性 否
        动画性 是

        适用于 置换元素
