## X window 与文本模式的切换

- 通常我们也称文本模式为终端机接口（Terminal 或 Console），Linux 预设的情况下会提供 6 个 Terminal 来让使用者登入，切换的方式为使用：<kbd>Ctrl</kbd> + <kbd>Alt</kbd> + <kbd>[F1]~[F6]</kbd> 的组合按键
- 注销 Linux 直接输入 `exit` 即可离开

系统会将[F1]~[F6]命名为 `tty1` ~ `tty6`的操作接口环境。在 CentOS7 环境下，当开机完成之后，默认系统只会提供给你一个 tty 而已，因此无论是文字界面还是图形界面，都是会出现在 tty1 中。tty2 ~ tty6 其实一开始是不存在的，但是当你要切换时（按下切换的组合键），系统才产生出额外的 tty2,tty3……

若你在纯文本环境中启动 X window，那么图形界面就会出现在当时的那个 tty 上面。举例来说，当你在 tty3 登入系统，然后输入 `startx` 启动个人的图形界面，那么这个图形界面就会产生在 tty3 上面

## 文本模式下的指令格式

![指令格式](https://cdn.jsdelivr.net/gh/sbgong/static/blog/server/linux/202205021433660.png)

::: tip 指令详细说明

- 一行指令中第一个输入的部分就是『指令 command』或『可执行文件，例如批次脚本 script』
- command 为指令的名称，例如切换工作目录的指令为 cd 等
- 中括号并不存在于实际的指令中
  - 加入选项设定时，通常选项前会带 `-`，例如：`-h`
  - 如果要带选项的完整全名，则选项前会带 `--`，例如：`--help`
- parameter1 parameter2 为依附在选项后面的参数，或者是 command 的参数
- 指令、选项、参数这几个内容中间以空格分隔，不论空几格 shell 都视为一格
- 按下『Enter』键后，该指令就立即执行
- 指令太长的时候，可以使用反斜杠『\』来跳脱『Enter』符号，使指令连续到下一行
- 在 Linux 系统中，英文大小写字母敏感，cd 和 CD 不同
- 指令后面的选项除了前面带有减号『-』之外，某些特殊情况下，选项或参数前面也会带有正号『+』的情况

:::

## 在线求助

### --help

> --help 用在协助你查询你曾经用过的指令所具备的选项和参数

事实上，几乎 Linux 上面的指令，在开发的时候，开发者就将可以使用的指令语法与参数写入指令操作过程中了。你只需要使用『--help』这个选项，就能够将该指令的用法做一个大致的了解

既然有 man page，自然就是因为有一些文件数据，所以才能够以 man page 读出来。那么这些 man page 的数据根据不同的 distribution，存放的地址会有些许差异，不过通常放在 `/usr/share/man` 这个目录中。当然这个路径也是可修改的：修改 `/etc/man_db.conf`即可（有的版本是 man.conf 或 manpath.conf 或 man.config 等）

**『man -f Command』可以找到更多有关这个指令的说明文件**

- 从输出的结果来看：
  - 左边部分：指令（或文件）以及该指令所代表的意义（就是那个数字）
  - 右边部分：这个指令建议说明
- 当使用该指令时，man 只会找数据中的左边那个指令（或文件）的完整名称，有一点不同都不行

::: info 案例说明
例如 `man -f man`，便可查出 man 指令的多个说明文件

- 使用 `man 1 man` 即可查看 man(1)的文件数据
- 使用 `man 7 man` 即可查看 man(7)的文件数据

如果只输入 `man man`，那么取出的数据就跟搜寻的顺序有关了，搜寻的顺序是记录在 `/etc/man_db.conf`这个配置文件中的，先搜寻到的那么说明文档就会先被显示出来。一般来说，通常会先找到数字较小的那个（因为排序的原因）。所以 `man man`会跟 `man 1 man`结果相同
:::

**『man -k Command』通过关键词来搜索更多有关这个指令的说明文件**

- 不同于『man -f Command』指令的全部匹配，『man -k Command』实现的是关键词的搜索
- 只要含有该指令那个字眼的（不见得是完整字符串）就都会显示出来

**以上两种搜索的缩写形式**

- `whatis` 是 `man -f`的缩写
- `apropos` 是 `man -k`的缩写

注意：这两个特殊指令要使用的前提是必须要有建立 whatis 数据库才行，这个数据库的建立需要以 root 的身份输入`mandb`指令，旧版则是`makewhatis`

### info page

在所有的 Unix Like 系统当中，都可以利用 man 来查询指令或者是相关文件的用法。但是在 Linux 里则又提供了一种在线求助的方法，那就是利用 info 这个命令

基本上 info 与 man 的用途差不多，都是用来查询指令的用法或者文件的格式。但是与 man page 一口气输出一堆信息不同，info page 则是将文件数据拆分成一个一个的段落，每个段落用自己的页面来编写，并且在各个页面中还有类似网页的超链接来调到各不同的页面中，每个独立的页面也被称为一个节点（node）。所以，你可以将 info page 想成是文本模式的网页显示数据

不过你要查询的目标数据的说明文件必须要以 info 的格式来写成才能够使用 info 的特殊功能。而这个支持 info 指令的文件默认是放置在 `/usr/share/info/` 这个目录当中的。举例来说，info 这个指令的说明文件有写成 info 格式，所以你使用『info info』可以得到如下画面

![info info](https://cdn.jsdelivr.net/gh/sbgong/static/blog/server/linux/202205021435164.png)
::: tip 界面详解

**第一行显示了很多信息，你可以从中了解这个节点的内容、来源与相关链接的信息。此外你还能通过『N P U』来到下一个、上一个和上一层的节点**

- File：代表这个 info page 的数据来源是 info.info 文件所提供的

- Up：回到上一层的节点总览画面，你也可以按『U』回到上一层

## 开机和关机

Windows 是单人多任务操作系统，而 Linux 则是多人多任务。因此在你看不到的屏幕背后其实可能有相当多人同时在一台主机上面工作。所以正常情况下，要关机需要注意以下几件事

- 观察系统的使用状态，通过这些信息判断是否可以关机了
  - 如果要查看目前有谁在线，可以使用『who』这个指令
  - 如果要查看网络的联机状态，可以使用『netstat -a』这个指令
  - 如果要查看进程情况，可以使用『ps -aux』这个指令
- 通知在线使用者关机的时刻：要关机总得给在线的使用者一些时间来结束他们的工作，这个时候可以使用 shutdown 的特别指令来达到这一功能
- 正确的关机指令使用：例如 `shutdown` 和 `reboot` 这两个指令

::: info 开机/关机相关指令

- 将数据同步写入磁盘的指令：`sync`
- 惯用的关机指令：`shutdown`
- 重新启动、关机：`reboot`、`halt`、`poweroff`

:::

### 数据同步写入磁盘：sync

由于所有的数据都得要被读入内存后才能够被 CPU 所处理，但是数据又常常需要由内存写回硬盘当中（例如储存的动作）。由于硬盘速度太慢（相对内存来说），如果常常让数据在内存与硬盘中来回写入/读出，系统的效能就不会太好。
因此在 Linux 系统中，为了加快数据的读取速度，某些已经加载到内存中的数据将不会直接被写回硬盘，而是先暂存在内存当中。如此一来，如果一个数据被你重复的改写，那么由于它尚未被写入硬盘当中，因此可以直接由内存当中读取出来，在速度上一定是快上很多
不过这也会造成些许困扰，那就是万一你的系统因为某些特殊情况造成不正常关机，由于数据尚未被写入硬盘当中，所以就会造成数据的更新不正常。这个时候就需要 `sync` 这个指令来进行数据的写入动作了。直接在 Terminal 下输入 sync，那么在内存中尚未被更新的数据，就会被写入硬盘中。**所以这个指令在系统关机或重新启动之前多执行几次**，虽然目前的 shutdown/reboot/halt 等指令均已经在关机前进行了 sync 的执行了，不过多执行几遍，比较放心

一般情况下，一般账户使用 sync 指令只会更新自己账户的硬盘数据，root 账户则会更新整个系统的数据。因此最好使用 root 账户做同步更新

### 关机指令：shutdown

- 如果是在主机前面以实体终端机（tty1~tty7）来登入系统时，不论用什么身份都能够关机
- 如果是使用远程管理工具来登入主机，那关机就只有 root 有权力

shutdown 指令会通知系统内的各个程序，并且将通知系统中的一些服务来关闭。shutdown 可以达成如下工作：

- 可以自由选择关机模式：是要关机还是重新启动
- 可以设定关机时间：可以设定成现在立刻关机，也可以设定某一个特定的时间才关机
- 可以自定义关机讯息：在关机前，可以将自己设定的信息传送给在线的 user
- 可以仅发出警告讯息：有时有可能你要进行一些测试，而不想让其他的使用者干扰，或者是明白的告诉使用者某段时间要注意以下。这个时候可以使用 shutdown 来吓一吓使用者，并不是真的要关机

::: info shutdown 具体语法
**/sbin/shutdown [-krhc] [时间] [警告讯息]**
| 选项与参数 | 说明 |
| ---------- | ----------------------------------------------------------- |
| -k | 不是真的要关机，只是发送警告讯息出去 |
| -r | 在将系统的服务停掉之后，重新启动（**常用**） |
| -h | 将系统的服务停掉之后，立即关机（**常用**） |
| -c | 取消已经在进行的 shutdown 指令内容 |
| 时间 | 指定系统的关机时间，若没有指定时间，则默认 1 分钟后自动进行 |

案例：`/sbin/shutdown -h 10 'I will shutdown after 10 mins'`

- 在执行 shutdown 之后，系统会告诉大家，这部机器将会在 10 分钟之后关机，这段信息将会显式在目前登入者的屏幕前方
- 你可以输入 `shutdown -c` 来取消这次的关机指令
- 而如果你什么参数也没有添加，单纯执行 shutdown 之后，系统默认会在 1 分钟后进行关机动作

:::

提供几个常见的时间参数：

| 选项与参数                  | 说明                                                             |
| --------------------------- | ---------------------------------------------------------------- |
| `shutdown -h now`           | 立即关机，其中 now 相当于时间为 0 的状态                         |
| `shutdown -h 20:25`         | 系统在今天的 20:25 关机，如果在 21:25 才下达此命令，则隔天才关机 |
| `shutdown -h +10`           | 系统再过 10 分钟后自动关机                                       |
| `shutdown -r row`           | 系统立即重新启动                                                 |
| `shutdown -r +30 'message'` | 再过 30 分钟会重新启动，并显示后面的讯息给所有在线的使用者       |
| `shutdown -k now 'message'` | 仅发出警告信息，系统并不会关机                                   |

### 重新启动、关机：reboot,halt,poweroff

基本上，在预设的情况下，这几个指令都会完成一样的工作（全部的动作都是去调用 `systemctl`这个重要的管理命令），所以你只要记得其中一个就好了

- reboot：重新启动
- halt：进入系统停止的模式，屏幕可能会保留系统已经停止的信息，这与你的电源管理模式有关
- poweroff：系统关机，所以没有提供额外的电力，屏幕空白
- suspend：进入休眠模式

目前系统中所有服务的管理指令是 systemctl，上面谈到的 reboot,halt,poweroff,shutdown 等，其实都是调用了这个 systemctl 指令。systemctl 指令基础语法：`systemctl [指令]`。例如：systemctl reboot 就代表系统重新启动
